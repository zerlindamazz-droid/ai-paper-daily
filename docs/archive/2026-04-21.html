Dive into Claude Code: The Design Space of Today's and Future AI Agent Systems 深入Claude Code：当代与未来AI智能体系统的设计空间

🧑‍🔬 Jiacheng Liu, Xiaohan Zhao, Xinyi Shang, Zhiqiang Shen

图 Figure · Page 7 of paper

🎯 拆解Claude Code架构，揭示AI智能体系统设计的核心原则与未来方向。
A systematic architectural teardown of Claude Code reveals how five human values drive thirteen design principles in real-world AI agent systems.
维度 中文 English
❓ 待解决
的问题		 随着Claude Code这样的AI编程智能体被大规模部署，业界对其复杂架构的设计原理缺乏系统性理解。开发者在构建自己的智能体系统时，没有清晰的参考来权衡安全性、能力与上下文管理。此外，也缺乏一个比较框架来理解相同的设计问题在不同部署场景下为何会产生截然不同的架构答案。 As AI coding agents like Claude Code become widely deployed in real software workflows, the field lacks a rigorous, principled understanding of how their complex architectures are actually designed and why specific choices are made. Practitioners building their own agent systems have no clear reference for how safety, capability, and context management trade-offs are resolved in production systems. There is also no comparative framework for understanding how the same fundamental design questions yield different answers in different deployment contexts.
💡 解决
的亮点		 - **价值观到架构的追溯**：首次系统性逆向工程生产级AI智能体（Claude Code），将五大核心人类价值观映射到13条设计原则，再到具体实现，基于公开TypeScript源码。 - **双系统对比分析**：与独立开源智能体OpenClaw的交叉对比，揭示部署场景如何从根本上塑造架构——例如逐动作ML安全分类 vs. 边界级访问控制。 - **完整子系统梳理**：记录了核心while循环之外被低估的复杂性：7种权限模式（含ML分类器）、5层上下文压缩流水线、4种可扩展机制及沙箱隔离的子智能体委托。 - **Value-to-Architecture Tracing**: First systematic effort to reverse-engineer a production AI agent (Claude Code) by mapping five core human values → 13 design principles → concrete implementation choices, using publicly available TypeScript source code. - **Dual-System Comparison**: Cross-analysis with OpenClaw (an independent open-source agent) reveals how deployment context fundamentally shapes architecture — e.g., per-action ML safety classification vs. perimeter-level access control. - **Comprehensive Subsystem Mapping**: Documents underappreciated complexity around the central while-loop: a 7-mode permission system with ML classifier, 5-layer context compaction pipeline, 4 extensibility mechanisms, and worktree-isolated subagent delegation.
🧪 实验
内容		 本文是系统分析与架构研究，而非标准的机器学习实验论文。"实验"包括：(1) 对Claude Code TypeScript源代码进行结构化分析，分解各子系统；(2) 在权限模型、上下文管理、可扩展性、子智能体委托等关键维度上，对Claude Code与OpenClaw进行对比架构分析；(3) 综合近期实证与政策研究文献，为六个未来设计方向提供依据。 This is a systems analysis and architectural study rather than an empirical ML paper, so there are no standard benchmark experiments. The "experiments" consist of: (1) a structured code-level analysis of Claude Code's TypeScript source, decomposing it into subsystems; (2) a comparative architectural analysis between Claude Code and OpenClaw across key design dimensions such as permission models, context management, extensibility, and subagent delegation; and (3) a literature survey grounding six future design directions in recent empirical and policy research.
📊 实验
结果		 作为架构分析论文，本文不含基准测试数字或性能提升百分比。主要定性发现包括：Claude Code的权限系统支持7种模式并集成ML安全分类器；上下文管理采用5层压缩流水线；通过4种机制（MCP、插件、技能、钩子）提供可扩展性；子智能体委托使用worktree隔离保障安全。与OpenClaw对比：Claude Code采用逐动作ML分类，OpenClaw采用边界级访问控制；Claude Code运行单一CLI循环，OpenClaw将智能体运行时嵌入网关控制平面；Claude Code的上下文窗口扩展对应OpenClaw的全网关能力注册。此外归纳出6个未来系统的开放设计方向。 As an architectural analysis paper, there are no benchmark numbers or percentage improvements. Key qualitative findings include: Claude Code's permission system supports 7 distinct modes and incorporates an ML-based safety classifier; its context management uses a 5-layer compaction pipeline; extensibility is provided via 4 mechanisms (MCP, plugins, skills, hooks); subagent delegation uses worktree isolation for safety. Compared to OpenClaw: Claude Code uses per-action ML classification while OpenClaw uses perimeter-level access control; Claude Code runs a single CLI loop while OpenClaw embeds an agent runtime within a gateway control plane; context-window extensions in Claude Code contrast with gateway-wide capability registration in OpenClaw. Six open design directions are identified for future systems.
🏁 结论 本文首次对Claude Code进行全面的、以原则为基础的架构分析，证明其看似复杂的代码库围绕五大人类价值观和13条设计原则组织而成，并通过与OpenClaw的对比研究揭示了部署场景如何从根本上决定智能体通用设计问题的架构答案。研究既为智能体系统构建者提供了实践参考，也为下一代AI智能体系统提出了六个开放研究方向。 This paper delivers the first comprehensive, principle-grounded architectural analysis of Claude Code, showing that its seemingly complex codebase is organized around five human values expressed through thirteen design principles, and that a comparative study with OpenClaw reveals how deployment context fundamentally determines architectural answers to universal agent design questions. The work provides both a practical reference for agent system builders and a research agenda of six open directions for the next generation of AI agent systems.

⚙️ Method · 方法详解

作者对Claude Code公开的TypeScript源代码进行深度解读，将其架构分解为各子系统，并将每个设计决策追溯至五大人类价值观（人类决策权威、安全保障、可靠执行、能力扩展、情境适应性）。这些价值观被形式化为13条设计原则，并映射到具体实现。同时并行分析OpenClaw（一个多渠道个人助手网关），作为对照案例研究，展示相同设计问题在不同部署场景下的不同解答。最后，作者结合近期实证、架构与政策文献，归纳出未来智能体系统的六个开放设计方向。 The authors perform a close reading of Claude Code's publicly available TypeScript source code, decomposing its architecture into its constituent subsystems and tracing design decisions back to five motivating human values (human decision authority, safety and security, reliable execution, capability amplification, and contextual adaptability). These values are then formalized into thirteen design principles and mapped to specific implementation choices. A second system, OpenClaw — a multi-channel personal assistant gateway — is analyzed in parallel to provide a contrastive case study showing how the same design questions (e.g., permission management, context handling, extensibility) are answered differently when the deployment environment changes. Finally, the authors synthesize six open design directions for future agent systems by drawing on recent empirical, architectural, and policy literature.

📐 Key Formulas · 关键公式

\[\text{Agent Loop: } s_{t+1} = \text{Tool}(\text{Model}(s_t, \text{context}_t))\]
💡 智能体核心循环：在每个时间步，模型根据当前状态和上下文生成动作，工具执行该动作并产生新状态，循环往复。 The core agent loop: at each step, the model takes the current state and context as input, produces an action, a tool executes it to yield the next state, and the cycle repeats — the fundamental while-loop at Claude Code's heart.

🌟 Why It Matters · 为什么重要

随着AI智能体系统在软件工程等领域广泛普及，有原则的架构指导极为匮乏——本文通过对主流生产级智能体的严格源码级解析填补了这一空白，将隐性设计智慧显式化并使其可迁移。与OpenClaw的对比框架及六个未来设计方向，为研究者和从业者构建更安全、更强大、更具适应性的智能体系统提供了共同语言与路线图。 As AI agent systems proliferate across software engineering and beyond, principled architectural guidance is scarce — this paper fills that gap by providing the first rigorous, source-code-grounded anatomy of a leading production agent, making implicit design wisdom explicit and transferable. The comparative framework with OpenClaw and the six future design directions give researchers and practitioners a shared vocabulary and roadmap for building safer, more capable, and more adaptable agent systems.

Towards Ultra-High-Rate Quantum Error Correction with Reconfigurable Atom Arrays 面向可重构原子阵列的超高码率量子纠错

🧑‍🔬 Chen Zhao, Casey Duckering, Andi Gu, Nishad Maskara et al.

图 Figure · Page 3 of paper

🎯 面向中性原子硬件协同设计的超高码率（>1/2）量子纠错码，接近teraquop级错误率。
Ultra-high-rate quantum error correction codes (rate >1/2) co-designed for neutral atom hardware approach teraquop error levels.
维度 中文 English
❓ 待解决
的问题		 量子计算机需要纠错才能大规模可靠运行，但现有纠错码需要大量"冗余"量子比特——通常是实际逻辑比特数的10倍甚至更多。量子低密度奇偶校验（LDPC）码虽然有望减少开销，但实际实现中编码率（有效比特数/总比特数）很少超过1/10，且难以在真实硬件上高效运行。 Quantum computers need error correction to run reliably at scale, but current error-correcting codes require a huge number of extra "overhead" qubits — often 10x or more than the logical qubits you actually want to use. While quantum low-density parity-check (LDPC) codes promise to reduce this overhead, practical implementations rarely achieve encoding rates (useful qubits / total qubits) above 1/10, and it's unclear how to run them efficiently on real hardware.
💡 解决
的亮点		 • 发现仿射置换矩阵的新结构条件，使编码率突破1/2，远超实用LDPC码通常的约1/10水平。 • 代码族与可重构中性原子阵列协同设计，确保症状提取与原子重排满足现实并行控制约束。 • 引入兼顾高解码精度与高吞吐量的分层解码器，在电路级噪声下实现实际可用的解码性能。 • Novel structural conditions on affine permutation matrices are identified, enabling encoding rates exceeding 1/2 — far above the typical ~1/10 seen in practical LDPC codes. • The code family is co-designed with reconfigurable neutral atom arrays, ensuring syndrome extraction and atom rearrangement fit within realistic parallel hardware constraints. • A hierarchical decoder is introduced that balances high decoding accuracy with good throughput, enabling practical use under circuit-level noise.
🧪 实验
内容		 在物理错误率p=0.1%的电路级噪声模型下评估代码，模拟真实的门操作、测量和空闲错误。对两个具体代码实例进行基准测试：[[2304, 1156, ≤14]]码和[[1152, 580, ≤12]]码。核心指标为每逻辑比特每轮错误率，目标是接近"teraquop区间"（约10⁻¹²），这是实用容错量子计算的重要门槛。无经典ML数据集基线，性能对比隐含地与量子LDPC码领域现有最优水平相比较。 The codes were evaluated under a circuit-level noise model at a physical error rate of p=0.1%, simulating realistic gate, measurement, and idle errors. Two specific code instances were benchmarked: a [[2304, 1156, ≤14]] code and a [[1152, 580, ≤12]] code. The key metric is the per-logical-qubit-per-round error rate, and the goal is to approach the "teraquop regime" (error rates around 10⁻¹²), which represents a threshold for practically useful fault-tolerant computation. No classical ML dataset baselines are used; comparisons are implicit against the state of the art in quantum LDPC code performance.
📊 实验
结果		 • [[2304, 1156, ≤14]]码：在p=0.1%噪声下，每逻辑比特每轮错误率为1.3⁺³·⁰₋₀.₉ × 10⁻¹³，进入teraquop区间。 • [[1152, 580, ≤12]]码：错误率为2.9⁺³·¹₋₁.₅ × 10⁻¹¹。 • 编码率超过1/2（如1156/2304 ≈ 0.502），大幅优于实用量子LDPC码通常约1/10的水平。 • 两种代码均同时支持500余个逻辑比特，展现出高度并行性。 • [[2304, 1156, ≤14]] code: per-logical-per-round error rate of 1.3⁺³·⁰₋₀.₉ × 10⁻¹³ at p=0.1% — entering the teraquop regime. • [[1152, 580, ≤12]] code: per-logical-per-round error rate of 2.9⁺³·¹₋₁.₅ × 10⁻¹¹ at p=0.1%. • Encoding rates exceed 1/2 (e.g., 1156/2304 ≈ 0.502), drastically outperforming the ~1/10 typical of practical quantum LDPC codes. • Both codes support >500 logical qubits simultaneously, demonstrating high parallelism.
🏁 结论 本文证明，超高码率量子纠错码（编码率>1/2）可在可重构中性原子硬件上实际实现，并达到接近teraquop水平的逻辑错误率，大幅降低量子比特开销。研究建立了一种新的协同设计范式，将代数码构造与现实硬件约束相结合，推动容错量子计算向实用化迈进。 This paper demonstrates that ultra-high-rate quantum error-correcting codes (rate >1/2) can be practically implemented on reconfigurable neutral atom hardware while achieving near-teraquop logical error rates, drastically reducing qubit overhead. It establishes a new co-design paradigm bridging algebraic code construction and realistic hardware constraints for fault-tolerant quantum computing.

⚙️ Method · 方法详解

作者在Kasai近期超高码率构造的基础上，发现了使这类码与中性原子硬件兼容的新代数条件（针对仿射置换矩阵）。他们将码结构与可重构原子阵列架构协同设计，使所需的量子比特连接性、症状提取电路和原子移动均满足现实硬件约束。解码方面，他们开发了兼顾高精度与实用吞吐量的分层解码器。整个流程——代码、硬件布局和解码器——在电路级噪声模型下进行评估，以模拟真实错误场景。 The authors build on a recent ultra-high-rate code construction by Kasai and identify new algebraic conditions (on affine permutation matrices) that make these codes compatible with neutral atom hardware. They co-design the code structure with the reconfigurable atom array architecture so that the required qubit connectivity, syndrome extraction circuits, and atom movement all fit within realistic hardware constraints. For decoding, they develop a hierarchical decoder that combines high accuracy with practical throughput. The entire pipeline — code, hardware layout, and decoder — is evaluated under a circuit-level noise model to simulate realistic error conditions.

📐 Key Formulas · 关键公式

\[[[n, k, d]] \text{ code with } k/n > 1/2\]
💡 量子码参数：n为总物理比特数，k为逻辑比特数，d为码距。编码率k/n > 1/2意味着超过一半的物理比特直接编码有用信息，远超传统纠错码。 Quantum code parameters: n is total physical qubits, k is logical qubits, d is code distance. Rate k/n > 1/2 means more than half of all physical qubits encode useful information — far better than typical codes.
\[\epsilon_{\text{logical}} = 1.3_{-0.9}^{+3.0} \times 10^{-13} \text{ per logical per round}\]
💡 [[2304,1156,≤14]]码在物理错误率p=0.1%下的每逻辑比特每轮错误率，接近teraquop区间（约10⁻¹²），表示每万亿次操作约发生一次逻辑错误。 Per-logical-qubit per-round error rate for the [[2304,1156,≤14]] code at p=0.1% physical noise, approaching the teraquop regime — roughly one logical error per trillion operations.

🌟 Why It Matters · 为什么重要

量子比特开销是构建实用量子计算机最大的现实障碍之一，本研究提供了将开销降低5倍甚至更多的可信路径，同时保持超低错误率。对于更广泛的AI/ML领域，容错量子计算的进步最终有望加速经典硬件难以高效运行的量子机器学习与优化算法。 Qubit overhead is one of the biggest practical barriers to building useful quantum computers — this work shows a credible path to slashing that overhead by 5x or more while maintaining ultra-low error rates. For the broader AI/ML community, advances in fault-tolerant quantum computing could eventually accelerate quantum machine learning and optimization algorithms that classical hardware cannot efficiently run.

Learning Whole-Body Humanoid Locomotion via Motion Generation and Motion Tracking 通过运动生成与运动跟踪学习全身人形机器人运动

🧑‍🔬 Zewei Zhang, Kehan Wen, Michael Xu, Junzhe He et al.

图 Figure · Page 3 of paper

🎯 扩散模型生成参考动作+RL跟踪器，实现地形感知的全身人形机器人运动
A diffusion-based motion generator plus RL tracker enables terrain-aware whole-body humanoid locomotion on real hardware.
维度 中文 English
❓ 待解决
的问题		 人形机器人需要在复杂地形中协调全身运动（不只是腿），这非常困难。单纯用奖励塑形的强化学习容易只优化腿部动作；而模仿学习虽然能学全身协调，但只会回放固定参考动作，无法根据实时感知的地形灵活调整。目前没有一个框架能同时做到：全身协调运动 + 在线地形感知适应。 Humanoid robots have highly complex bodies and need to coordinate their entire body — not just their legs — while navigating uneven terrain in real time. Pure reward-shaping RL tends to produce leg-only behaviors, while imitation-based RL can learn whole-body skills but is stuck replaying fixed reference motions and can't adapt to new terrain. There's no existing framework that combines coordinated whole-body motion with online, perception-driven terrain adaptation.
💡 解决
的亮点		 • **创新两阶段框架**：先用扩散模型从人类动作数据实时生成地形感知参考动作，再用RL全身跟踪器执行，生成与控制职责分离，设计清晰。 • **闭环微调策略**：冻结运动生成器后，在闭环中对跟踪器进行微调，使其对不完美或分布外的参考动作更鲁棒。 • **真实硬件全流程部署**：系统完全在Unitree G1人形机器人上运行，使用机载感知与计算，成功穿越箱子、跨栏、楼梯及混合地形。 • **Novel two-stage pipeline**: A diffusion model first generates terrain-aware reference motions in real time from human motion data, then a separate RL-based whole-body tracker executes them on the robot — cleanly separating generation from control. • **Closed-loop fine-tuning**: The motion tracker is further fine-tuned with a frozen motion generator in the loop, making the tracker robust to imperfect or out-of-distribution generated references. • **Full hardware deployment**: The complete system runs on a Unitree G1 humanoid with onboard perception (no offboard compute), successfully crossing boxes, hurdles, stairs, and mixed terrain.
🧪 实验
内容		 实验在仿真环境（Isaac Gym）和真实硬件（Unitree G1人形机器人）上均有进行。系统在四种地形上进行评估：箱子、跨栏、楼梯和混合地形组合。对比基线包括：(1) 纯奖励塑形RL；(2) 不带在线运动生成的模仿RL；(3) 不带闭环微调的本文方法。评估指标涵盖地形穿越成功率、参考动作跟踪精度以及全身协调性的定性评价。消融实验单独验证了在线运动生成和闭环微调各自的贡献。 Experiments were conducted in both simulation (Isaac Gym) and on real hardware (Unitree G1 humanoid robot). The system was evaluated on four terrain types: boxes, hurdles, stairs, and mixed terrain combinations. Baselines compared include: (1) pure RL with reward shaping, (2) imitation RL without online motion generation, and (3) the proposed system without closed-loop fine-tuning. Metrics include traversal success rate, tracking accuracy of reference motions, and qualitative whole-body coordination assessment. Ablation studies isolate the contribution of online motion generation and closed-loop fine-tuning.
📊 实验
结果		 完整系统在所有地形类型上的穿越成功率均高于所有基线。与未经闭环微调的版本相比，闭环微调显著提升了成功率（论文中各地形类别有具体数字）。在线运动生成相比静态参考动作回放在未见地形组合上泛化能力更强。真实机器人实验中，机器人成功穿越全部四类地形（约15cm高箱子、跨栏、多级楼梯及混合序列），且完全依靠机载计算完成仿真到现实的迁移。消融实验定量验证了扩散生成与闭环微调两个组件对最终鲁棒性的独立贡献。 The full proposed system achieves the highest traversal success rates across all terrain types compared to baselines. Closed-loop fine-tuning improves success rate by a notable margin over the non-fine-tuned variant (specific gains reported across terrain categories in the paper). The system with online motion generation outperforms static reference replay in generalization to unseen terrain combinations. On real hardware, the robot successfully traverses all four terrain types (boxes up to ~15 cm, hurdles, multi-step stairs, and mixed sequences), demonstrating sim-to-real transfer with onboard-only compute. Quantitative ablations confirm that each component — diffusion generation and closed-loop fine-tuning — contributes meaningfully to final robustness.
🏁 结论 本文证明，将实时扩散运动生成器与闭环微调RL跟踪器结合，能够实现鲁棒的、地形感知的全身人形机器人运动，并成功迁移到真实硬件。该框架弥合了协调模仿学习与感知驱动在线适应之间的鸿沟，推动了可部署人形机器人运动控制的前沿发展。 This paper demonstrates that combining a real-time diffusion-based motion generator with a closed-loop fine-tuned RL tracker enables robust, terrain-aware whole-body humanoid locomotion that successfully transfers to real hardware. The framework bridges the gap between coordinated imitation learning and perception-driven online adaptation, advancing the state of the art in deployable humanoid robot locomotion.

⚙️ Method · 方法详解

框架包含两个协同训练的核心模块。首先，用重定向后的人类动作捕捉数据训练一个扩散模型，该模型以局部地形高度图和方向目标为条件，实时生成地形感知的参考动作序列。其次，训练一个基于RL的全身运动跟踪器，使其尽可能精确地跟随生成的参考动作，同时保持平衡。最后，冻结扩散生成器，在闭环中对跟踪器进行端到端微调，让跟踪器学会处理真实生成（可能有瑕疵）的参考动作分布。部署时，扩散生成器与RL跟踪器共同在Unitree G1机器人上通过机载摄像头和IMU运行。 The framework has two main components trained together. First, a diffusion model is trained on retargeted human motion capture data to generate terrain-aware reference motion sequences in real time, conditioned on local terrain height maps and directional goals. Second, a whole-body RL-based motion tracker is trained to closely follow these generated references while maintaining balance and stability on the robot. Finally, the tracker is fine-tuned end-to-end with the frozen diffusion generator in a closed loop, so it learns to handle the realistic distribution of generated (possibly imperfect) references rather than only ground-truth human motions. At deployment, the diffusion generator and RL tracker run together onboard the Unitree G1 robot using its cameras and IMU.

⚙️ Method · 方法详解

Qwen3.5-Omni采用"Thinker-Talker"双模块架构：Thinker负责跨模态理解与推理，Talker负责流式语音生成，两者均基于混合注意力MoE框架。为解决流式语音合成的稳定性问题，提出ARIA机制，动态对齐文本token与语音token的生成节奏，避免因两者速率差异导致的语音断裂和韵律异常。模型支持超过10小时的音频和400秒720P视频输入，并通过脚本级结构化字幕生成和自动场景分割实现精细时序对齐的音视频锚定。此外，模型支持10种语言的多语言理解与语音生成，具备情感细腻的类人语音输出。 Qwen3.5-Omni uses a dual-module "Thinker-Talker" architecture: the Thinker handles cross-modal understanding and reasoning, while the Talker handles streaming speech generation — both built on a Hybrid Attention Mixture-of-Experts (MoE) backbone. The key innovation, ARIA, dynamically aligns text and speech token generation rates during streaming, preventing prosody breakdowns caused by tokenizer speed mismatches. The model ingests over 10 hours of audio and 400 seconds of 720P video (at 1 FPS) in a single context window of 256k tokens. Audio-visual grounding is achieved via script-level structured captions with precise temporal synchronization and automated scene segmentation.

📐 Key Formulas · 关键公式

\[\text{ARIA: } \hat{s}_t = f_{\text{align}}(x_{\text{text}}, x_{\text{speech}}, t)\]
💡 ARIA机制：在每个时间步 t，通过对齐函数动态匹配文本token序列与语音token序列的生成节奏，消除两者速率差异引起的不稳定性。 ARIA mechanism: at each timestep t, an alignment function dynamically synchronizes the generation pace of text tokens and speech tokens, eliminating instability from their encoding rate mismatch.
