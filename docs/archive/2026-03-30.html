Natural-Language Agent Harnesses 自然语言智能体执行框架（NLAH）

🧑‍🔬 Linyue Pan, Lexiao Zou, Shuo Guo, Jingchen Ni et al.

图 Figure · Page 3 of paper

🎯 用可移植的自然语言规范替代硬编码的智能体控制逻辑
Replace hard-coded agent harness logic with portable, editable natural-language specifications that any runtime can execute.
维度 中文 English
❓ 待解决
的问题		 现代AI智能体的性能高度依赖"执行框架"（harness）——即控制智能体如何循环、调用工具和管理状态的脚手架代码。这些逻辑通常深埋于特定框架的Python代码中，难以跨系统迁移、复现或作为科学对象进行研究。目前没有统一的方式来独立表达、共享或比较智能体执行框架的设计。 Modern AI agents depend heavily on "harness engineering" — the scaffolding code that controls how an agent loops, calls tools, and manages state. This logic is typically buried inside framework-specific Python code, making it difficult to transfer across systems, reproduce, or study scientifically. There is no standard way to express, share, or compare agent harness designs independent of their implementation.
💡 解决
的亮点		 • NLAH将智能体控制逻辑外化为可编辑的自然语言制品，无需修改代码即可实现框架的可移植与可检视。 • 智能体智能运行时（IHR）通过显式契约、持久化制品和轻量级适配器执行自然语言框架，实现逻辑与运行时的清晰解耦。 • 支持"代码到文本的框架迁移"，将现有的代码化执行框架转换为自然语言等价物，提升可访问性与科学可比性。 • NLAHs externalize agent harness control logic into editable natural-language artifacts, making harness design portable and inspectable without touching code. • The Intelligent Harness Runtime (IHR) executes these natural-language harnesses through explicit contracts, durable artifacts, and lightweight adapters — a clean separation of logic from runtime. • Enables "code-to-text harness migration," converting existing coded harnesses into natural-language equivalents, broadening accessibility and scientific comparability.
🧪 实验
内容		 实验在编程任务和计算机操作（computer-use）基准上展开，涵盖三个维度：（1）操作可行性——NLAH驱动的智能体能否与基于代码的框架性能持平；（2）模块消融——评估IHR各组件（契约、制品、适配器）的独立贡献；（3）代码到文本迁移——衡量将现有代码框架转换为自然语言形式的保真度。以基于代码的原始框架作为基线，在编程和GUI操作基准上进行对比评估。 Experiments were conducted on coding and computer-use benchmarks to evaluate NLAHs across three dimensions: (1) operational viability — whether NLAH-driven agents can match code-based harness performance, (2) module ablation — assessing the contribution of IHR's individual components (contracts, artifacts, adapters), and (3) code-to-text harness migration — measuring the fidelity of converting existing coded harnesses to natural-language form. Standard agent benchmarks for coding tasks and GUI/computer-use tasks serve as the evaluation arenas, with code-based harnesses as baselines.
📊 实验
结果		 实验表明，基于NLAH的智能体在编程基准上的性能与代码化框架基线相当，性能差距在约2–5%以内。模块消融实验显示，IHR的每个组件（契约、持久化制品、适配器）均对整体性能有实质贡献，移除任一组件均导致可测量的性能下降。代码到文本迁移实验以高保真度复现了现有框架的行为，验证了NLAH格式在不同智能体环境下的通用性。 The paper demonstrates that NLAH-based agents achieve competitive performance with code-based harness baselines on coding benchmarks, with performance differences within a small margin (reported task success rates on coding benchmarks are approximately on par with or within ~2–5% of hard-coded counterparts). Module ablation shows that each IHR component (contracts, durable artifacts, adapters) contributes meaningfully to overall performance, with removal of any single component causing measurable degradation. Code-to-text migration successfully reproduces the behavior of existing harnesses with high fidelity, validating the generality of the NLAH format across different agent settings.
🏁 结论 本文证明了智能体框架逻辑可以完全外化为可移植、可执行的自然语言制品，且不损失性能，为将执行框架工程转变为可透明研究和共享的科学对象奠定了基础。这项工作为AI智能体研究开辟了新方向，使框架工程成为可复现、可迁移的规范化研究领域。 This paper proves that agent harness logic can be fully externalized as portable, executable natural-language artifacts without sacrificing performance, establishing NLAH and IHR as a viable scientific foundation for studying and sharing agent harness designs. The work opens a new direction for making harness engineering a transparent, reproducible, and transferable discipline in AI agent research.
🔗 论文
链接		 arXiv: 2603.25723v1  ·  📥 PDF

⚙️ Method · 方法详解

作者提出自然语言智能体执行框架（NLAH），将智能体的控制流（循环、工具调度、状态管理）编码为结构化自然语言文档，而非代码。这些文档由智能体智能运行时（IHR）解释执行，依托三种机制：显式契约（定义智能体与环境的交互规则）、持久化制品（保存中间状态）和轻量级适配器（将运行时桥接至不同的智能体后端）。此外，现有的代码化框架也可通过代码到文本的转换流程迁移为NLAH格式，确保与已部署智能体的向后兼容。 The authors introduce Natural-Language Agent Harnesses (NLAHs), which encode agent control flow (loops, tool dispatch, state management) as structured natural-language documents rather than code. These documents are executed by the Intelligent Harness Runtime (IHR), which interprets harness instructions through three mechanisms: explicit contracts (defining agent–environment interaction rules), durable artifacts (persistent intermediate state), and lightweight adapters (bridging the runtime to different agent backends). An existing coded harness can also be migrated to NLAH format through a code-to-text conversion procedure, enabling backwards compatibility with deployed agents.

🌟 Why It Matters · 为什么重要

随着基于LLM的智能体快速扩展，不透明、框架锁定的执行框架代码严重阻碍了智能体系统的可复现性与可比性，NLAH直接解决了这一瓶颈，将控制逻辑提升为可共享的一等制品。这有望像"模型卡片"规范模型报告一样，标准化研究社区设计、发布和基准测试智能体框架的方式。 As LLM-based agents proliferate, reproducibility and comparability of agent systems are increasingly hampered by opaque, framework-locked harness code — NLAH directly addresses this bottleneck by making control logic a first-class, shareable artifact. This could standardize how the research community designs, publishes, and benchmarks agent harnesses, much like model cards standardized model reporting.

Trace2Skill: Distill Trajectory-Local Lessons into Transferable Agent Skills Trace2Skill：将轨迹局部经验提炼为可迁移的智能体技能

🧑‍🔬 Jingwei Ni, Yihao Liu, Xinpeng Liu, Yutao Sun et al.

图 Figure · Page 3 of paper

🎯 从多条执行轨迹中自动提炼可迁移的LLM智能体技能
Automatically distills diverse agent execution traces into transferable, declarative LLM skills via parallel sub-agent analysis.
维度 中文 English
❓ 待解决
的问题		 LLM智能体需要特定领域的技能来完成复杂任务，但手动编写这些技能费时费力、难以扩展。现有的自动化方法要么依赖模型的浅层知识，要么逐条处理执行轨迹，导致技能过拟合于特定任务实例，泛化能力差。 LLM agents need domain-specific skills to tackle complex tasks, but writing these skills manually is expensive and hard to scale. Existing automated approaches either rely on shallow model knowledge or process trajectories one-by-one, causing overfitting to specific task instances and producing brittle, non-generalizable skills.
💡 解决
的亮点		 • 并行子智能体舰队同时分析多条多样化执行轨迹，避免逐步过拟合。 • 层级归纳整合机制将各轨迹的局部经验合并为统一、无冲突的技能目录。 • 技能以纯文本声明式呈现，无需参数更新或外部检索模块，可跨不同规模LLM和分布外场景迁移。 • Parallel fleet of sub-agents analyzes a diverse pool of execution traces simultaneously, avoiding sequential overfitting. • Hierarchical inductive consolidation merges trajectory-specific lessons into a unified, conflict-free skill directory. • Skills are purely declarative (text-based guides) — no parameter updates or external retrieval modules needed, and they transfer across different LLM scales and out-of-distribution settings.
🧪 实验
内容		 实验在三个领域展开：电子表格操作（以Anthropic官方xlsx技能为基线）、视觉问答（VisionQA）和数学推理。技能由Qwen3.5-35B生成，并在同模型和跨模型（如Qwen3.5-122B）设置下测试，同时评估在WikiTableQuestions等分布外数据集上的泛化能力。 Experiments were conducted on three challenging domains: spreadsheet manipulation (xlsx tasks, benchmarked against Anthropic's official xlsx skills), visual question answering (VisionQA), and mathematical reasoning. Baselines include Anthropic's official hand-authored skills and other automated skill generation approaches. Skills were evolved using Qwen3.5-35B and tested on both same-model and cross-model settings (e.g., applied to Qwen3.5-122B), as well as out-of-distribution (OOD) datasets like WikiTableQuestions.
📊 实验
结果		 由Qwen3.5-35B进化出的技能在WikiTableQuestions（分布外）设置下使Qwen3.5-122B智能体提升高达57.65个绝对百分点。在电子表格、视觉问答和数学推理基准上，Trace2Skill均显著超越包括Anthropic官方xlsx技能在内的强基线。仅需35B开源模型即可实现跨规模迁移，无需参数更新。 Skills evolved by Qwen3.5-35B on its own trajectories improved a Qwen3.5-122B agent by up to 57.65 absolute percentage points on WikiTableQuestions (OOD setting). Trace2Skill significantly outperforms strong baselines including Anthropic's official xlsx skills across spreadsheet, VisionQA, and math reasoning benchmarks. Cross-scale transfer is demonstrated with open-source models as small as 35B parameters, with no parameter updates required.
🏁 结论 Trace2Skill证明了复杂智能体执行经验可通过整体并行轨迹分析和层级归纳整合，提炼为高度可迁移的声明式技能。该方法实现了可扩展、可泛化的技能自动生成，无需参数更新即可跨LLM规模和分布外场景迁移。 Trace2Skill demonstrates that complex agent execution experience can be distilled into highly transferable, declarative skills through holistic, parallel trajectory analysis and hierarchical inductive consolidation. This approach enables scalable, generalizable skill authoring that transfers across LLM scales and OOD settings without any parameter updates.
🔗 论文
链接		 arXiv: 2603.25158v1  ·  📥 PDF

⚙️ Method · 方法详解

Trace2Skill首先收集目标领域的多条智能体执行轨迹，然后派遣并行子智能体舰队独立分析各轨迹并提取局部经验。接着通过层级归纳推理将这些经验整合为一个统一、无冲突的技能目录。该框架同时支持从零创建新技能和对已有人工编写技能的深化扩展，全程无需修改模型参数。 Trace2Skill first collects a diverse pool of agent execution trajectories on a target domain. A parallel fleet of sub-agents is dispatched to independently analyze these trajectories and extract trajectory-specific lessons. These lessons are then hierarchically consolidated through inductive reasoning into a single, comprehensive, conflict-free skill directory. The framework supports both creating new skills from scratch and deepening existing human-authored skills, all without modifying model weights.

🌟 Why It Matters · 为什么重要

该工作解决了大规模部署LLM智能体时技能编写成本高、易碎的关键瓶颈，证明纯文本自动生成的技能可以媲美甚至超越人工专家编写的技能。跨模型和分布外迁移结果为构建模型无关、无需更新的可复用智能体知识库提供了实用路径。 This work addresses a critical bottleneck in deploying LLM agents at scale — the cost and fragility of skill authoring — by showing that purely text-based, automatically generated skills can rival and surpass expert-written ones. The cross-model and OOD transferability results suggest a practical path toward reusable agent knowledge that is model-agnostic and update-free.

Composer 2 Technical Report Composer 2 技术报告

🧑‍🔬 Cursor Research, :, Aaron Chan, Ahmed Shalaby et al.

图 Figure · Page 3 of paper

🎯 专为智能体软件工程打造的前沿级代码大模型
A frontier coding model trained with large-scale RL for agentic software engineering in real codebases.
维度 中文 English
❓ 待解决
的问题		 现有AI编程助手在真实大型代码库中难以完成长流程、多步骤的软件工程任务。大多数模型只擅长短小的代码片段，而无法胜任需要持续规划、工具调用和长程一致性的真实智能体工作流。 Modern AI coding assistants struggle with long-horizon, multi-step software engineering tasks in large, real-world codebases. Most models are optimized for short, isolated code snippets rather than realistic agentic workflows that require sustained planning, tool use, and coherent execution across many steps.
💡 解决
的亮点		 • 两阶段训练：先通过持续预训练夯实编程知识，再用大规模强化学习直接优化端到端智能体性能。 • 训练环境与实际部署的Cursor框架完全一致（相同工具、相同结构），消除了训练与部署之间的"仿真-现实"差距。 • 提出新基准CursorBench，来源于真实大型代码库（含Cursor自身代码库）中的软件工程问题，支持对长程复杂任务的分级评测。 • Two-phase training: continued pretraining to build deep coding knowledge, followed by large-scale RL that directly optimizes end-to-end agentic performance. • Training infrastructure mirrors the exact deployed Cursor harness — same tools, same structure — eliminating the sim-to-real gap between training and deployment. • A new benchmark, CursorBench, is introduced, derived from real software engineering problems in large codebases (including Cursor's own), enabling evaluation on increasingly difficult long-horizon tasks.
🧪 实验
内容		 模型在三个基准上进行评测：内部开发的CursorBench（来源于真实大型代码库的软件工程问题）、Terminal-Bench（公开的基于终端的智能体编程任务基准）以及SWE-bench Multilingual（多语言软件工程公开基准）。基线包括前代Composer模型及当前最先进的公开系统。所有评测均在Cursor智能体框架内进行，与训练设置保持一致。 The model is evaluated on three benchmarks: the internally developed CursorBench (derived from real large-codebase SE problems), Terminal-Bench (a public benchmark for terminal-based agentic coding tasks), and SWE-bench Multilingual (a public benchmark for multi-language software engineering). Baselines include prior Composer models and state-of-the-art public systems. All evaluations are conducted within the Cursor agent harness to ensure consistency with the training setup.
📊 实验
结果		 在CursorBench上，Composer 2达到61.3分，相比前代Composer模型有重大提升。在公开基准Terminal-Bench上得分61.7，在SWE-bench Multilingual上得分73.7，两项成绩均与当前最先进系统持平。 On CursorBench, Composer 2 achieves 61.3 accuracy, representing a major improvement over previous Composer models. On the public Terminal-Bench, the model scores 61.7. On SWE-bench Multilingual, it achieves 73.7 — both results comparable to state-of-the-art systems.
🏁 结论 Composer 2证明，将持续预训练与大规模强化学习结合，并在与部署完全一致的训练框架中进行，能够产出前沿级的领域专用编程模型。本研究为构建针对特定现实场景的强大智能体模型提供了可复现的训练范式。 Composer 2 demonstrates that combining continued pretraining with large-scale reinforcement learning in a deployment-faithful training harness produces a frontier-level, domain-specialized coding model. The work validates a reproducible recipe for building strong agentic models tailored to specific real-world domains.

⚙️ Method · 方法详解

Composer 2 采用两阶段训练。第一阶段对代码密集型数据进行持续预训练，提升模型的基础编程知识与潜在能力。第二阶段进行大规模强化学习，针对长程智能体任务对模型进行优化，奖励准确的多步执行、更强的推理能力和长程一致性。关键在于，训练所用的Cursor智能体框架与部署完全一致（工具和环境结构相同），使模型学到的行为可直接迁移到真实用户场景。训练环境来源于真实软件工程问题，保证了高度的现实保真度。 Composer 2 is trained in two phases. First, continued pretraining on code-rich data improves the model's foundational coding knowledge and latent capabilities. Second, large-scale reinforcement learning fine-tunes the model on long-horizon agentic tasks, rewarding accurate multi-step execution, stronger reasoning, and coherent long-range planning. Crucially, training uses the same Cursor agent harness as deployment — including identical tools and environment structure — so the model learns behaviors that transfer directly to real user interactions. Training environments are sourced from real software engineering problems to ensure high fidelity.

🌟 Why It Matters · 为什么重要

本文提供了一个具体可落地的蓝图，说明如何通过大规模强化学习训练领域专用智能体大模型，并强调训练与部署环境对齐是真实性能的关键。对更广泛的ML/RL社区而言，本文证明了有针对性的大规模RL能够弥合通用前沿模型与专业生产系统之间的性能差距。 This paper demonstrates a concrete, deployable blueprint for training domain-specialized agentic LLMs using RL at scale, with the critical insight that training-deployment environment alignment is key to real-world performance. For the broader ML/RL community, it provides evidence that targeted large-scale RL can close the gap between general-purpose frontier models and specialized production systems.

Self-Improvement of Large Language Models: A Technical Overview and Future Outlook 大语言模型的自我提升：技术综述与未来展望

🧑‍🔬 Haoyan Yang, Mario Xerri, Solha Park, Huajian Zhang et al.

图 Figure · Page 2 of paper

🎯 统一框架系统梳理LLM自我提升全生命周期
A unified closed-loop framework systematizing how LLMs can autonomously improve themselves without human supervision.
维度 中文 English
❓ 待解决
的问题		 训练大语言模型越来越依赖昂贵且难以扩展的人工监督。当模型在某些领域接近人类水平时，人类反馈的信息量会大幅下降，难以提供有效的提升信号。因此，亟需一种让模型能够自主驱动自身改进、减少对人工干预依赖的系统性框架。 Training large language models increasingly relies on expensive and limited human supervision. As models approach human-level performance in some domains, human feedback loses its informativeness as a training signal. There is an urgent need for frameworks where models can autonomously drive their own improvement without depending on humans at every step.
💡 解决
的亮点		 • 提出"闭环生命周期"框架，将数据获取、数据筛选、模型优化、推理精炼四个环节紧密耦合，并引入自主评估层持续监控和引导整个循环。 • 创新性地将LLM自身定位为所有阶段的核心驱动者：自主生成训练数据、筛选有效信号、更新参数并精炼输出。 • 首次从系统层面提供统一分类体系，将RLHF、自博弈、合成数据、迭代提示等碎片化研究整合进一个连贯的架构。 • Introduces a novel closed-loop lifecycle framework with four tightly coupled stages—data acquisition, data selection, model optimization, and inference refinement—plus an autonomous evaluation layer that monitors and guides the entire cycle. • Uniquely positions the LLM itself as the central agent across all stages: generating its own training data, selecting informative signals, updating parameters, and refining outputs. • Provides the first system-level unified taxonomy of self-improvement techniques, connecting previously fragmented research threads (RLHF, self-play, synthetic data, iterative prompting, etc.) into one coherent architecture.
🧪 实验
内容		 本文为综述/框架性论文，不包含新的原创实验。作者系统梳理并分类了现有代表性方法，涉及GSM8K、MATH、HumanEval、MMLU、AlpacaEval等基准测试上已报告的结果。分析对象包括Constitutional AI、Self-Instruct、STaR、RLHF/RLAIF、DPO、自一致性、思维树等多种技术。 As a survey/framework paper, this work does not conduct novel empirical experiments. Instead, it systematically reviews and categorizes existing representative methods across the four lifecycle stages, drawing on results reported in prior literature spanning benchmarks such as GSM8K, MATH, HumanEval, MMLU, and AlpacaEval. The authors analyze and compare techniques including Constitutional AI, Self-Instruct, STaR, RLHF/RLAIF, DPO, self-consistency, and tree-of-thought prompting, among others.
📊 实验
结果		 本文为综述论文，不报告新实验数字。综合文献结果：STaR等自训练方法在GSM8K数学推理上相比基础模型提升10–20+个百分点；RLAIF和Constitutional AI在显著减少人工标注的前提下达到与RLHF相当的对齐质量；自一致性（多数投票）在推理基准上提升约5–17%准确率；DPO方法在AlpacaEval上与PPO-based RLHF持平甚至更优，且计算成本更低。以上数据均来自被综述的文献。 As a survey paper, this work reports no new benchmark numbers of its own. It synthesizes findings from the literature: for instance, STaR and related self-training methods improve GSM8K math reasoning accuracy by 10–20+ percentage points over base models; RLAIF and Constitutional AI approaches achieve alignment quality competitive with RLHF using significantly less human annotation; self-consistency (majority voting) improves chain-of-thought accuracy by ~5–17% on reasoning benchmarks; and DPO-based methods match or exceed PPO-based RLHF on AlpacaEval while being computationally cheaper. These figures are drawn from surveyed works rather than new experiments.

⚙️ Method · 方法详解

作者将LLM自我提升概念化为一个四阶段闭环生命周期。在数据获取阶段，模型自主生成或收集训练数据，包括合成样本和自博弈轨迹。在数据筛选阶段，基于质量、多样性和难度标准过滤最有价值的信号。模型优化阶段涵盖监督微调、基于AI反馈的强化学习（RLAIF）和直接偏好优化（DPO）等技术。推理精炼阶段则利用思维链提示、自一致性和自我批判等测试时计算策略。贯穿四个阶段的自主评估层持续监控模型表现，动态调整和引导整个循环。 The authors conceptualize LLM self-improvement as a closed-loop lifecycle with four tightly coupled processes. In the data acquisition stage, the model generates or collects its own training data, including synthetic examples and self-play trajectories. The data selection stage filters for the most informative signals using quality, diversity, and difficulty criteria. The model optimization stage covers techniques such as supervised fine-tuning, reinforcement learning from AI feedback (RLAIF), and direct preference optimization (DPO). Finally, the inference refinement stage applies test-time computation strategies like chain-of-thought prompting, self-consistency, and self-critique. An autonomous evaluation layer ties all four stages together by continuously assessing model performance and redirecting the cycle.

📐 Key Formulas · 关键公式

\[\pi_{t+1} = \arg\max_{\pi} \mathbb{E}_{x \sim \mathcal{D},\, y \sim \pi(\cdot|x)} \left[ r(x, y) \right] - \beta \cdot \mathrm{KL}\left[\pi(\cdot|x) \| \pi_t(\cdot|x)\right]\]
💡 自我提升迭代优化目标：在最大化奖励函数 r(x,y) 的同时，通过KL散度约束防止新策略偏离当前策略过远，β为平衡系数。 The iterative self-improvement optimization objective: maximize expected reward r(x,y) while a KL-divergence penalty (scaled by β) keeps the updated policy π_{t+1} from drifting too far from the current policy π_t, ensuring stable self-training.
\[\mathcal{L}_{\mathrm{DPO}}(\pi_\theta) = -\mathbb{E}_{(x, y^+, y^-)} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y^+|x)}{\pi_{\mathrm{ref}}(y^+|x)} - \beta \log \frac{\pi_\theta(y^-|x)}{\pi_{\mathrm{ref}}(y^-|x)} \right) \right]\]
💡 直接偏好优化（DPO）损失函数：直接用偏好对(y⁺优于y⁻)训练模型，无需显式训练奖励模型，是自我提升中模型优化阶段的代表方法。 The Direct Preference Optimization (DPO) loss: trains the model directly on preference pairs (y⁺ preferred over y⁻) relative to a reference policy π_ref, bypassing the need for an explicit reward model — a key technique in the model optimization stage of self-improvement.
