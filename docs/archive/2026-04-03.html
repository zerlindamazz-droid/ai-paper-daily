⚙️ Method · 方法详解

本文采用教程式写作方式，从第一性原理逐步构建Transformer。从文本分词和向量空间嵌入出发，引入缩放点积注意力公式，解释其捕捉语义关系的机制；进而扩展到多头注意力，说明Transformer编码器/解码器模块的组装方式，并调研多种架构变体。最后介绍KV缓存（避免冗余计算）、分组查询注意力（降低内存开销）和潜在注意力（键值对的低秩压缩策略）。 The paper takes a tutorial approach, building up the Transformer from first principles. It starts with text tokenization and vector space embeddings, then introduces the scaled dot-product attention formula and its role in capturing semantic relationships. It extends this to Multi-Head Attention, explains how Transformer encoder/decoder blocks are assembled, and surveys architectural variants. Finally, it explains KV caching to avoid redundant computation, Grouped Query Attention to reduce memory overhead, and Latent Attention as a low-rank compression strategy for key-value pairs.

📐 Key Formulas · 关键公式

\[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V\]
💡 缩放点积注意力公式：查询矩阵Q与键矩阵K的点积除以维度平方根进行缩放，经softmax归一化后对值矩阵V加权求和，输出上下文感知的表示。 Scaled dot-product attention: the dot product of queries Q and keys K is scaled by the square root of key dimension d_k, normalized via softmax, then used to weight-sum the values V, producing context-aware representations.
\[\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_H)W^O, \quad \text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)\]
💡 多头注意力：将输入分别用H组不同的线性投影矩阵映射为查询、键、值，各头独立计算注意力后拼接，再通过输出投影矩阵W^O融合，允许模型同时关注多种语义关系。 Multi-Head Attention: input is projected H times with different learned matrices to form queries, keys, and values; each head independently computes attention, outputs are concatenated and linearly projected, allowing the model to jointly attend to information from multiple representation subspaces.

🌟 Why It Matters · 为什么重要

随着Transformer在各类AI任务中的核心地位日益突出，帮助数学家和数值分析学者深入理解其机制，有助于加速跨学科创新，例如基于随机低秩近似的注意力优化。本教程降低了RNLA及更广泛应用数学社区为大语言模型效率与可扩展性做出理论贡献的门槛。 As Transformers become central to AI across modalities, enabling mathematicians and numerical analysts to engage deeply with their mechanics accelerates interdisciplinary innovations—such as randomized low-rank approximations of attention. This tutorial lowers the barrier for the RNLA and broader applied math community to contribute theoretically grounded improvements to LLM efficiency and scalability.

🧑‍🔬 Zefeng He, Siyuan Huang, Xiaoye Qu, Yafu Li et al.

🎯 赋予多模态生成模型记忆与技能的智能体框架
An agent-native framework with memory and skills that supercharges multimodal generation models on complex tasks.
维度 中文 English
❓ 待解决
的问题		 当前最先进的多模态生成模型在面对复杂指令和专业下游任务时仍表现不佳，其能力上限受限于模型本身的参数。这些模型缺乏迭代优化输出、积累经验或按需调用领域专业知识的能力。 State-of-the-art multimodal generation models still struggle with complex instructions and specialized downstream tasks — they hit a performance ceiling baked into their weights. There is no principled way for these models to iteratively refine their outputs, accumulate experience, or leverage domain-specific expertise on demand.
💡 解决
的亮点		 • **智能体循环（Agent Loop）**：多智能体闭环结构，通过迭代批评与优化持续提升生成质量，突破单次推理的上限。 • **智能体记忆（Agent Memory）**：持久化的轨迹级分层记忆，同时存储事实状态和压缩经验摘要，在降低冗余的同时保持对优化过程的全局视野。 • **智能体技能（Agent Skill）**：可扩展的领域专属技能库，支持按需动态加载，使轻量级6B模型也能胜任多样化专业任务。 • **Agent Loop**: A structured multi-agent closed-loop that iteratively critiques and refines generated outputs, enabling self-improvement beyond a single forward pass. • **Agent Memory**: A persistent, trajectory-level hierarchical memory that stores factual states and compressed experiential summaries, giving the system a global view of optimization while cutting redundancy. • **Agent Skill**: An extensible, on-demand library of domain-specific skills that can be loaded dynamically, letting even a lightweight 6B model tackle diverse specialized tasks.
🧪 实验
内容		 GEMS在五个主流多模态生成任务和四个专业下游任务上进行了评估，使用了多个不同规模的生成后端。主要评测基准为GenEval2（综合组合图像生成基准）。基线模型包括当时最强的Nano Banana 2以及骨干模型Z-Image-Turbo（6B参数）。同时进行了消融实验，验证每个组件（智能体循环、记忆、技能）的独立贡献。 GEMS was evaluated across five mainstream multimodal generation tasks and four specialized downstream tasks, using multiple generative backends of varying scales. The primary benchmark is GenEval2, a comprehensive compositional image generation benchmark. Baselines include strong state-of-the-art models such as Nano Banana 2 and the underlying backbone Z-Image-Turbo (a 6B-parameter model). Ablation studies were conducted to validate the contribution of each component (Agent Loop, Memory, Skill).
📊 实验
结果		 在GenEval2基准上，GEMS使轻量级6B模型Z-Image-Turbo的性能超越了当时最先进的Nano Banana 2，实现了对基础模型能力上限的突破。在所有五个主流任务和四个下游任务上，GEMS在各类生成后端下均取得了一致且显著的性能提升。仅靠智能体框架（无需重新训练模型），即可使小模型超越更大的专门构建的SOTA竞品。 On GenEval2, GEMS enables the lightweight 6B model Z-Image-Turbo to surpass the state-of-the-art Nano Banana 2, demonstrating a leap beyond what the base model could achieve alone. GEMS achieves consistent and significant performance gains across all five mainstream tasks and all four downstream tasks tested, regardless of the generative backend used. The agent harness alone — without any model retraining — is sufficient to push a smaller model past a much larger, purpose-built SOTA competitor.
🏁 结论 GEMS证明了：通过为生成模型配备分层记忆和按需技能的智能体框架，即使是轻量级6B模型也能超越最先进的竞品，确立了原生智能体设计作为提升多模态生成能力的强大且模型无关的新范式。 GEMS proves that wrapping a generative model in an agent framework with hierarchical memory and on-demand skills can push even a lightweight 6B model past state-of-the-art competitors, establishing agent-native design as a powerful and model-agnostic path to advancing multimodal generation.

⚙️ Method · 方法详解

GEMS将任意生成模型包装在三个智能体组件中，无需重新训练底层模型。智能体循环协调多个专业子智能体，对当前生成结果进行批评并提出改进意见，形成闭环反馈。智能体记忆在整个优化轨迹中维护一个分层存储结构，细粒度保留事实状态日志，同时对经验摘要进行压缩，从而在不爆炸上下文窗口的情况下保持长程上下文感知。智能体技能提供即插即用的领域专业知识库，仅在需要时按需加载，保持推理效率。三者协同工作，覆盖通用与专业下游任务。 GEMS wraps any generative backend inside a three-component agent harness. First, an Agent Loop orchestrates multiple specialized sub-agents that critique the current generation and propose targeted edits in a closed feedback cycle. Second, Agent Memory maintains a structured, hierarchical store across the entire optimization trajectory — factual state logs are kept at fine granularity while experiential summaries are compressed, so the system retains long-horizon context without blowing up the context window. Third, Agent Skill provides a plug-and-play repository of domain expertise (e.g., style, composition, domain knowledge) that is loaded only when needed, keeping inference efficient. Together, these three components operate on top of an unmodified base model, requiring no retraining.

📐 Key Formulas · 关键公式

\[\text{GEMS} = \text{Agent Loop} \oplus \text{Agent Memory} \oplus \text{Agent Skill}\]
💡 GEMS框架由三个核心组件的协同组合构成：闭环优化的智能体循环、持久化分层记忆、以及按需加载的领域技能库。 GEMS is the combination of three synergistic components: a closed-loop Agent Loop for iterative refinement, Agent Memory for persistent hierarchical experience storage, and Agent Skill for on-demand domain expertise.

🌟 Why It Matters · 为什么重要

本文表明生成模型的能力上限并非固定不变——智能体框架可以在不修改模型权重的情况下大幅解锁额外能力。这为社区提供了一个实用且可扩展的新思路：无需一味堆叠模型规模，而是用更智能的智能体循环、记忆与技能来赋能小模型。 This work demonstrates that the performance ceiling of generation models is not fixed — an agent harness can unlock capabilities far beyond what training alone delivers, without touching model weights. It opens a practical, scalable recipe for the community: instead of always scaling up models, wrap smaller ones in smarter agent loops with memory and skills.

SKILL0: In-Context Agentic Reinforcement Learning for Skill Internalization SKILL0：用于技能内化的上下文智能体强化学习

🧑‍🔬 Zhengxi Lu, Zhiyuan Yao, Jinyang Wu, Chengcheng Han et al.

图 Figure · Page 3 of paper

🎯 课程强化学习将智能体技能内化进模型参数，实现零样本自主行为。
A curriculum RL framework that internalizes agent skills into model weights, enabling zero-shot task completion.
维度 中文 English
❓ 待解决
的问题		 大语言模型智能体通常依赖推理时动态检索"技能包"来增强能力。这种方式有三大缺陷：检索噪声引入无关信息、大量注入内容带来严重的token开销、模型本身从未真正"学会"技能只是在执行指令。能否把技能直接烧录进模型参数，彻底摆脱运行时检索？ LLM agents are typically augmented with external "skills" — structured knowledge packages retrieved at inference time. This approach suffers from retrieval noise, heavy token overhead, and a fundamental limitation: the model never truly learns the skill, it only follows it. Can skills be baked directly into model weights instead?
💡 解决
的亮点		 • **渐进式技能撤除课程**：训练从完整技能上下文开始，逐步减少直至完全移除，迫使模型将知识内化进参数而非依赖外部检索。 • **动态课程 + 在线有效性过滤**：实时评估每个技能文件对当前策略的帮助程度，在线性衰减的token预算下淘汰无益技能，防止训练停滞。 • **可视化技能渲染**：将技能与交互历史渲染为紧凑视觉上下文（每步不超过0.5k tokens），高效教导多轮工具调用，不造成上下文窗口压力。 • **Progressive Skill Withdrawal Curriculum**: Training starts with full skill context visible, then gradually removes it, forcing the model to internalize knowledge into its weights rather than rely on external lookup. • **Dynamic Curriculum with On-Policy Helpfulness Filtering**: Each skill file is evaluated for how much it still benefits the current policy; unhelpful skills are dropped under a linearly decaying token budget, preventing stagnation. • **Visual Skill Rendering**: Skills and interaction history are rendered into a compact visual context (under 0.5k tokens/step), teaching multi-turn tool use efficiently without overwhelming the context window.
🧪 实验
内容		 实验在两个智能体基准上进行：ALFWorld（具身家庭任务完成）和Search-QA（多步网页搜索问答）。SKILL0与标准RL基线（无技能增强）和推理时技能检索基线进行对比。评估指标为任务成功率，同时衡量上下文效率（每步token数）。消融实验验证了各课程组件的独立贡献。 Experiments were conducted on two agentic benchmarks: ALFWorld (embodied household task completion) and Search-QA (multi-step web search question answering). SKILL0 was compared against standard RL baselines (without skill augmentation) and inference-time skill retrieval baselines. The evaluation metric is task success rate, and experiments measured both performance and context efficiency (tokens per step). Ablations tested the contribution of each curriculum component.
📊 实验
结果		 SKILL0在ALFWorld上相比标准RL基线提升+9.7%，在Search-QA上提升+6.6%。推理阶段完全零样本运行，每步上下文不超过0.5k tokens，远低于推理时技能检索方法需要注入大量技能文档的开销。 SKILL0 achieves +9.7% improvement over the standard RL baseline on ALFWorld and +6.6% on Search-QA. At inference time, the model operates in a fully zero-shot setting with a highly efficient context of fewer than 0.5k tokens per step — far lower than inference-time skill retrieval methods that inject large skill documents into every prompt.
🏁 结论 SKILL0证明了智能体技能可以通过渐进式上下文强化学习课程完全内化进模型参数，在零样本设置下同时超越标准RL和推理时技能检索方法，并彻底消除运行时token开销。这为LLM智能体建立了技能内化作为推理时增强的原则性实用替代方案。 SKILL0 demonstrates that agent skills can be fully internalized into model parameters via a progressive in-context RL curriculum, achieving stronger zero-shot performance than both standard RL and inference-time skill retrieval while eliminating runtime token overhead. This establishes skill internalization as a principled and practical alternative to inference-time augmentation for LLM agents.

⚙️ Method · 方法详解

SKILL0是一个三阶段设计的上下文强化学习框架。首先，技能按类别离线分组，与交互历史一起渲染成紧凑的视觉上下文，为智能体提供工具调用和多轮推理的结构化示范。其次，渐进课程从完整技能上下文开始训练，随训练进程线性撤除技能信息，让模型逐步脱离对外部引导的依赖。最后，动态课程在训练时评估每个技能文件的"在线有效性"，在不断收缩的token预算内只保留当前策略仍能从中受益的技能，直到推理阶段完全零样本运行。 SKILL0 is an in-context reinforcement learning framework with a three-stage design. First, skills are grouped offline by category and rendered alongside interaction history into a compact visual context, giving the agent structured demonstrations of tool use and multi-turn reasoning. Second, a progressive curriculum trains the model by starting with full skill context and linearly withdrawing it over training, so the model is gradually weaned off external guidance. Third, a Dynamic Curriculum evaluates each skill file's "on-policy helpfulness" at training time and retains only the skills from which the current policy still learns, under a shrinking token budget — until the agent operates in a fully zero-shot regime at inference time.

📐 Key Formulas · 关键公式

\[\text{Budget}(t) = B_{\max} \cdot \left(1 - \frac{t}{T}\right)\]
💡 线性衰减的token预算：随训练步数t增加，允许注入的技能上下文token总量从最大值线性减少到零，驱动课程从有监督到零样本过渡。 Linearly decaying token budget: as training step t increases, the maximum allowed skill-context tokens shrink linearly to zero, driving the curriculum from fully guided to fully zero-shot.

🌟 Why It Matters · 为什么重要

随着LLM智能体被部署于日益复杂的真实任务，运行时技能检索的扩展性瓶颈（成本、噪声、延迟）愈发关键。SKILL0提供了一条构建紧凑、自给自足智能体的路径，让模型将专业知识携带在权重中，既降低推理成本又提升鲁棒性。 As LLM agents are deployed in increasingly complex real-world tasks, the scalability bottleneck of runtime skill retrieval — cost, noise, and latency — becomes critical. SKILL0 offers a path to compact, self-sufficient agents that carry their expertise in their weights, reducing inference cost while improving robustness.

🎯 扩散语言模型驱动的600+语言零样本语音合成系统
A diffusion language model TTS system scaling zero-shot voice cloning to 600+ languages.
维度 中文 English
❓ 待解决
的问题		 大多数语音合成系统只支持少数几种语言，且需要特定说话人的训练数据。零样本TTS（无需微调即可克隆任意声音）在数百种语言上的扩展尤为困难。现有离散非自回归方法依赖繁琐的两阶段流水线（文本→语义token→声学token），容易造成误差累积和性能瓶颈。 Most text-to-speech (TTS) systems only support a handful of languages and require speaker-specific training data. Zero-shot TTS — cloning any voice without fine-tuning — is especially hard to scale across hundreds of languages. Existing discrete non-autoregressive (NAR) approaches rely on cumbersome two-stage pipelines (text → semantic tokens → acoustic tokens) that introduce error accumulation and performance bottlenecks.
💡 解决
的亮点		 • 创新的扩散语言模型式离散NAR架构，直接将文本映射到多码本声学token，彻底绕过两阶段流水线。 • 全码本随机掩码训练策略，支持对所有码本层级的高效联合训练。 • 利用预训练LLM进行参数初始化，借助强语言先验大幅提升零样本场景下的语音可懂度。 • 基于58.1万小时完全开源的多语言数据，实现迄今最广泛的语言覆盖（600+语言）。 • Novel diffusion language model-style discrete NAR architecture that directly maps text to multi-codebook acoustic tokens, bypassing the two-stage pipeline entirely. • Full-codebook random masking training strategy that enables efficient joint learning across all codebook levels simultaneously. • LLM-based initialization that bootstraps the model with strong language priors, dramatically improving speech intelligibility in zero-shot settings. • Scales to 600+ languages using 581k hours of entirely open-source multilingual audio data — the broadest language coverage ever achieved in TTS.

⚙️ Method · 方法详解

OmniVoice用单阶段扩散语言模型替代了传统的两阶段TTS流水线，通过迭代精炼被掩码的声学token来生成语音。训练时，全码本随机掩码策略在所有码本层级上随机遮盖token，迫使模型一次性联合学习各级声学结构。模型由预训练大语言模型（LLM）初始化，获得强语言理解先验，从而显著提升可懂度。推理时，输入文本和短暂的参考音频，模型以类扩散过程逐步解掩码声学token，生成自然且与说话人匹配的语音，无需级联阶段或说话人特定微调。 OmniVoice replaces the conventional two-stage TTS pipeline with a single-stage diffusion language model that iteratively refines masked acoustic tokens. During training, a full-codebook random masking strategy randomly masks tokens across all codebook levels, forcing the model to jointly learn acoustic structure at every level in one shot. The model is initialized from a pre-trained large language model (LLM), giving it strong language understanding priors that boost intelligibility. At inference, it takes text and a short speaker reference audio, then progressively unmasks acoustic tokens in a diffusion-like process to produce natural, speaker-matched speech — with no cascaded stages and no speaker-specific fine-tuning required.

📐 Key Formulas · 关键公式

\[\hat{x}_0 = f_\theta(x_t, t, c)\]
💡 扩散语言模型的核心去噪函数：给定被掩码/加噪的声学token序列 x_t、时间步 t 和文本条件 c，模型参数 θ 预测原始干净的声学token x_0。 The core denoising function of the diffusion language model: given a masked/noisy acoustic token sequence x_t, timestep t, and text condition c, the model with parameters θ predicts the original clean acoustic tokens x_0.
\[\mathcal{L} = \mathbb{E}_{x_0, t, m}\left[\|x_0 - f_\theta(x_t, t, c)\|^2\right]\]
💡 训练目标：对原始声学token、时间步和随机掩码取期望，最小化模型预测值与真实声学token之间的均方误差，驱动全码本联合学习。 Training objective: minimize the expected mean squared error between the model's predictions and the true acoustic tokens over sampled original tokens, timesteps, and random masks — driving joint full-codebook learning.

🌟 Why It Matters · 为什么重要

全球绝大多数语言目前仍得不到现有TTS技术的有效支持，OmniVoice的600+语言零样本系统大幅降低了低资源语言语音无障碍访问的门槛。通过开源代码与模型，它为社区提供了构建真正全语言语音AI的可扩展、可复现的技术蓝图。 The vast majority of the world's languages remain underserved by current TTS technology; OmniVoice's 600+ language zero-shot system dramatically lowers the barrier for speech accessibility across low-resource languages. By open-sourcing both code and models, it provides the community with a scalable, reproducible blueprint for building truly omnilingual voice AI.
