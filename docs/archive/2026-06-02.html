❓ 待解决
的问题		 现有的AI图像模型虽然很擅长识别物体和回答问题，但在理解三维世界方面却很薄弱——比如判断物体有多远、摄像头移动了多少。目前解决这类3D任务通常需要专门设计复杂的模型。问题是：我们真的需要这么复杂吗？通用AI模型能不能只靠正确的训练方式就学会3D理解？ AI models that understand images have gotten very good at recognizing objects and answering questions, but they still struggle to understand the 3D world — like figuring out how far away something is, or how a camera has moved. Today, solving 3D tasks usually requires building specialized, complicated models from scratch. The question is: do we really need all that complexity, or can a general-purpose AI already do 3D if trained the right way?
💡 解决
的亮点		 • 打破了"3D视觉必须用专用模型"的固有认知，普通VLM就能胜任。 • 只需三个关键要素：统一焦距处理、用文字描述像素位置、以及合理的数据混合与扩展。 • 用一个统一模型同时达到专家级精度，覆盖深度估计、相机位姿估计、像素对应和物体级3D理解等多种任务。 • Challenges the assumption that 3D vision requires specialized architectures — a standard VLM can do it all. • Identifies just three ingredients for 3D success: unifying focal length (camera settings), using text to describe pixel locations, and mixing/scaling the right training data. • Achieves expert-level accuracy on depth estimation, camera pose estimation, pixel correspondence, and object-level 3D understanding — all with one unified model.

⚙️ Method · 方法详解

研究团队用三个关键技巧训练了一个标准的视觉语言模型。第一，统一所有训练图像的相机焦距，让模型始终以一致的视角感知距离，就像确保所有照片都用同一个镜头拍摄。第二，不用复杂的空间编码，直接用文字描述像素位置，比如"第120行、第45列的像素"。第三，精心混合并扩展多种3D数据集进行训练。令人惊讶的是，不需要修改模型结构、不需要复杂的损失函数、也不需要大量数据增强。 The team trained a standard vision-language model (the kind used for image Q&A) on 3D tasks using three key tricks. First, they normalized the camera's focal length across all training images so the model sees distances consistently, like making sure every photo is taken with the same lens. Second, instead of using special spatial encodings, they simply described pixel positions using plain text — like saying "the pixel at row 120, column 45." Third, they carefully mixed and scaled different 3D datasets together during training. Remarkably, no changes to the model architecture, no complex loss functions, and no heavy data augmentation were needed.

📐 Key Formulas · 关键公式

\[\delta_1 = \text{accuracy} \uparrow: 0.84 \rightarrow 0.90\]
💡 深度估计的核心精度指标delta-1，衡量预测深度与真实深度的误差在阈值内的比例，VLM3将其从0.84提升到0.90。 Delta-1 is the key depth accuracy metric measuring what fraction of predictions fall within a threshold of the true depth; VLM3 raises this from 0.84 to 0.90.

🌟 Why It Matters · 为什么重要

目前，用于机器人、自动驾驶或AR眼镜的3D AI通常需要昂贵且复杂的定制模型。VLM3证明用标准通用AI模型就能达到同样效果，大幅降低了在实际产品中部署3D AI的成本和门槛。这可能加速从智能手机到自动驾驶汽车等各类产品中3D AI能力的普及。 Right now, building AI that understands 3D — for robots, self-driving cars, or AR glasses — requires expensive, complex, custom-built models. VLM3 shows you can get the same results with a standard off-the-shelf AI model, dramatically lowering the cost and complexity of deploying 3D AI in real products. This could accelerate 3D AI capabilities in everything from smartphones to autonomous vehicles.

维度 中文 English
❓ 待解决
的问题		 现有的AI编程助手要么太大、跑起来很慢很贵，要么太小、能力不足。开发者希望有一个既快速省资源，又能真正帮助写代码、调试、多步骤推理的AI工具。现有开源模型在代码任务上常常顾此失彼，难以在性能和效率之间取得平衡。 Existing AI coding assistants are either too big and expensive to run, or too small and not capable enough. 的亮点		 • 混合专家架构（64个专家，每次激活8个）：总参数12B但每次运算只用2.5B，像一个大团队中只派少数专才上阵，效率极高。 • 多令牌预测头（MTP）身兼两职：既是训练时的辅助目标，又是推理时的"草稿模型"，让解码速度更快。 • 三阶段渐进训练课程：从广泛网页数据逐步转向精选代码和数学内容，共训练约10.6万亿个token。 • 同时发布Instruct和Thinking两个变体：一个直接回答，一个会先"展示思考过程"再给出答案。 • Mixture-of-Experts with 64 experts, only 8 active at once — the model has big capacity but low cost per step. • A Multi-Token Prediction head that doubles as a training helper AND a built-in speed booster for inference. • A three-phase training curriculum of ~10.6 trillion tokens, gradually shifting from general web text to code and math. • Two released variants: an Instruct model (direct answers) and a Thinking model (shows its reasoning chain first).

⚙️ Method · 方法详解

Mellum 2采用"混合专家"结构，模型内部有64个"专家"子网络，但每次处理一个词时只激活其中8个，这样总参数量虽大（12B），实际计算量却只相当于一个2.5B的小模型。训练分三个阶段，先用海量网页数据打基础，再逐渐加入更多代码和数学数据，最后通过监督微调和强化学习（RLVR）让模型更听话、更善于推理。此外，模型内置了一个"多令牌预测头"，可以一次预测多个输出token，加快生成速度。上下文窗口通过YaRN技术扩展到128K，支持处理超长代码文件。 Mellum 2 uses a "Mixture-of-Experts" design: imagine a company with 64 specialized departments, but for each task only 8 departments are called in. This means the model has a huge knowledge base (12B parameters) but only uses a small fraction of its compute for each word it generates — equivalent to a much smaller 2.5B model. Training happened in three phases, starting broad (web text) and zooming in on code and math. After pre-training, the model was fine-tuned with human-style instructions and then further refined using reinforcement learning so it reasons better. A built-in "draft head" lets the model generate multiple tokens at once, speeding up responses.

📐 Key Formulas · 关键公式

\[\text{Active params per token} = \frac{\text{Total MoE params}}{\text{Total experts}} \times \text{Active experts} \approx 2.5\text{B}\]
💡 每处理一个词（token），模型只激活64个专家中的8个，实际运算量约等于2.5B参数的小模型，而非全部12B。 For each token processed, only 8 out of 64 experts are activated, so the actual compute cost equals roughly a 2.5B dense model, not the full 12B.
\[\text{Context window} = 128\text{K tokens (via layer-selective YaRN)}\]
💡 通过YaRN位置编码扩展技术，模型可以处理长达128,000个token的超长输入，相当于约10万个汉字的代码文件。 Using a technique called YaRN, the model can handle inputs up to 128,000 tokens long — roughly equivalent to a very large codebase or document.

🌟 Why It Matters · 为什么重要

对于普通开发者和企业来说，能在消费级GPU上运行的强大编程AI意味着更低的成本和更快的响应速度。开放权重（Apache 2.0）让任何人都可以自由使用、修改和部署，推动AI编程工具的民主化。 This matters because powerful AI coding assistants no longer have to require expensive cloud servers — Mellum 2 can run on a regular GPU while still being highly capable. The fully open Apache 2.0 license means anyone, from solo developers to startups, can freely build on top of it, democratizing access to high-quality AI programming help.

⚙️ Method · 方法详解

系统接收来自某位专家或角色的各种原始材料（对话记录、文档、操作日志等），通过自动化流程将其蒸馏成一个结构化的"技能包"。技能包包含两条并行轨道：一条记录专家的做事方法、思维框架和决策规律；另一条记录他们的沟通方式、互动风格和历史纠错记录。这个包就像一个"随身携带的专家手册"，可以被任何兼容的AI智能体直接加载和调用。用户还可以用日常语言给出反馈来更新技能包，系统会自动记录版本变更，支持随时回滚。 The system takes raw materials from a target person or role — think chat logs, documents, decision records — and automatically processes them into a tidy "skill package." This package has two coordinated parts: one captures how the expert thinks and makes decisions, the other captures how they talk and interact. The finished package can be loaded into any compatible AI agent like installing an app. Users can update it anytime using plain English feedback, and every change is versioned so you can undo mistakes. ⚙️ Method · 方法详解

表示强制分两步走。第一步：生成图像时，模型不是直接画像素，而是先产生一批"视觉摘要令牌"——对图像高层结构的压缩描述，就像先打一个草稿或蓝图。第二步：这些摘要令牌留在模型的"记忆"中，指导后续的精细像素绘制，整个过程在同一个神经网络主干内完成。训练时，模型被强制让这些摘要令牌与感知编码器的输出对齐，从而自学如何在高层理解和低层像素之间搭桥。这样就彻底去掉了外部生成帮手的依赖。 Representation Forcing works in two clever steps. First, when generating an image, the model doesn't jump straight to drawing pixels. Instead, it first produces a set of "visual summary tokens" — compact descriptions of what the image should look like at a high level (think of them as a rough sketch or blueprint). Second, those summary tokens stay visible in the model's memory and guide the fine-grained pixel-level drawing process, all within the same neural network backbone. During training, the model is forced to predict these visual summaries that match what a perception encoder would output, effectively teaching itself to bridge high-level understanding and low-level pixel painting. This eliminates the need for any separately trained generative helper.

🌟 Why It Matters · 为什么重要

去掉冻结辅助模块的依赖，意味着未来的AI助手（比如既能看图又能画图的聊天机器人）可以作为一个统一系统训练和提升——更简单、更易扩展、潜力更大。这是迈向像人类一样无缝融合图像与语言思维的AI的重要一步。 Removing the dependency on frozen helper modules means future AI assistants (like chatbots that see and draw) can be trained and improved as a single unified system — simpler to build, easier to scale, and potentially much more powerful. This is a meaningful step toward AI that seamlessly thinks in images and words the same way humans do.
