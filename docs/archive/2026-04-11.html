Your Agent Is Mine: Measuring Malicious Intermediary Attacks on the LLM Supply Chain 你的智能体是我的：衡量LLM供应链中的恶意中间人攻击

🧑‍🔬 Hanzhi Liu, Chaofan Shou, Hongbo Wen, Yanju Chen et al.

图 Figure · Page 2 of paper

🎯 恶意LLM API路由器可悄无声息地劫持智能体工具调用，注入恶意代码并窃取密钥。
Malicious LLM API routers can silently hijack agent tool calls to inject code and steal secrets.
维度 中文 English
❓ 待解决
的问题		 LLM智能体越来越依赖第三方API路由器来转发请求，而这些路由器以明文方式完整访问所有流量。目前没有任何提供商在客户端与上游模型之间强制执行加密完整性验证，形成了盲目信任关系。这意味着恶意路由器运营者可以悄无声息地篡改智能体发送或接收的每一条消息。 LLM agents increasingly rely on third-party API routers to forward requests to model providers, but these routers sit in the middle with full plaintext access to all traffic. No current provider enforces cryptographic integrity between the client and the upstream model, creating a blind trust relationship. This means a malicious router operator can silently tamper with every message an agent sends or receives.
💡 解决
的亮点		 - 首次系统性地形式化恶意LLM API路由器的威胁模型，定义了四类攻击（负载注入、密钥窃取、依赖定向注入、条件触发投递）。 - 在真实市场（淘宝、闲鱼、Shopify）购买或收集的428个路由器上进行实证测量，在野外发现了正在发生的真实攻击，包括ETH钱包被实时盗取。 - 构建了研究代理工具"Mine"，可对四个主流智能体框架复现全部四类攻击，并评估了三种可部署的客户端防御机制。 - First systematic formalization of the malicious LLM API router threat model, defining four attack classes (payload injection, secret exfiltration, dependency-targeted injection, conditional delivery). - Real-world empirical measurement across 428 routers purchased or collected from marketplaces (Taobao, Xianyu, Shopify), uncovering live attacks in the wild including active ETH wallet draining. - Introduction of "Mine," a research proxy tool that reproduces all four attack classes against four major agent frameworks, paired with evaluation of three deployable client-side defenses.
🧪 实验
内容		 研究者购买并收集了来自淘宝、闲鱼和Shopify店铺的28个付费路由器，以及从公开社区收集的400个免费路由器，共428个样本进行测量。实验还包括两项蓄意投毒研究：一是泄露一个真实OpenAI API密钥作为金丝雀凭据，观察其被滥用情况；二是部署配置薄弱的诱饵路由器，记录凭据收集和自主会话情况。最后使用Mine代理工具对LangChain等四个主流智能体框架测试四类攻击，并评估三种防御方案的有效性。 Researchers purchased 28 paid routers from Taobao, Xianyu, and Shopify storefronts and collected 400 free routers from public communities (428 total) for empirical measurement. Two poisoning studies were also conducted: one leaked a real OpenAI API key as a canary credential to track exploitation, and another deployed weakly configured decoy routers to monitor credential harvesting and autonomous agent sessions. Finally, the Mine research proxy was used to test all four attack classes against four major agent frameworks and evaluate three proposed client-side defenses.
📊 实验
结果		 在428个被调查路由器中：1个付费和8个免费路由器正在主动注入恶意代码；2个部署了自适应规避触发器；17个访问了研究者布置的AWS金丝雀凭据；1个正在从研究者控制的私钥中实时盗取ETH。在投毒研究中：泄露的OpenAI密钥产生了1亿个GPT-5.4 token以及超过7个Codex会话；配置薄弱的诱饵路由器带来了20亿计费token、440个Codex会话中收集的99个凭据，以及401个已在完全自主"YOLO模式"下运行的会话。Mine代理成功对全部四个智能体框架复现了四类攻击。 Among 428 surveyed routers: 1 paid and 8 free routers were actively injecting malicious code; 2 routers deployed adaptive evasion triggers; 17 routers accessed researcher-owned AWS canary credentials; and 1 router actively drained ETH from a researcher-controlled private key. In the poisoning studies: a leaked OpenAI key generated 100 million GPT-5.4 tokens and more than 7 Codex sessions; weakly configured decoys yielded 2 billion billed tokens, 99 harvested credentials across 440 Codex sessions, and 401 sessions already running in fully autonomous "YOLO mode." The Mine proxy successfully demonstrated all four attack classes against all four tested agent frameworks.
🏁 结论 本文首次系统性地测量了恶意LLM API路由器这一真实且活跃的攻击面，证明供应链中间商可在野外悄无声息地劫持智能体行为、窃取凭据并盗取资产。Mine工具及所提出的客户端防御方案为社区提供了评估基准和可操作的缓解措施。 This paper presents the first systematic measurement of malicious LLM API routers as a real and active attack surface, demonstrating that supply chain intermediaries can silently hijack agent behavior, steal credentials, and drain assets in the wild. The Mine tool and proposed client-side defenses provide both an evaluation benchmark and actionable mitigations for the community.

⚙️ Method · 方法详解

作者将路由器形式化为具有完整明文访问权限的应用层代理，定义了两类核心攻击：负载注入（AC-1，向智能体响应中插入恶意工具调用指令）和密钥窃取（AC-2，从截获的消息中静默收集API密钥和凭据）。同时定义了两种自适应规避变体：依赖定向注入（AC-1.a，针对智能体使用的特定依赖包）和条件触发投递（AC-1.b，仅在特定条件下触发恶意行为以逃避检测）。随后构建了研究代理"Mine"，对四个真实智能体框架复现全部四类攻击，并评估三种客户端防御：失败关闭策略门、响应侧异常筛查和仅追加透明日志。 The authors formalize a threat model where a router acts as an application-layer proxy with full plaintext access to JSON payloads, defining two core attack classes — payload injection (AC-1), which inserts malicious tool-call instructions into agent responses, and secret exfiltration (AC-2), which silently harvests API keys and credentials from intercepted messages. Two adaptive evasion variants are also defined: dependency-targeted injection (AC-1.a), which targets specific packages used by the agent, and conditional delivery (AC-1.b), which only triggers malicious behavior under certain conditions to evade detection. They then build Mine, a controlled research proxy that implements all four attacks against four real agent frameworks (LangChain, AutoGPT, etc.), and evaluate three client-side defenses: a fail-closed policy gate, response-side anomaly screening, and append-only transparency logging.

📐 Key Formulas · 关键公式

\[\text{AC-1: } r' = \text{Router}(r) = r \oplus \delta_{\text{inject}}\]
💡 负载注入攻击：路由器将原始响应r篡改为r'，其中δ_inject为注入的恶意负载（工具调用指令等）。 Payload injection attack: the router tampers with the original response r by appending or replacing content with a malicious delta (injected tool calls or instructions), producing r'.
\[\text{AC-2: } \text{Exfil}(m) = \{s \mid s \in m,\ s \in \mathcal{S}_{\text{secret}}\}\]
💡 密钥窃取攻击：路由器从截获的消息m中扫描并提取属于密钥集合S_secret的所有敏感字符串s（如API Key、私钥等）。 Secret exfiltration attack: the router scans each intercepted message m and extracts any string s that matches a known secret pattern set, such as API keys or private keys.

🌟 Why It Matters · 为什么重要

随着LLM智能体被部署在可访问代码执行、云凭据和金融资产的高风险场景中，API供应链的完整性已成为关键安全问题。本研究表明这一威胁并非理论假设——野外的主动利用攻击已经发生——并为社区提供了正式的威胁模型、实证证据和具体防御手段。 As LLM agents are deployed in high-stakes settings with access to code execution, cloud credentials, and financial assets, the integrity of the API supply chain becomes a critical security concern. This work reveals that the threat is not theoretical — active exploitation is already happening in the wild — and equips the community with a formal threat model, empirical evidence, and concrete defenses.

Exponential quantum advantage in processing massive classical data 处理海量经典数据的指数级量子优势

🧑‍🔬 Haimeng Zhao, Alexander Zlokapa, Hartmut Neven, Ryan Babbush et al.

图 Figure · Page 3 of paper

🎯 超小量子计算机在经典数据机器学习中可指数级超越经典机器
A polylogarithmic-size quantum computer provably outperforms exponentially larger classical machines on real ML tasks.
维度 中文 English
❓ 待解决
的问题		 量子计算机在经典数据上的机器学习任务中是否真正优于经典计算机，一直是个未解之谜。经典方法处理大规模分类和降维任务时需要指数级增长的计算资源。更关键的问题是：量子计算机如何高效读取经典数据，而不失去其量子优势？ Demonstrating practical, broadly applicable quantum advantage over classical computers — especially for machine learning on classical data — has remained an open problem. Classical approaches to large-scale classification and dimension reduction require massive computational resources that scale exponentially with problem size. The key challenge is that quantum computers struggle to efficiently load and read out classical data without losing their advantage.
💡 解决
的亮点		 • **量子神谕草图（Quantum Oracle Sketching）**：全新算法，仅用随机经典数据样本就能在量子叠加态中访问数据，彻底绕过数据加载瓶颈。 • **可证明的指数级优势**：数学上严格证明，多项式对数大小的量子机器可超越任何非指数级大的经典机器，即使给经典机器无限时间或BPP=BQP也成立。 • **结合经典阴影技术**：将量子神谕草图与经典阴影（classical shadows）结合，从量子处理中产出可部署的紧凑经典模型，绕过读出瓶颈。 • **Quantum oracle sketching**: A novel algorithm that accesses classical data in quantum superposition using only random classical samples, bypassing the data-loading bottleneck. • **Provable exponential advantage**: Mathematically proven that a polylogarithmic-size quantum machine outperforms any classical machine that is not exponentially larger — even with unlimited time or if BPP=BQP. • **Classical shadow integration**: Combines quantum oracle sketching with classical shadows to produce compact, deployable classical models from quantum processing, sidestepping the readout bottleneck.
🧪 实验
内容		 实验在两个真实数据集上展开：单细胞RNA测序数据（高维生物数据集）和电影评论情感分析数据集（自然语言处理任务）。将量子算法的分类与降维性能与不同规模的经典基线进行比较。核心评估指标是预测性能（分类准确率）关于模型/机器规模的函数，目标是在实际应用中验证理论上的指数级差距。 Experiments were conducted on two real-world datasets: single-cell RNA sequencing data (a high-dimensional biological dataset) and a movie review sentiment analysis dataset (a natural language processing task). The quantum algorithm's classification and dimension reduction performance was compared against classical baselines of varying sizes. The key evaluation metric was prediction performance (classification accuracy) as a function of model/machine size, with the goal of validating the theoretical exponential separation in practice.
📊 实验
结果		 量子方法相比达到同等预测性能的经典方法，实现了4到6个数量级（即1万倍到100万倍）的机器规模缩减。这一优势仅需不到60个逻辑量子比特即可实现。指数级差距在单细胞RNA测序和电影评论情感分析两个基准上均得到验证，与理论预测高度吻合。 The quantum approach demonstrated 4 to 6 orders of magnitude (10,000× to 1,000,000×) reduction in machine size compared to classical methods achieving equivalent prediction performance. This advantage was achieved with fewer than 60 logical qubits. The exponential separation was validated on both the single-cell RNA sequencing and movie review sentiment analysis benchmarks, confirming the theoretical predictions in practical, real-world settings.
🏁 结论 本文严格证明了一台多项式对数大小的量子计算机可在海量经典数据的分类和降维任务上指数级超越任何经典机器，且这一优势即使在BPP=BQP条件下仍然成立，仅依赖量子力学的正确性。这将经典数据上的机器学习确立为量子优势的广泛而自然的领域，并成为复杂性前沿量子力学的基础性验证。 This paper rigorously proves that a polylogarithmic-size quantum computer achieves exponential advantage over classical machines for large-scale classification and dimension reduction on classical data, with the separation holding even if BPP=BQP and relying solely on the correctness of quantum mechanics. The work establishes machine learning on classical data as a natural and broad domain of provable quantum advantage.

⚙️ Method · 方法详解

核心方法是"量子神谕草图"：将随机经典数据样本输入量子计算机，在量子叠加态中构建海量数据集的压缩量子表示，无需将完整数据集加载进量子内存。量子处理器直接在这一草图上执行分类或降维。为使输出可被经典系统使用，该方法与"经典阴影"协议结合，高效提取量子态的紧凑经典描述。整个流水线产出一个小型经典模型，其预测性能等同于在全量数据上训练的模型——而这对任何次指数规模的经典机器来说被证明是不可能完成的任务。 The core method is "quantum oracle sketching," which feeds random classical data samples into a quantum computer to build a compressed quantum representation (sketch) of a massive dataset in superposition — without needing to load the full dataset into quantum memory. The quantum processor then performs classification or dimension reduction directly on this sketch. To make the output classically usable, the method is paired with classical shadows, a protocol that efficiently extracts a compact classical description of the quantum state. This pipeline produces a small classical model that achieves the same predictive performance as one trained on the full data, provably impossible for any classical machine of sub-exponential size.

📐 Key Formulas · 关键公式

\[\text{Size}_{\text{quantum}} = \text{polylog}(N) \ll \text{Size}_{\text{classical}} = \Omega(N^c)\]
💡 量子机器的规模仅需数据规模N的多项式对数级，而达到同等性能的经典机器规模需要N的多项式级别，二者之间存在指数级差距。 The quantum machine size scales only polylogarithmically with data size N, while any classical machine achieving the same performance requires size polynomial in N — an exponential separation.
\[\rho_{\text{sketch}} = \mathbb{E}_{x \sim \mathcal{D}}\left[ U(x)|0\rangle\langle 0|U(x)^\dagger \right]\]
💡 量子神谕草图：对数据分布D中随机抽取的样本x，应用量子电路U(x)后取平均，得到数据集的量子密度矩阵表示（草图）。 Quantum oracle sketch: the quantum sketch is constructed as the average quantum state obtained by applying a data-dependent circuit U(x) to random samples x drawn from the data distribution D.

🌟 Why It Matters · 为什么重要

这是首批针对主流机器学习任务（而非人为构造的玩具问题）提供可证明、广泛适用量子优势的成果之一，直接挑战了"量子计算机对经典数据机器学习无益"的传统观点。由于所需逻辑量子比特不足60个，该结果在近未来量子硬件上具有可行性，有望重塑学界对量子-经典混合机器学习架构的认知。 This is one of the first results to offer a provable, broadly applicable quantum advantage for mainstream ML tasks on classical data — not just contrived toy problems — directly challenging the assumption that quantum computers cannot help with classical machine learning. With fewer than 60 logical qubits needed, this result is within reach of near-future quantum hardware and could reshape how the community thinks about quantum-classical hybrid ML pipelines.

KnowU-Bench: Towards Interactive, Proactive, and Personalized Mobile Agent Evaluation KnowU-Bench：面向交互式、主动式与个性化移动智能体的评测基准

🧑‍🔬 Tongbo Chen, Zhengxi Lu, Zhan Xu, Guocheng Shao et al.

图 Figure · Page 5 of paper

🎯 首个测试移动智能体个性化与主动干预能力的在线评测基准
A new benchmark exposing that frontier agents fail at personalized, proactive mobile assistance despite strong GUI skills.
维度 中文 English
❓ 待解决
的问题		 现有移动智能体评测基准只关注"能否完成明确任务"，却忽略了真实数字助手的核心能力：从用户行为推断偏好、判断何时主动介入或请求授权、以及何时保持沉默。目前没有任何基准在真实交互的图形界面环境中测试这些能力。 Current mobile agent benchmarks only test whether agents can complete explicit tasks, but real-world digital assistants must also learn user preferences from behavior, decide when to proactively help or ask for permission, and know when to stay silent. No existing benchmark tests this full picture in a live, interactive GUI environment.
💡 解决
的亮点		 • 提出KnowU-Bench，首个在可复现安卓模拟器上运行的在线个性化移动智能体评测基准，涵盖192个任务。 • 将用户画像完全隐藏，强迫智能体从行为日志中真正推断偏好，而非简单查询上下文，难度更贴近现实。 • 评测完整的主动决策链：GUI执行、授权协商、拒绝后克制行为，采用规则验证与LLM-as-Judge混合评分协议。 • 引入基于结构化用户画像驱动的LLM用户模拟器，支持真实的多轮偏好澄清对话。 • Introduces KnowU-Bench, the first online benchmark for personalized mobile agents running on a reproducible Android emulator, covering 192 tasks across general GUI, personalized, and proactive categories. • Hides user profiles from the agent entirely, forcing genuine preference inference from behavioral logs rather than simple context lookup — a much harder and more realistic setting. • Evaluates the full proactive decision chain: GUI execution, consent negotiation, and post-rejection restraint, using a hybrid rule-based + LLM-as-a-Judge scoring protocol. • Deploys an LLM-driven user simulator grounded in structured profiles to enable realistic multi-turn clarification dialogues.
🧪 实验
内容		 实验在KnowU-Bench上评测了多个主流前沿模型作为移动智能体，包括Claude Sonnet 4.6等顶级模型。测试集涵盖42个通用GUI任务、86个个性化任务和64个主动任务，在安卓模拟器真实环境中运行。评估指标包括任务完成率、偏好推断准确性、主动干预决策质量以及拒绝后克制行为，分别通过规则验证和LLM评判打分。 Multiple frontier models including Claude Sonnet 4.6 were evaluated as mobile agents on KnowU-Bench in a live Android emulator environment. The benchmark covers 42 general GUI tasks, 86 personalized tasks, and 64 proactive tasks. Evaluation metrics include task completion rate, preference inference accuracy, proactive intervention quality, and post-rejection restraint, scored via a hybrid rule-based and LLM-as-a-Judge protocol.
📊 实验
结果		 在需要用户偏好推断或干预校准的个性化与主动任务上，即便是顶级模型如Claude Sonnet 4.6，成功率也跌破50%，而这些模型在显式GUI执行任务上表现优秀。实验明确指出核心瓶颈是偏好获取能力与干预时机判断，而非GUI导航能力本身。 Agents that excel at explicit GUI task execution drop below 50% success rate on personalized and proactive tasks requiring user preference inference or intervention calibration — even frontier models like Claude Sonnet 4.6. The core bottlenecks identified are preference acquisition and intervention calibration, not GUI navigation skill.
🏁 结论 KnowU-Bench揭示了GUI操作能力与可信个人助理能力之间一个此前未被量化的根本性差距——当前顶级模型在个性化与主动任务上表现大幅下滑。该基准为领域提供了首个严格的交互式评测框架，为推动真正个性化移动智能的发展奠定了基础。 KnowU-Bench reveals a fundamental and previously unmeasured gap between agents that are competent at GUI operation and agents that can serve as trustworthy personal assistants — current frontier models fail dramatically on personalized and proactive tasks. The benchmark provides the field with the first rigorous, interactive evaluation framework to drive progress toward genuine personalized mobile intelligence.

⚙️ Method · 方法详解

KnowU-Bench基于可复现的安卓模拟环境构建，将任务分为三层：通用GUI任务（基础执行能力）、个性化任务（需从隐藏画像和行为日志推断偏好）、主动任务（需判断是否及如何主动干预）。系统部署了一个基于结构化用户画像驱动的LLM用户模拟器，用于响应智能体的澄清请求并模拟真实授权对话。评估采用混合协议：规则验证具体GUI操作的正确性，LLM-as-Judge对开放式偏好获取质量、授权协商过程以及拒绝后克制行为进行打分。 KnowU-Bench is built on a reproducible Android emulation environment and organizes tasks into three tiers: general GUI tasks (baseline execution), personalized tasks (requiring preference inference from hidden profiles and behavioral logs), and proactive tasks (requiring the agent to decide whether and how to intervene). An LLM-driven user simulator, grounded in structured user profiles, responds to agent clarification requests and simulates realistic consent dialogues. Evaluation uses a hybrid protocol: rule-based checks verify concrete GUI actions (e.g., correct settings applied), while an LLM-as-a-Judge scores open-ended preference elicitation quality, consent negotiation, and post-rejection restraint behavior.

🌟 Why It Matters · 为什么重要

随着大模型驱动的移动智能体走向实际部署，可信的偏好推断与主动辅助能力变得至关重要，而此前没有基准对此进行衡量。KnowU-Bench填补了这一空白，并表明即使最强的模型也远未达到真正个性化助理的水平，为研究界指明了正确的攻关方向。 As LLM-powered mobile agents move toward real deployment, the ability to infer preferences and proactively assist users in a trustworthy way becomes critical — yet no prior benchmark measured this. KnowU-Bench fills this gap and reveals that even the strongest available models are far from ready for genuine personalized assistance, redirecting research attention to the right bottlenecks.

🎯 多用户经验驱动的LLM智能体技能自动集体进化框架
A framework that automatically evolves LLM agent skills collectively from multi-user experience.
维度 中文 English
❓ 待解决
的问题		 像OpenClaw这样的LLM智能体依赖可复用的"技能"来完成复杂任务，但这些技能在部署后基本保持静止不变。不同用户在使用过程中会反复遭遇相似的失败模式和有效工作流，却无法被系统自动学习和沉淀。现有系统缺乏将多用户的异质交互经验转化为可靠技能更新的机制。 LLM agents like OpenClaw rely on reusable skills to handle complex tasks, but these skills stay frozen after deployment. When multiple users interact with the system, similar failure patterns and useful workflows get rediscovered repeatedly from scratch — the system never learns from accumulated experience. There is no mechanism to turn diverse, heterogeneous user interactions into reliable, system-wide skill improvements.
💡 解决
的亮点		 - **集体技能进化**：利用跨用户交互轨迹而非单一用户反馈持续改进技能，实现系统级的多样化经验学习。 - **自主进化器**：由LLM驱动的进化器自主识别轨迹中的重复行为模式，并将其转化为具体的技能精炼或新能力扩展，无需人工标注。 - **共享技能库与知识传播**：改进后的技能存储在共享库中并同步至所有用户，一处发现立即惠及全体，实现跨用户知识迁移。 - **Collective Skill Evolution**: Skills are improved continuously using cross-user interaction trajectories, not just single-user feedback, enabling system-wide learning from diverse experiences. - **Autonomous Evolver**: An LLM-powered evolver autonomously identifies recurring behavioral patterns across trajectories and translates them into concrete skill refinements or new capabilities — with no manual labeling needed. - **Shared Skill Repository with Propagation**: Improved skills are stored in a shared repository and synchronized across all users, so a discovery in one context instantly benefits everyone else.

内容		 实验在WildClawBench基准上进行，该基准面向真实世界的智能体场景。以Qwen3-Max作为基础模型，对比了SkillClaw框架与静态技能系统（无进化机制）的基线表现，评估了在有限交互和反馈条件下的任务完成性能，并考察了跨用户知识迁移和累积能力提升的效果。 Experiments were conducted on WildClawBench, a benchmark designed for real-world agent scenarios. The base model used was Qwen3-Max. SkillClaw was compared against a static-skill baseline (no evolution mechanism), evaluating task performance under conditions of limited user interaction and feedback. The study also examined cross-user knowledge transfer and cumulative capability gains over time.

结果		 在WildClawBench基准上，SkillClaw在有限交互和反馈条件下显著提升了Qwen3-Max在真实世界智能体场景中的性能表现。摘要中描述结果为"显著提升"，超越静态技能基线，尤其在低数据条件下展现出明显的跨用户知识迁移与累积性能增益（具体精确数值见正文实验部分）。 On WildClawBench, SkillClaw significantly improves the performance of Qwen3-Max in real-world agent scenarios even with limited interaction and feedback. Specific numerical improvements (e.g., exact task success rates or percentage gains) are described as significant in the abstract but precise figures are not detailed in the provided abstract; the paper reports meaningful gains over the static-skill baseline under low-data conditions.
🏁 结论 SkillClaw证明了跨用户的集体经验可以被自动收集并转化为持续改进的智能体技能，在无需用户额外付出的前提下实现累积性能力增长。这确立了一种新范式：部署后的LLM智能体系统能够随使用不断变得更聪明，而非保持静止。 SkillClaw proves that collective, cross-user experience can be automatically harvested and converted into continuously improving agent skills, enabling cumulative capability growth without any extra user effort. This establishes a new paradigm where deployed LLM agent systems become progressively smarter through use, rather than remaining static.

⚙️ Method · 方法详解

SkillClaw持续收集多用户在正常使用过程中产生的交互轨迹。一个由LLM驱动的自主进化器模块定期处理这些轨迹，识别重复出现的工作流模式、常见失败模式和有效的工具使用策略。基于这些模式，进化器对已有技能进行精炼（提升可靠性或泛化性），或合成全新技能。所有更新的技能被推送至集中式共享库并分发给所有用户，形成零用户额外负担的集体改进闭环。 SkillClaw continuously collects interaction trajectories generated by multiple users during normal agent usage. An autonomous evolver module — itself powered by an LLM — periodically processes these trajectories to detect recurring workflow patterns, common failure modes, and effective tool-use strategies. Based on these patterns, the evolver refines existing skills (improving reliability or generality) or synthesizes entirely new skills. All updated skills are pushed to a centralized shared repository and distributed back to all users, creating a closed feedback loop of collective improvement that requires zero extra effort from end users.

🌟 Why It Matters · 为什么重要

大多数生产级LLM智能体系统部署后便固化不变，浪费了日常用户交互中蕴含的大量隐性信号。SkillClaw提供了一种将这些信号转化为持续复利式改进的原则性可扩展方案，推动AI领域迈向真正自我进化的智能体生态系统。 Most production LLM agent systems are frozen after deployment, wasting the enormous implicit signal embedded in daily user interactions. SkillClaw offers a principled, scalable solution to turn this signal into compounding improvements, moving the field toward truly self-improving agent ecosystems.
