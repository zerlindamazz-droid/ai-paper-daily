OpenSeeker-v2: Pushing the Limits of Search Agents with Informative and High-Difficulty Trajectories OpenSeeker-v2：用高信息量、高难度轨迹突破搜索智能体的能力上限

🧑‍🔬 Yuwen Du, Rui Ye, Shuo Tang, Keduan Huang et al.

图 Figure · Page 1 of paper

🎯 仅用1万条高质量轨迹做SFT，超越工业级重训练流水线搜索智能体
SFT on 10.6k high-quality trajectories beats industry CPT+SFT+RL pipelines on frontier search benchmarks.
维度 中文 English
❓ 待解决
的问题		 训练顶尖的大语言模型搜索智能体一直被工业界把持，需要经历预训练、持续预训练（CPT）、监督微调（SFT）和强化学习（RL）等极其耗费资源的流程。学术团队由于算力和数据资源不足，难以与之竞争，导致前沿搜索智能体研究对开放社区几乎不可及。 Training state-of-the-art LLM search agents has been dominated by industry players using extremely resource-heavy pipelines that combine pre-training, continual pre-training (CPT), supervised fine-tuning (SFT), and reinforcement learning (RL). Academic teams lack the compute and data resources to compete, making frontier search agent research largely inaccessible to the open community.
💡 解决
的亮点		 • 仅用SFT即可媲美甚至超越工业界CPT+SFT+RL重型流水线，证明高质量数据比复杂训练流程更关键。 • 提出三项数据合成改进：①扩大知识图谱规模以增加探索多样性；②扩展工具集以覆盖更广功能；③严格的低步骤过滤，仅保留高难度、高信息量轨迹。 • 首个由纯学术团队基于SFT打造、在30B规模下达到SOTA的搜索智能体，并完全开源模型权重。 • Demonstrates that a simple SFT-only approach, when powered by high-quality data, can match or exceed complex CPT+SFT+RL industrial pipelines — a surprising and practically important finding. • Introduces three targeted data synthesis modifications: (1) scaling knowledge graph size for richer exploration diversity, (2) expanding the tool set for broader functional coverage, and (3) strict low-step filtering to retain only high-difficulty, informative trajectories. • First state-of-the-art search agent at the 30B-model scale developed entirely by an academic team using only SFT, with full open-source release of model weights.
🧪 实验
内容		 在四个挑战性基准上评估模型的深度搜索和复杂推理能力：BrowseComp（英文）、BrowseComp-ZH（中文）、Humanity's Last Exam（HLE）和xbench。基线包括通义DeepResearch等工业界强模型（采用CPT+SFT+RL训练），以及同等30B规模的ReAct范式智能体，所有对比均在相同模型规模和范式下进行，确保公平性。 The model was evaluated on four challenging benchmarks targeting deep search and complex reasoning: BrowseComp (English), BrowseComp-ZH (Chinese), Humanity's Last Exam (HLE), and xbench. Baselines include strong industrial systems such as Tongyi DeepResearch (trained with CPT+SFT+RL), as well as other 30B-scale ReAct-paradigm agents. All comparisons are performed at the same model scale (30B) and under the same ReAct paradigm to ensure fair comparison.
📊 实验
结果		 OpenSeeker-v2在全部4个基准上均达到30B规模ReAct智能体的SOTA：BrowseComp 46.0%（超通义DeepResearch的43.4%，高2.6个百分点）、BrowseComp-ZH 58.1%（超通义46.7%，高11.4个百分点）、Humanity's Last Exam 34.6%（超通义32.9%，高1.7个百分点）、xbench 78.0%（超通义75.0%，高3.0个百分点）。全部成果仅用1.06万条训练数据、不依赖CPT和RL实现。 OpenSeeker-v2 achieves SOTA across all 4 benchmarks among 30B-scale ReAct agents: 46.0% on BrowseComp (+2.6pp over Tongyi DeepResearch's 43.4%), 58.1% on BrowseComp-ZH (+11.4pp over Tongyi's 46.7%), 34.6% on Humanity's Last Exam (+1.7pp over Tongyi's 32.9%), and 78.0% on xbench (+3.0pp over Tongyi's 75.0%). All this is achieved with only 10.6k training examples and no CPT or RL.
🏁 结论 OpenSeeker-v2证明，通过精心设计的高信息量、高难度轨迹数据合成，仅用1.06万条样本进行SFT，即可在前沿搜索基准上超越工业界CPT+SFT+RL重型流水线，使学术团队也能开展顶级搜索智能体研究。 OpenSeeker-v2 proves that a carefully designed SFT pipeline with informative and high-difficulty trajectories — using just 10.6k examples — can surpass industrial CPT+SFT+RL pipelines on frontier search benchmarks, democratizing access to state-of-the-art search agent research for academic teams.

链接		 arXiv: 2605.04036v1  ·  📥 PDF

⚙️ Method · 方法详解

作者构建了一套数据合成流水线，无需昂贵的RL或持续预训练即可生成高质量、高难度的搜索轨迹。首先，扩大知识图谱规模，创造更复杂多样的推理环境；其次，扩展可用工具集，让模型学习更丰富的动作空间；最后，应用严格的低步骤过滤，丢弃过于简单的轨迹，只保留具有挑战性和信息量的样本。最终用筛选出的1.06万条训练数据，在ReAct范式下对30B参数基础模型进行标准SFT微调。 The authors build a data synthesis pipeline that generates high-quality, high-difficulty search trajectories without requiring expensive RL or continual pre-training. First, they scale up knowledge graphs to create more complex and diverse reasoning environments. Second, they expand the available tool set so the agent learns to use a richer action space. Third, they apply strict low-step filtering — discarding trajectories that are solved too easily — ensuring only challenging, informative examples survive for training. The resulting 10.6k training examples are then used to fine-tune a 30B-parameter base model via standard SFT under the ReAct paradigm.

🌟 Why It Matters · 为什么重要

本文从根本上挑战了"前沿AI智能体训练必须依赖工业级算力和复杂多阶段流水线"的假设，证明数据质量可以弥补流程复杂度的不足。通过开源模型权重和方法，大幅降低了学术研究者参与和推进搜索智能体研究的门槛。 This paper fundamentally challenges the assumption that frontier AI agent training requires massive industrial compute and complex multi-stage pipelines, showing data quality can compensate for pipeline complexity. By open-sourcing both the model weights and methodology, it significantly lowers the barrier for academic researchers to contribute to and advance search agent development.

ProgramBench: Can Language Models Rebuild Programs From Scratch? ProgramBench：语言模型能从零重建程序吗？

🧑‍🔬 John Yang, Kilian Lieret, Jeffrey Ma, Parth Thakkar et al.

图 Figure · Page 6 of paper

🎯 全面测试大模型从零重建真实程序的能力基准
A holistic benchmark challenging LLM agents to rebuild real-world programs from scratch using only documentation.
维度 中文 English
❓ 待解决
的问题		 现有代码生成基准只考察"修一个bug"或"加一个功能"这类局部任务，无法衡量AI从零设计并实现一个完整软件项目的能力。而现实中，AI编程智能体正越来越多地被用于从头创建和维护整个代码库。因此，业界缺乏一个能评估模型做出高层软件架构决策的基准。 Current benchmarks for code-generating LLMs test narrow tasks like fixing a single bug or adding one feature. They don't capture the harder, real-world challenge of building a complete software project from nothing — the kind of work AI coding agents are increasingly being deployed to do. There is no good way to measure whether a model can make high-level software architecture decisions end-to-end.
💡 解决
的亮点		 • 全新"从零重建"评估范式：智能体仅凭程序文档和可执行文件，独立设计并实现功能等价的代码库，考察高层架构决策能力。 • 智能体驱动的模糊测试：自动生成端到端行为测试用例，无需预设实现结构，测试更真实客观。 • 任务覆盖广泛：200个任务，从小型命令行工具到FFmpeg、SQLite、PHP解释器等主流真实软件，是目前最全面的代码生成基准。 • Novel "rebuild from scratch" evaluation paradigm: agents receive only a program binary and its documentation, then must independently architect and implement a functionally equivalent codebase. • Agent-driven fuzzing for test generation: behavioral end-to-end tests are automatically generated via fuzzing, avoiding the need to prescribe any implementation structure. • Broad task coverage: 200 tasks spanning from small CLI utilities to major real-world software (FFmpeg, SQLite, PHP interpreter), making it the most holistic code-generation benchmark to date.
🧪 实验
内容		 研究者在200个任务上评估了9个主流大语言模型，任务范围从命令行小工具到FFmpeg、SQLite、PHP解释器等大型软件。测试用例通过智能体驱动的模糊测试自动生成，而非人工编写。评估指标包括：每个任务的测试通过率，以及完整解决某任务（所有测试全部通过）的比例。 The authors evaluated 9 state-of-the-art language models on 200 tasks spanning CLI tools to major software projects like FFmpeg, SQLite, and the PHP interpreter. Behavioral tests were generated via agent-driven fuzzing rather than hand-written test cases. Models were assessed by the percentage of generated tests they pass and the percentage of tasks fully resolved (all tests passed).
📊 实验
结果		 没有任何模型完整解决了哪怕一个任务（完整任务完成率为0%）。表现最好的模型，仅在3%的任务上通过了95%的测试。所有9个被评估的大模型都倾向于生成单体、单文件代码，与人类编写的模块化代码库差异显著。随着程序复杂度提升（如FFmpeg、SQLite），模型性能急剧下降。 No model fully resolved any task (0% full task completion). The best-performing model passed 95% of tests on only 3% of tasks. All 9 evaluated LMs showed a strong tendency to produce monolithic, single-file implementations that diverge significantly from human-written, modular codebases. Performance drops sharply as program complexity scales up toward real-world software like FFmpeg and SQLite.
🏁 结论 ProgramBench揭示出，当前大语言模型距离真正的全流程软件工程能力还相差甚远——没有任何模型能完整重建200个真实程序中的任何一个。该基准建立了一个严格、不依赖实现结构的评估框架，明确指出了模型在架构级推理和长程代码生成方面的关键缺口。 ProgramBench reveals that current LLMs are far from capable of holistic software engineering — none could fully reconstruct even one of 200 real-world programs. The benchmark establishes a rigorous, structure-agnostic evaluation framework that exposes critical gaps in architecture-level reasoning and long-horizon code generation for the field to address.

链接		 arXiv: 2605.03546v1  ·  📥 PDF

⚙️ Method · 方法详解

ProgramBench只给智能体提供两样东西：参考程序的文档和可执行二进制文件。智能体需要自主设计代码库，使其行为与参考程序完全一致。为了不限制实现方式，评估采用"智能体驱动的模糊测试"——由LLM智能体自动生成多样化的输入测试用例，比对被测程序与参考程序的输出是否一致。这种黑盒行为测试方法对各种不同实现方式都公平适用。 ProgramBench provides agents with only two inputs: the reference program's documentation and its executable binary. Agents must produce a codebase that behaviorally matches the reference. To evaluate without locking in any specific implementation approach, the benchmark uses agent-driven fuzzing — an LLM agent automatically generates diverse input test cases and checks whether the agent-built program's outputs match the reference binary's outputs. This black-box behavioral testing allows fair comparison across wildly different implementation strategies.

🌟 Why It Matters · 为什么重要

随着LLM编程智能体越来越多地被用于自主构建和维护完整代码库，ProgramBench首次对这一端到端能力进行了严格评估，揭示出当前模型仍存在根本性不足。该基准将引导未来研究从"修补单一bug"走向真正意义上的软件工程智能。 As LLM-powered coding agents are increasingly trusted to build and maintain entire codebases autonomously, ProgramBench provides the first rigorous evaluation of this end-to-end capability, revealing that current models still fall critically short. This benchmark will guide future research toward closing the gap between narrow code-fixing skills and true software engineering competence.

ARIS: Autonomous Research via Adversarial Multi-Agent Collaboration ARIS：通过对抗性多智能体协作实现自主科学研究

🧑‍🔬 Ruofeng Yang, Yongcan Li, Shuai Li

图 Figure · Page 7 of paper

🎯 对抗多智能体框架，自动执行机器学习科研流程并内置声明核查机制
An adversarial multi-agent harness that autonomously runs ML research workflows with built-in claim verification.
维度 中文 English
❓ 待解决
的问题		 基于大语言模型的科研智能体容易产生看似合理但缺乏证据支撑的研究结论——这是一种"静默失败"模式：长时间运行的智能体流程会生成证据不完整、被曲解或被执行者框架"偷换"的结论。现有智能体系统缺乏系统性机制来捕捉自主科研流程中的这类隐性错误。 LLM-based research agents can produce plausible-sounding but unsupported claims — a "silent failure" where long-running workflows generate results whose evidence is incomplete, misreported, or inherited from the executor's own framing. Existing agent systems lack principled mechanisms to catch these subtle integrity failures in autonomous research pipelines.
💡 解决
的亮点		 • 跨模型对抗协作：执行者LLM推进研究进展，来自不同模型家族的审稿者LLM负责批评中间产物，防止"回音室"式的确认偏误。 • 三层架构（执行层/编排层/保障层），包含65+个可复用的Markdown定义技能模块和持久化研究知识库，支持先验发现的迭代复用。 • 三阶段声明保障流水线（完整性验证→结果-声明映射→声明审计），将论文陈述与原始证据和结构化声明账本交叉核验。 • Cross-model adversarial collaboration: an executor LLM drives research while a reviewer from a *different* model family critiques artifacts, preventing echo-chamber confirmation bias. • Three-layer architecture (execution / orchestration / assurance) with 65+ reusable Markdown-defined skills and a persistent research wiki for iterative knowledge reuse. • A three-stage claim assurance pipeline (integrity verification → result-to-claim mapping → claim auditing) that cross-checks manuscript statements against raw evidence and a structured claim ledger.
🧪 实验
内容		 本文描述的是ARIS作为开源科研框架的早期部署经验，而非受控基准测试研究。评估主要集中在自主ML科研工作流的定性案例分析上，考察对抗性审稿者是否能成功捕捉无证据支撑的声明、声明审计流水线是否能正确将实验结果映射到论文陈述，以及自改进循环是否能产出有效的框架升级方案。系统在包括实验设计、代码执行、结果分析和论文撰写等机器学习科研任务上进行了测试。 The paper describes early deployment experience of ARIS as an open-source research harness rather than a controlled benchmark study. Evaluations center on qualitative case studies of autonomous ML research workflows, assessing whether the adversarial reviewer successfully catches unsupported claims, whether the claim-auditing pipeline correctly maps results to manuscript statements, and whether the self-improvement loop produces valid harness improvements. The system is tested on machine-learning research tasks including experiment design, code execution, result analysis, and paper writing.
📊 实验
结果		 作为系统报告，本文未提供标准化的基准测试数字或与其他自主科研系统的直接对比数据。主要报告成果包括：65+技能库可在多种ML科研工作流中复用；三阶段保障流水线能在论文定稿前有效拦截无证据支撑的声明；对抗性跨模型审稿者配置相较单模型配置降低了确认偏误。摘要中未报告声明核查的精确率/召回率或论文质量评分等具体数字指标。 As a system report describing early deployment experience, the paper does not provide standardized benchmark numbers or head-to-head comparisons with other autonomous research systems. Key reported outcomes include: the 65+ skill library enables reuse across diverse ML research workflows; the three-stage assurance pipeline demonstrably intercepts unsupported claims before manuscript finalization; and the adversarial cross-model reviewer setup reduces confirmation bias relative to single-model configurations. Specific precision/recall figures for claim verification or paper quality scores are not reported in the abstract.
🏁 结论 ARIS证明了对抗性跨模型协作结合结构化声明保障流水线是一种有效的架构策略，可以减少自主ML科研流程中的"静默失败"（看似合理但缺乏支撑的声明）。其开源设计——包含模块化技能、持久化知识库和审稿者门控的自改进机制——为可信赖的自主科研智能体建立了可复用的参考蓝图。 ARIS demonstrates that adversarial cross-model collaboration combined with a structured claim-assurance pipeline is an effective architectural strategy for reducing silent failures (plausible but unsupported claims) in autonomous ML research workflows. Its open-source design with modular skills, persistent wikis, and reviewer-gated self-improvement establishes a reusable blueprint for trustworthy autonomous research agents.
🔗 论文
链接		 arXiv: 2605.03042v1  ·  📥 PDF

⚙️ Method · 方法详解

ARIS通过三层架构协调自主ML科研流程。执行层提供65+个Markdown定义的可复用技能、基于MCP的模型集成、持久化研究知识库以及确定性图表生成能力。编排层管理五个端到端研究工作流，支持可调节的努力程度和可配置的审稿者路由。保障层执行三阶段流程——完整性验证、结果-声明映射、声明审计——并辅以五轮科学编辑流水线、数学证明检查和PDF渲染视觉检查。系统还内置自改进循环，记录研究痕迹并提议框架升级方案，只有在审稿者批准后才会正式采纳。 ARIS coordinates autonomous ML research through three architectural layers. The execution layer supplies 65+ reusable skills (coded in Markdown), MCP-based model integrations, a persistent research wiki, and deterministic figure generation. The orchestration layer manages five end-to-end research workflows with configurable effort and reviewer routing. The assurance layer applies a three-stage process — integrity verification, result-to-claim mapping, and claim auditing — alongside a five-pass scientific editing pipeline, mathematical proof checking, and visual PDF inspection. A self-improvement loop records research traces and proposes harness upgrades, which are adopted only after reviewer approval.

🌟 Why It Matters · 为什么重要

随着LLM被越来越多地部署于自动化科学研究，生成看似可信但实为幻觉的结论对科研诚信构成直接威胁。ARIS提供了一个具体且开源的框架，研究社区可以直接采用并扩展，以构建在每个步骤都具备原则性核查机制的更可信AI科研助手。 As LLMs are increasingly deployed to automate scientific research, the risk of generating convincing but hallucinated results poses a direct threat to research integrity. ARIS offers a concrete, open-source framework that the community can adopt and extend to build more trustworthy AI research assistants with principled verification at every step.

HeavySkill: Heavy Thinking as the Inner Skill in Agentic Harness HeavySkill：将深度思考作为智能体框架中的内在技能

🧑‍🔬 Jianing Wang, Linsen Guo, Zhengyu Chen, Qi Guo et al.

图 Figure · Page 6 of paper

🎯 将"深度思考"定义为可学习内在技能，通过强化学习扩展推理能力。
Heavy thinking reframed as a learnable inner skill that outperforms Best-of-N and scales via RL.
维度 中文 English
❓ 待解决
的问题		 现代智能体AI系统依赖复杂的编排框架（多个智能体、记忆模块、工具调用）来完成困难推理任务，但真正驱动性能的机制并不清晰。这些脆弱的多层系统难以理解、维护和扩展。核心问题是：能否将深度推理能力直接内化到模型参数中，而不依赖外部脆弱的编排层？ Modern agentic AI systems rely on complex orchestration frameworks (multiple agents, memory, tools) to tackle hard reasoning tasks, but it's unclear what actually drives their performance. These brittle multi-layer systems are hard to interpret, maintain, and scale. The core question is: can the heavy reasoning capability be internalized directly into a model rather than delegated to fragile external scaffolding?
💡 解决
的亮点		 • 全新视角：将"深度思考"重新定义为模型参数中内化的技能，而非仅仅是系统层面的执行单元——从系统设计转向模型能力的概念性转变。 • 两阶段流程：将深度思考具体分解为（1）并行推理和（2）汇总归纳两个阶段，可插入任意智能体框架之下运行。 • 强化学习可扩展性：证明推理的深度（推理链长度）和广度（推理路径多样性）均可通过强化学习进行训练和扩展，支持LLM自我进化。 • Novel Perspective: Reframes "heavy thinking" not just as an execution unit in agent pipelines, but as an *internalized skill* encoded in model weights — a conceptual shift from system design to model capability. • Two-Stage Pipeline: Introduces a concrete decomposition of heavy thinking into (1) parallel reasoning and (2) summarization, which can plug under any agentic harness. • RL Scalability: Shows that both the depth (length of reasoning chains) and width (diversity of reasoning paths) of this skill are learnable and scalable via reinforcement learning, enabling self-evolving LLMs.
🧪 实验

⚙️ Method · 方法详解

HeavySkill提出，与其依赖外部编排系统的复杂性，不如将"深度思考"技能直接内化到模型参数中。该技能被构建为两阶段流程：首先，模型生成多条并行推理链（广度维度）；然后，将这些推理链综合归纳为最终答案（深度维度）。这种设计使该技能能够透明地运行在任意现有智能体框架之下，无需修改框架结构。此外，通过强化学习对该技能进行进一步训练和扩展，奖励模型在内部推理过程中同时提升广度和深度。 HeavySkill proposes that instead of relying on external orchestration complexity, models should internalize a "heavy thinking" skill directly into their parameters. This skill is structured as a two-stage pipeline: first, the model generates multiple parallel reasoning chains (width), then it synthesizes/summarizes them into a final answer (depth). This design allows the skill to operate transparently beneath any existing agentic harness without modifying the framework. The skill is further trained and scaled using reinforcement learning, rewarding the model for improving both the breadth and depth of its internal reasoning process.
