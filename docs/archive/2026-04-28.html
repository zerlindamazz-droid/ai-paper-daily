AgenticQwen: Training Small Agentic Language Models with Dual Data Flywheels for Industrial-Scale Tool Use AgenticQwen：用双数据飞轮训练小型智能体语言模型以支持工业级工具使用

🧑‍🔬 Yuanjie Lyu, Chengyu Wang, Haonan Zheng, Yuanhao Yue et al.

图 Figure · Page 3 of paper

🎯 双飞轮强化学习让小模型高效掌握多步工具调用能力
Dual data flywheels with RL train small models to match large ones on agentic tool-use tasks.
维度 中文 English
❓ 待解决
的问题		 现实工业场景需要语言模型像智能体一样，能够进行多步推理并调用外部工具（如搜索、数据分析）。但大型模型成本高、延迟大，在工业部署中受限；而小模型在复杂任务上表现往往不足。如何让小模型也能胜任高难度的多步工具使用任务，是当前的核心挑战。 Real-world industrial applications need language models that can reason across multiple steps and call external tools like search engines or data analysis APIs. Large models are too expensive and slow for production use, but small models struggle with complex agentic tasks. The challenge is closing this performance gap for small models under cost and latency constraints.
💡 解决
的亮点		 • **双数据飞轮机制**：推理飞轮从模型错误中自动生成更难的训练样本；智能体飞轮将线性工作流扩展为多分支行为树，模拟真实复杂决策场景。 • **推理RL + 智能体RL联合训练**：两阶段强化学习分别强化模型的推理能力和工具调用规划能力，互相增强。 • **合成数据与开源数据结合**：仅用少量开源数据配合大规模合成数据，显著降低数据获取成本，同时保持泛化性。 • **Dual Data Flywheels**: A reasoning flywheel that mines hard examples from model errors, and an agentic flywheel that expands linear workflows into multi-branch behavior trees mimicking real-world decision complexity. • **Two-Stage RL Training**: Reasoning RL and agentic RL are combined in a multi-round framework, each reinforcing different competencies — logical inference and multi-step tool planning. • **Synthetic-First Data Strategy**: Training relies heavily on automatically generated synthetic data with only a small amount of open-source data, drastically reducing data collection overhead.
🧪 实验
内容		 实验在多个公开智能体基准和工业内部系统上进行，基线涵盖同规模及更大规模的主流语言模型。公开基准包括工具调用、多步推理等智能体任务评测集；工业实验则在搜索和数据分析两类真实任务上，将 AgenticQwen 与更大规模模型对比，评估任务完成质量与效率。 Experiments were conducted on multiple public agentic benchmarks covering tool use and multi-step reasoning tasks, with baselines including both same-scale and much larger language models. Additionally, the models were evaluated in a real industrial agent system on search and data analysis tasks, measuring task completion quality against larger proprietary and open models under production conditions.
📊 实验
结果		 AgenticQwen 在多个公开智能体基准上取得强劲表现，具体数值据论文报告优于同规模模型；在工业智能体系统中，AgenticQwen 在搜索和数据分析任务上显著缩小了与更大规模模型之间的差距，以小模型成本接近大模型效果。模型检查点和合成数据已开源（HuggingFace），代码已集成到 EasyDistill 框架。 AgenticQwen achieves strong results across multiple public agentic benchmarks, outperforming models of comparable size. In the industrial agent system, AgenticQwen closes the performance gap with significantly larger models on both search and data analysis tasks — delivering near-large-model quality at small-model cost. Exact benchmark scores, model checkpoints, and part of the synthetic data are publicly released on HuggingFace; training code is available on GitHub and integrated into EasyDistill.
🏁 结论 本文证明了通过双数据飞轮结合两阶段强化学习，可以高效训练出在多步工具使用任务上接近大型模型表现的小型智能体语言模型。AgenticQwen 为工业级智能体部署提供了一条低成本、可扩展的训练路径，并通过开源推动了该领域的可复现研究。 This paper demonstrates that small language models trained with dual data flywheels and multi-round RL can match much larger models on complex agentic tool-use tasks. AgenticQwen provides a cost-effective, scalable training paradigm for industrial-grade agent deployment, with full open-source release enabling reproducible research.

⚙️ Method · 方法详解

AgenticQwen 的训练框架分为两个相互配合的强化学习阶段。首先是推理RL阶段：从模型推理错误中挖掘难例，自动构造更具挑战性的训练数据（推理飞轮），让模型在迭代中不断提升推理能力。其次是智能体RL阶段：智能体飞轮将简单的线性工具调用流程扩展为具有多分支条件判断的行为树，模拟真实场景中的复杂决策路径，并以此生成合成训练任务。两个飞轮自动迭代生成越来越难的任务，形成持续自我改进的训练循环，最终使小型模型（如Qwen系列）在工具使用上媲美大型模型。 AgenticQwen uses a two-phase training framework built around reinforcement learning. In the first phase (Reasoning RL), a reasoning flywheel identifies examples where the model makes mistakes and generates progressively harder training data from those errors, iteratively improving logical reasoning. In the second phase (Agentic RL), an agentic flywheel converts simple linear tool-call sequences into multi-branch behavior trees that capture the conditional complexity of real-world agent workflows, producing rich synthetic training tasks. Both flywheels operate automatically and continuously, creating a self-improving data loop. The resulting model family (AgenticQwen) is trained on Qwen base checkpoints using this combined synthetic + limited open-source data pipeline.

📐 Key Formulas · 关键公式

\[\mathcal{D}_{t+1} = \mathcal{D}_t \cup \{x \mid x \in \text{ErrorSet}(\pi_t)\}\]
💡 推理飞轮的核心逻辑：下一轮训练数据集由当前数据集加上当前策略模型出错的样本构成，从错误中持续挖掘难例。 The reasoning flywheel's core idea: the next training dataset is the current dataset augmented with examples where the current policy model made errors, continuously mining hard cases from failures.
\[\mathcal{T}_{\text{branch}} = \text{Expand}(\mathcal{T}_{\text{linear}}, \mathcal{C})\]
💡 智能体飞轮的扩展操作：将线性工具调用流程 T_linear 在条件集合 C 下扩展为多分支行为树 T_branch，模拟真实决策复杂度。 The agentic flywheel's expansion: a linear tool-call workflow T_linear is expanded under condition set C into a multi-branch behavior tree T_branch, simulating real-world decision complexity.

🌟 Why It Matters · 为什么重要

在工业部署中，大型模型的推理成本和延迟往往是瓶颈，本文提供了一套系统性方法让小模型具备大模型的智能体能力，直接降低了落地门槛。双飞轮自动数据生成机制为未来智能体训练提供了可复用的范式，有望推动更广泛的自主AI应用。 For the AI field, this work shows a principled path to making small models viable for production agentic systems, dramatically reducing inference cost without sacrificing capability. The dual flywheel data synthesis paradigm is a reusable framework that could accelerate the development of autonomous AI agents across diverse domains.

Hyperloop Transformers 超循环Transformer

🧑‍🔬 Abbas Zeitoun, Lucas Torroba-Hennigen, Yoon Kim

图 Figure · Page 6 of paper

🎯 用约一半参数量超越标准Transformer的超循环架构
A looped Transformer with hyper-connections matches or beats standard models using ~50% fewer parameters.
维度 中文 English
❓ 待解决
的问题		 大语言模型越来越多地被部署在手机、嵌入式设备等内存有限的边缘端，但现有架构研究大多只关注算力和延迟，忽视了内存占用问题。普通Transformer通过堆叠独立层来提升效果，导致参数量随深度线性增长，不适合内存受限的场景。 Large language models are increasingly deployed on edge and on-device platforms where memory is tightly constrained, yet most architecture research focuses only on compute/latency efficiency. Standard Transformers scale quality by stacking more unique layers, which linearly grows parameter count and memory footprint, making them ill-suited for memory-limited settings.
💡 解决
的亮点		 • **循环Transformer作为核心模块**：中间层被反复复用（循环），而非每层独立，大幅减少参数量。 • **仅在循环边界施加超连接**：用矩阵值残差流（hyper-connections）取代标量残差，仅在每次循环后应用，参数增量极小但显著增强循环间的信息流动。 • **三块结构（始/中/终）**：只有中间块循环，首尾块保持独立，兼顾参数效率与表达能力。 • **Looped Transformer as core primitive**: Instead of using all unique layers, a middle block of Transformer layers is repeatedly reused (looped) across depth, drastically cutting parameter count. • **Hyper-connections on loop boundaries**: Matrix-valued residual streams (hyper-connections) replace scalar residuals, applied only after each loop iteration — adding minimal parameters but greatly enriching inter-loop information flow. • **Three-block structure (Begin / Middle / End)**: Only the middle block loops, while begin and end blocks remain unique, preserving representational flexibility at input/output boundaries.
🧪 实验
内容		 在多个模型规模下训练并在标准语言建模基准上进行评估。基线模型包括：深度匹配的标准Transformer（相同有效层数）和mHC Transformer（有超连接但不循环）。论文还测试了训练后权重量化（post-training quantization），以评估在真实部署条件下的内存效率。评估指标涵盖困惑度（perplexity）和多个常见NLP下游任务的表现。 Models were trained at multiple scales and evaluated on standard language modeling benchmarks. Baselines included depth-matched standard Transformers (same number of effective layers) and mHC Transformers (hyper-connected but not looped). The paper also tested post-training weight quantization to assess memory efficiency under real deployment conditions. Evaluation metrics covered perplexity and downstream task performance across common NLP benchmarks.
📊 实验
结果		 在各个模型规模下，Hyperloop Transformer在参数量减少约50%的情况下，性能仍优于深度匹配的标准Transformer和mHC Transformer基线。这一性能优势在训练后权重量化后依然保持，证明其在内存受限部署场景下的鲁棒性。具体基准测试结果显示，在所有测试规模下，困惑度和下游任务准确率均有持续提升。 Across various model scales, the Hyperloop Transformer outperforms both the depth-matched standard Transformer and the mHC Transformer baselines while using approximately 50% fewer parameters. The performance advantage is maintained after post-training weight quantization, confirming robustness in memory-constrained deployment scenarios. Specific benchmark scores showed consistent improvements in perplexity and downstream task accuracy across all tested scales.
🏁 结论 Hyperloop Transformer证明了将层循环与循环边界超连接相结合，可以构建出在约50%参数量下仍优于标准Transformer的高效大语言模型架构，且量化后性能优势依然保持。这使其成为内存受限的端侧语言模型部署的有力方案。 The Hyperloop Transformer demonstrates that combining layer-looping with loop-boundary hyper-connections yields a parameter-efficient LLM architecture that outperforms standard Transformers at ~50% of the parameter count, including after quantization. This positions it as a compelling solution for memory-constrained, on-device language model deployment.

⚙️ Method · 方法详解

Hyperloop Transformer将网络划分为三个连续模块：始块、循环中间块和终块。每个模块都由多个标准Transformer层组成，但只有中间块跨循环次数共享权重。每完成一次中间块的完整循环后，应用超连接（hyper-connections）——将1D标量残差流扩展为2D矩阵值残差流，使不同循环间积累的信息能够更丰富地融合。由于超连接仅插入在循环边界，而非每一层内部，额外引入的参数量极少。首尾块不参与循环，分别负责输入嵌入的专化处理和输出投影。 The Hyperloop Transformer divides the network into three sequential blocks: a begin block, a looped middle block, and an end block. Each block contains multiple standard Transformer layers, but only the middle block shares its weights across repeated applications (loops). After each complete pass through the middle block, hyper-connections are applied — these expand the standard 1D residual stream into a 2D matrix-valued stream, allowing richer mixing of information accumulated across loops. Because hyper-connections are only inserted at loop boundaries (not inside every layer), they introduce very few additional parameters. The begin and end blocks remain non-looped to handle input embedding specialization and output projection, respectively.

📐 Key Formulas · 关键公式

\[\mathbf{H}^{(l+1)} = \text{TransformerLayer}(\mathbf{H}^{(l)}), \quad l \in \{1, \ldots, L\}, \text{ with shared weights for middle block}\]
💡 中间块的每次循环复用同一组Transformer层权重，输入输出均为隐藏状态矩阵H，循环L次。 The middle block applies the same Transformer layer weights repeatedly across L loops, with H representing the hidden state matrix at each step.
\[\mathbf{R} \in \mathbb{R}^{T \times d} \rightarrow \mathbf{R} \in \mathbb{R}^{T \times d \times k}\]
💡 超连接将标准的1D残差流（序列长度×维度）扩展为矩阵值的2D残差流（序列长度×维度×超连接数k），增强循环间的信息融合。 Hyper-connections expand the standard 2D residual stream (tokens × hidden dim) into a 3D matrix-valued stream by adding a k-dimensional hyper-connection axis, enabling richer cross-loop information mixing.

🌟 Why It Matters · 为什么重要

随着大语言模型向边缘端和设备端部署迈进，如何在不牺牲性能的前提下缩减内存占用是一个关键且尚未解决的挑战——本文提供了一个原则性、实用的架构解决方案。通过将权重复用（循环）与更丰富的残差流（超连接）两个互补思路相结合，为LLM设计开辟了超越单纯堆叠参数的新维度。 As LLMs push toward edge and on-device deployment, reducing memory footprint without sacrificing quality is a critical unsolved challenge — this paper offers a principled, practical architectural solution. By combining two complementary ideas (weight reuse via looping and richer residual streams via hyper-connections), it opens a new design axis beyond simply scaling parameters.

ClawMark: A Living-World Benchmark for Multi-Turn, Multi-Day, Multimodal Coworker Agents ClawMark：面向多轮、多日、多模态协作智能体的真实世界基准测试

🧑‍🔬 Fanqing Meng, Lingxiao Du, Zijian Wu, Guanzheng Chen et al.

图 Figure · Page 4 of paper

🎯 首个面向多日多轮多模态工作流的持续演化环境智能体基准
A living-world benchmark that tests AI coworker agents across multi-day, multi-turn, multimodal workflows in evolving environments.
维度 中文 English
❓ 待解决
的问题		 现有的语言模型智能体评测基准通常只覆盖单次静态任务，且以纯文本为主，无法反映真实职场中AI"同事"需要跨越多个工作日、应对环境动态变化（如新邮件、日历更新、知识库修改）以及处理图像、音频、PDF等多模态信息的实际需求。这导致业界缺乏衡量智能体是否真正胜任持续性复杂工作流的有效工具。 Current benchmarks for language-model agents are designed around short, single-episode, mostly text-based tasks — they don't capture how real AI "coworkers" need to operate across multiple days, adapt to a changing environment (new emails, shifted calendar entries, updated knowledge bases), and process diverse modalities like images, audio, PDFs, and spreadsheets. This gap means we don't have good tools to evaluate whether agents can truly handle realistic, persistent workplace workflows.
💡 解决
的亮点		 • **动态演化环境**：基准测试引入了一个有状态的沙箱服务环境，其状态在智能体每次交互之间独立演化（如新邮件到达、日历变动、知识库更新），真实模拟外生环境变化，而非静态冻结场景。 • **无LLM裁判**：评分完全依赖1537个确定性Python检查器对执行后服务状态进行验证，彻底避免了LLM-as-judge带来的主观性与不稳定性。 • **多轮多日多模态覆盖**：任务横跨13个专业场景，涉及5类沙箱服务（文件系统、邮件、日历、知识库、电子表格）以及图像、PDF、音频、视频、表格等多种模态。 • **Living-world environment**: The benchmark introduces a stateful sandboxed service environment whose state evolves *between* agent turns, simulating real-world exogenous changes (new emails, calendar shifts, updated records) rather than a frozen episode. • **No LLM-as-judge**: Scoring is performed entirely by 1,537 deterministic Python checkers on post-execution service state, eliminating subjective or unreliable LLM-based evaluation. • **Multi-turn, multi-day, multimodal scope**: Tasks span 13 professional scenarios with five sandboxed services (filesystem, email, calendar, knowledge base, spreadsheet) and evidence spread across images, scanned PDFs, audio, video, and spreadsheets.
🧪 实验
内容		 研究对七个前沿智能体系统在13个专业场景的100个任务上进行了评测，任务执行于五类有状态沙箱服务之上。评测采用两项指标：加权得分（允许部分得分）和严格任务成功率（要求端到端全部完成）。同时进行了轮次级别分析，考察外生环境更新发生后智能体性能的变化趋势。 Seven frontier agent systems (including leading LLM-based agents) were benchmarked on 100 tasks across 13 professional scenarios using five stateful sandboxed services. Evaluation used two metrics: a weighted score (allowing partial credit) and strict Task Success Rate (requiring full end-to-end completion). Turn-level analysis was also conducted to measure how agent performance changes after exogenous environment updates occur mid-task.
📊 实验
结果		 最强模型的加权得分为75.8，但最佳严格任务成功率仅为20.0%，揭示了"部分完成"与"全流程完成"之间的巨大差距。轮次级别分析表明，所有七个被测系统在首次外生环境更新发生后性能均出现明显下滑，动态状态适应能力成为尚未解决的关键挑战。 The strongest model achieved a weighted score of 75.8, but the best strict Task Success Rate was only 20.0%, revealing a large gap between partial progress and complete workflow completion. Turn-level analysis confirmed that performance drops significantly after the first exogenous environment update, pinpointing dynamic state adaptation as a key unsolved challenge across all seven benchmarked systems.
🏁 结论 ClawMark建立了一个严格、可复现的基准测试，揭示了即使最强的当前AI智能体也难以端到端完成真实多日协作工作流，动态环境适应能力是核心开放挑战。基准、评测框架和构建流程全部开源，旨在推动持续性多模态协作智能体评测领域的发展。 ClawMark establishes a rigorous, reproducible benchmark revealing that even the best current AI agents struggle to complete real-world, multi-day coworker workflows end-to-end, with adaptation to dynamic environmental changes identified as the central open problem. The benchmark, harness, and construction pipeline are released to drive progress on persistent, multimodal coworker agent evaluation.

⚙️ Method · 方法详解

ClawMark在13个专业职场场景中构建了100个任务，每个任务在五类有状态沙箱服务上执行，这些服务的状态可以在智能体的不同交互轮次之间独立变化，模拟真实多日工作环境的动态演化。智能体需要在多轮交互中应对外生事件（如任务进行中新邮件到达）引发的状态变化。任务执行完成后，1537个确定性Python检查器检查服务的最终状态，分配部分或全额得分，实现细粒度评估，全程无需LLM裁判。基准还同步开源了评测框架与构建流程，支持可复现评测。 ClawMark constructs 100 tasks across 13 professional workplace scenarios. Each task is executed against five stateful sandboxed services that can change independently between turns, mimicking how a real work environment evolves over multiple days. Agents must handle multi-turn interactions where exogenous events (e.g., an email arriving mid-task) alter the state they need to reason about. After task execution, 1,537 deterministic Python checkers inspect the final service state and assign partial or full credit, enabling fine-grained evaluation without any LLM-based judging. The benchmark also releases the evaluation harness and construction pipeline for reproducibility.

📐 Key Formulas · 关键公式

\[\text{Weighted Score} = \frac{\sum_{i} w_i \cdot s_i}{\sum_{i} w_i}\]
💡 加权得分：每个子任务按其权重加权平均，衡量智能体在部分完成情况下的整体表现。 Weighted Score: each sub-task's score is weighted by its assigned weight, measuring overall partial-credit performance across all checkers.
\[\text{Task Success Rate} = \frac{\text{Number of fully completed tasks}}{\text{Total tasks}} \times 100\%\]
💡 严格任务成功率：要求智能体完全正确地完成整个端到端工作流才算成功，反映真实完成能力。 Strict Task Success Rate: a task counts as success only if all checkers pass, reflecting true end-to-end workflow completion ability.

🌟 Why It Matters · 为什么重要

随着LLM智能体被越来越多地部署为持续性职场助手，学术界亟需能够反映真实世界复杂性的评测工具，包括动态环境、多日工作流和多模态数据。ClawMark以严格、无LLM裁判的基准测试填补了这一空白，清晰诊断出即便是顶尖模型，完整端到端工作流完成仍是尚未解决的重大挑战。 As LLM agents are increasingly deployed as persistent workplace assistants, the field urgently needs evaluation tools that reflect real-world complexity — including dynamic environments, multi-day workflows, and multimodal data. ClawMark fills this gap with a rigorous, LLM-judge-free benchmark, providing a clear-eyed diagnosis that full workflow completion remains a largely unsolved problem even for frontier models.

❓ 待解决
的问题		 现有的统一多模态模型通常依赖预训练的视觉编码器（如VAE或CLIP类编码器）来处理图像。这导致理解任务和生成任务使用不同的视觉表示，造成两者之间的不对齐，同时也阻碍了从原始像素端到端的联合优化。 Most unified multimodal models depend on pretrained vision encoders (like VAEs or CLIP-style encoders) to process visual inputs. This creates a mismatch between the understanding and generation tasks, since they use separate visual representations. It also prevents truly end-to-end learning directly from raw pixels.
💡 解决
的亮点		 • 完全抛弃VAE、CLIP等模块化视觉编码器，仅用简单的图像块嵌入层（patch embedding），大幅简化模型架构。 • 在统一的像素空间范式下同时完成视觉理解与生成，消除两任务间的表示不对齐问题。 • 在大规模训练下，无编码器设计在细粒度视觉感知任务上超越基于编码器的变体，挑战了"预训练编码器不可或缺"的传统认知。 • Completely discards modular vision encoders (VAE, CLIP, etc.) and uses only simple patch embedding layers — a radical architectural simplification. • Unifies visual understanding and generation under a single pixel-space paradigm, eliminating representation misalignment between the two tasks. • Demonstrates that at scale, the encoder-free design surpasses encoder-based variants on fine-grained visual perception, challenging the assumption that pretrained encoders are necessary.

Baselines include both encoder-based unified multimodal models (using VAE or CLIP-type encoders) and latent-space generation approaches. The paper also conducts ablation studies comparing the encoder-free Tuna-2 against an encoder-based variant across different training scales to analyze convergence behavior and final performance.

⚙️ Method · 方法详解

Tuna-2用轻量级的图像块嵌入层（patch embedding）替换所有模块化视觉编码器，将原始像素块直接映射为语言模型骨干网络可处理的token嵌入。图像理解和图像生成均在同一像素空间表示下完成，实现从原始像素到输出的完全端到端优化。模型在理解与生成目标上联合训练，无需任何特定于任务的视觉编码器。同时还研究了带编码器的变体作为消融对比，分析无编码器设计在不同训练规模下的效果差异。 Tuna-2 replaces all modular vision encoders with lightweight patch embedding layers that directly map raw pixel patches into token embeddings fed to a language model backbone. Both image understanding (perception) and image generation are handled within the same pixel-space representation, enabling fully end-to-end optimization from raw pixels. The model is trained jointly on understanding and generation objectives, so a single unified architecture handles both tasks without task-specific visual encoders. An encoder-based variant is also studied as an ablation to isolate the effect of the encoder-free design at different training scales.

📐 Key Formulas · 关键公式

\[\mathbf{e}_i = \text{PatchEmbed}(\mathbf{p}_i)\]
💡 像素块嵌入：将原始图像的第i个像素块p_i直接映射为token嵌入向量e_i，无需任何预训练视觉编码器。 Patch embedding: each raw image patch p_i is directly mapped to a token embedding e_i via a simple linear projection layer, replacing all pretrained vision encoders.

🌟 Why It Matters · 为什么重要

本工作从根本上挑战了"强大的预训练视觉编码器是高性能多模态模型必要条件"这一主流设计假设，为更简洁、更统一的架构打开了大门。通过证明端到端像素空间学习具有良好的扩展性，可能深刻影响未来视觉语言模型的构建与训练方式。 This work fundamentally challenges the prevailing design assumption that powerful pretrained vision encoders are a prerequisite for competitive multimodal models, opening the door to far simpler and more unified architectures. By showing that end-to-end pixel-space learning scales favorably, it could reshape how future vision-language models are built and trained.
