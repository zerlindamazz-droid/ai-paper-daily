Trace2Skill: Distill Trajectory-Local Lessons into Transferable Agent Skills Trace2Skill：将轨迹局部经验提炼为可迁移的智能体技能

🧑‍🔬 Jingwei Ni, Yihao Liu, Xinpeng Liu, Yutao Sun et al.

图 Figure · Page 3 of paper

🎯 从多条执行轨迹中并行提炼可迁移的声明式智能体技能
Distill diverse LLM execution traces into transferable, declarative agent skills via parallel sub-agent analysis.
维度 中文 English
❓ 待解决
的问题		 LLM智能体在处理复杂任务时需要领域专属技能，但人工编写技能效率低、难以扩展。现有的自动化技能生成方法要么依赖模型内部的浅层知识，要么对单条轨迹过拟合，导致生成的技能脆弱、不具备泛化能力，换个任务或换个模型就失效。 LLM agents tackling complex tasks need domain-specific skills, but writing these skills manually doesn't scale. Automated skill generation methods either rely too heavily on the model's built-in knowledge (shallow) or overfit to single trajectories, producing brittle, non-generalizable skills that fail on new tasks or different models.
💡 解决
的亮点		 • **并行子智能体分析**：不再逐条处理轨迹，而是派遣一组子智能体并行分析多条多样化执行记录，从根本上避免顺序过拟合问题。 • **层级归纳合并**：将各轨迹提取的局部经验通过归纳推理逐层汇总，构建统一、无冲突的技能目录，模拟人类专家从广泛经验中提炼通用指南的过程。 • **跨模型与跨分布迁移**：由小模型（35B）进化出的技能可大幅提升大模型（122B）的表现，且无需参数更新或外部检索模块，可直接泛化到分布外场景。 • **Parallel fleet distillation**: Instead of processing trajectories one-by-one, Trace2Skill dispatches a fleet of sub-agents to analyze a diverse pool of executions simultaneously, avoiding sequential overfitting. • **Hierarchical inductive consolidation**: Trajectory-specific lessons are inductively merged into a unified, conflict-free skill directory, mimicking how human experts synthesize broad experience into reusable guides. • **Cross-model and OOD transferability**: Skills evolved by a smaller model (35B) can boost a larger model (122B) by huge margins, and generalize to out-of-distribution settings — with no parameter updates or retrieval modules required.
🧪 实验
内容		 实验覆盖三个具有挑战性的领域：电子表格操作（以Anthropic官方xlsx技能库为强基线）、视觉问答（VisionQA）和数学推理。基线包括人工编写技能、ReAct风格智能体和顺序技能进化方法。评估指标为任务成功率，测试集包含分布内和分布外（OOD）场景。跨模型迁移实验中，技能由Qwen3.5-35B生成，随后在Qwen3.5-122B上进行评估。 Experiments were conducted across three challenging domains: spreadsheet manipulation (using xlsx/CSV tasks with Anthropic's official skill library as a strong baseline), Visual Question Answering (VisionQA), and mathematical reasoning. Baselines included human-authored skills (e.g., Anthropic's official xlsx skills), ReAct-style agents, and sequential skill evolution methods. Evaluation measured task success rates on both in-distribution and out-of-distribution (OOD) test sets, and cross-model transfer was tested by evolving skills with Qwen3.5-35B and evaluating on Qwen3.5-122B.
📊 实验
结果		 在分布外基准WikiTableQuestions上，由Qwen3.5-35B进化的技能使Qwen3.5-122B智能体的成功率提升了高达**57.65个绝对百分点**。在电子表格任务上，Trace2Skill全面超越包括Anthropic官方xlsx技能库在内的强基线。三个测试领域（电子表格、视觉问答、数学推理）均取得显著提升，且进化技能在分布外场景和跨模型迁移中均表现出强健的泛化能力，所用开源模型最小仅需35B参数。 On WikiTableQuestions (OOD setting), skills evolved by Qwen3.5-35B improved a Qwen3.5-122B agent by up to **57.65 absolute percentage points**. Trace2Skill consistently outperforms strong baselines including Anthropic's official xlsx skill library across spreadsheet tasks. Gains were observed in all three domains (spreadsheet, VisionQA, math reasoning), with evolved skills showing robust generalization to out-of-distribution settings and cross-LLM transfer — all using open-source models as small as 35B parameters.
🏁 结论 Trace2Skill证明了通过并行、整体的轨迹分析与层级归纳合并，复杂的智能体执行经验可以被系统地提炼为高度可迁移的声明式技能——无需任何参数更新或检索模块。这为以可扩展、模型无关的方式为LLM智能体赋予跨模型、跨任务泛化的领域专业知识提供了切实可行的路径。 Trace2Skill demonstrates that complex agent experience can be systematically distilled into highly transferable, declarative skills through parallel, holistic trajectory analysis and hierarchical inductive consolidation — without any parameter updates or retrieval modules. This establishes a scalable, model-agnostic path for equipping LLM agents with domain expertise that generalizes across models and unseen tasks.
🔗 论文
链接		 arXiv: 2603.25158v2  ·  📥 PDF

⚙️ Method · 方法详解

Trace2Skill首先收集智能体在目标任务上的多样化执行轨迹。随后，一组并行子智能体各自独立分析部分轨迹，提取轨迹级别的经验教训（哪些做法有效、哪些失败、原因是什么）。这些局部经验通过归纳推理被逐层合并，最终生成一份统一、结构化、无冲突的技能目录。生成的技能为纯声明式文本，不需要对模型进行微调。推理时，智能体只需在行动前读取进化后的技能指南即可，支持对已有人工技能的深化改进，也支持从零创建全新技能。 Trace2Skill collects a diverse pool of agent execution trajectories on target tasks. A parallel fleet of sub-agents each independently analyzes a subset of these trajectories and extracts trajectory-specific lessons — identifying what worked, what failed, and why. These lessons are then hierarchically consolidated through inductive reasoning into a single, structured, conflict-free skill directory. The resulting skills are purely declarative text (no model fine-tuning), and can either deepen existing human-authored skills or create entirely new ones from scratch. At inference time, the agent simply reads the evolved skill guide before acting.

🌟 Why It Matters · 为什么重要

随着LLM智能体应用向更复杂领域扩展，人工技能工程已成为关键瓶颈，Trace2Skill提供了一种原则性的自动化替代方案，兼具可扩展性与真正的泛化能力。小型开源模型生成的技能能够大幅提升更大模型表现的发现，挑战了"智能体AI知识迁移必须依赖大规模参数"的固有假设，对整个领域具有重要启示。 Manual skill engineering for LLM agents is a critical bottleneck as applications scale to more complex domains; Trace2Skill offers a principled, automated alternative that is both scalable and genuinely generalizable. The finding that small open-source models can produce skills that improve much larger models challenges assumptions about the necessity of scale for knowledge transfer in agentic AI systems.

Meta-Harness: End-to-End Optimization of Model Harnesses Meta-Harness：模型执行框架的端到端优化

🧑‍🔬 Yoonho Lee, Roshen Nair, Qizheng Zhang, Kangwook Lee et al.

图 Figure · Page 2 of paper

🎯 自动搜索LLM执行框架代码，替代手工设计
An outer-loop optimizer that automatically searches over LLM harness code to replace hand-engineering.
维度 中文 English
❓ 待解决
的问题		 LLM系统的性能不只取决于模型本身，还依赖于"执行框架"（harness）——即负责决定检索、存储和传递哪些信息给模型的外围代码。这些框架目前大多依赖人工设计，费时费力且效果有限。现有的自动文本优化方法会过度压缩反馈信息，导致关键细节丢失，无法有效指导框架代码的改进。 LLM systems depend not just on model weights but on their "harness" — the surrounding code that decides what data to retrieve, store, and feed to the model. These harnesses are still mostly hand-crafted by engineers, which is time-consuming and suboptimal. Existing automated text optimizers compress feedback too aggressively and lose important details needed to improve harness code.
💡 解决
的亮点		 • 首次将"执行框架优化"作为独立问题提出，自动搜索包裹LLM的代码层，而非仅优化提示词或权重。 • 采用基于文件系统的记忆机制，让智能体提案器可以浏览所有历史候选方案的完整源码、执行轨迹和评分，保留丰富上下文而非压缩丢弃。 • 单次优化发现的执行框架可跨多个未见模型零样本迁移，证明所发现的策略具有广泛通用性。 • Introduces the concept of "harness optimization" as a first-class problem: automatically searching over the code layer that wraps LLMs, not just prompts or weights. • Uses a filesystem-based memory system so the agentic proposer can browse full source code, execution traces, and scores of ALL prior candidates — preserving rich context instead of compressing it away. • A single optimized harness generalizes across multiple unseen models (zero-shot transfer), demonstrating that discovered harnesses encode broadly useful strategies.
🧪 实验
内容		 实验覆盖三个不同LLM应用场景：（1）在线文本分类任务，以最先进的上下文管理系统为基线；（2）检索增强数学推理，在200道IMO级竞赛题上测试，并对5个未见模型进行迁移评估；（3）在TerminalBench-2基准上测试智能体编程任务。基线包括最先进的上下文管理系统、人工设计的执行框架以及标准文本优化器，同时考察分布内性能与对未见模型的零样本泛化能力。 Experiments covered three diverse LLM application domains: (1) online text classification using a context management system as baseline; (2) retrieval-augmented math reasoning evaluated on 200 IMO-level competition problems, testing transfer to five held-out models; and (3) agentic coding on the TerminalBench-2 benchmark. Baselines included a state-of-the-art context management system, hand-engineered harnesses, and standard text optimizers. The setup tested both in-distribution performance and zero-shot generalization to unseen models.
📊 实验
结果		 在在线文本分类任务上，Meta-Harness比最先进的上下文管理基线高出7.7个准确率百分点，同时仅使用后者1/4的上下文token。在检索增强数学推理任务（200道IMO级竞赛题）上，单个发现的执行框架在5个未见模型上平均提升准确率4.7个百分点。在智能体编程任务（TerminalBench-2基准）上，所发现的执行框架超越了最佳人工设计基线。 On online text classification, Meta-Harness outperforms the state-of-the-art context management baseline by +7.7 accuracy points while consuming 4× fewer context tokens. On retrieval-augmented math reasoning (200 IMO-level problems), a single discovered harness improves accuracy by an average of +4.7 points across five held-out models. On agentic coding (TerminalBench-2), discovered harnesses surpass the best hand-engineered baselines.
🏁 结论 Meta-Harness证明了通过文件系统保留完整历史经验（代码、轨迹、评分）能够实现有效的自动化执行框架工程，在多种LLM应用场景下超越人工设计方案。本文确立了"执行框架优化"作为提升LLM系统性能（超越提示词与权重调优）的可行且重要的新范式。 Meta-Harness demonstrates that richer access to prior experience — via a filesystem storing full code, traces, and scores — enables automated harness engineering that surpasses hand-crafted designs across diverse LLM application settings. It establishes harness optimization as a viable and impactful paradigm for improving LLM systems beyond prompt and weight tuning.

⚙️ Method · 方法详解

Meta-Harness作为外层优化循环包裹任意LLM应用。每轮迭代中，智能体提案器会查阅存储在文件系统中的所有历史候选框架的完整源码、执行轨迹和性能评分。基于这些丰富的历史信息，提案器通过编写或修改代码生成新的候选框架，如此循环往复，逐步提升框架质量。核心思路是：保留完整历史记录（而非摘要压缩）能让优化器更准确地判断哪些改动真正有效，从而做出更优的决策。 Meta-Harness operates as an outer optimization loop around any LLM application. At each iteration, an agentic proposer reviews the complete history of past harness candidates — including their source code, execution traces, and performance scores — stored in a structured filesystem. The proposer then generates a new harness candidate by writing or modifying code based on this rich prior experience. This cycle repeats, progressively refining the harness. The key insight is that keeping the full history accessible (rather than summarizing it) allows the optimizer to reason more carefully about what changes actually improve performance.

🌟 Why It Matters · 为什么重要

随着LLM系统日益复杂，外围基础代码对性能的影响愈发关键，但这一层面长期被自动化优化研究所忽视。Meta-Harness为全自动LLM系统工程化奠定了基础，有望大幅降低专家为每个新应用手动调优流水线的需求，推动AI系统开发的民主化。 As LLM systems grow more complex, the surrounding infrastructure code — not just the model itself — becomes a major determinant of performance, yet it has largely been ignored by automated optimization research. Meta-Harness opens the door to fully automated LLM system engineering, potentially reducing the need for expert ML engineers to hand-tune pipelines for every new application.

🎯 AI应以人为本，增强而非取代人类数学思维
A philosophical argument that AI should be human-centered, augmenting rather than replacing human mathematical thought.
维度 中文 English
❓ 待解决
的问题		 AI正快速渗透数学等高度复杂的认知领域，但其社会合理性、哲学基础以及对人类智识生活的长远影响，目前仍缺乏清晰阐述。AI的发展可能消耗大量资源并威胁技术人才的职业前景。本文核心问题是：AI与人类思维的关系究竟应该是怎样的，尤其在数学这类严格的领域中？ AI is rapidly being integrated into complex cognitive domains — including mathematics — yet its societal justification, philosophical grounding, and long-term impact on human intellectual life remain poorly articulated. 💡 解决
的亮点		 • 将AI定位为人类工具演化史（书写、印刷、计算机）的自然延续，而非颠覆性断裂。 • 明确倡导数学领域的"以人为本AI"范式：AI承担常规验证、检索与形式化工作，人类主导创造力与洞察力。 • 由菲尔兹奖得主陶哲轩联合撰写，为AI作为前沿数学研究协作者的愿景赋予了重要权威性。 • Frames AI as a natural continuation of humanity's long history of tool-building (writing, printing, computers), not a disruptive rupture. • Argues specifically for a "human-centered AI" paradigm in mathematics: AI handles routine verification, search, and formalization while humans drive creativity and insight. • Co-authored by Fields Medalist Terence Tao, lending significant authority to the vision of AI as a collaborator in frontier mathematical research.

📊 实验
结果		 本文不含定量实验结果。论文的"成果"属于规范性与定性层面：AI最宜被理解为延伸人类能力的认知工具；数学是人机协作最具前景的领域之一；社会必须确保AI发展与人类智识及社会价值观保持一致，而非单纯追求商业或效率目标。 No quantitative experimental results are reported. The paper's "results" are normative and qualitative: it concludes that AI is best understood as a cognitive tool extending human capability, that mathematics is a particularly promising domain for beneficial human-AI collaboration, and that society must ensure AI development remains aligned with human intellectual and social values rather than purely commercial or efficiency-driven metrics.

Its key contribution is a philosophically grounded, authoritative call — from one of the world's leading mathematicians — for AI to serve as a collaborator that amplifies human mathematical thought rather than supplanting it.

Shor's algorithm is possible with as few as 10,000 reconfigurable atomic qubits Shor算法仅需10,000个可重构原子量子比特即可实现

🧑‍🔬 Madelyn Cain, Qian Xu, Robbie King, Lewis R. B. Picard et al.

图 Figure · Page 4 of paper

🎯 万量子比特中性原子系统可破解工业级RSA与椭圆曲线加密
Shor's algorithm for breaking real-world cryptography needs only ~10,000 physical qubits, not millions.
维度 中文 English
❓ 待解决
的问题		 长期以来，人们认为运行Shor算法破解RSA-2048等工业级加密需要数百万个物理量子比特，使得近期实现量子密码攻击被视为遥不可及。现有量子纠错方案的开销极高，导致实用量子计算机的规模门槛居高不下。这篇论文要回答的问题是：能否大幅压缩这一量子比特需求，让密码学相关的量子计算在近期工程目标内成为可能？ Running Shor's algorithm on cryptographically relevant problems like RSA-2048 has traditionally been estimated to require millions of physical qubits due to the enormous overhead of quantum error correction. This places practical quantum cryptographic attacks far beyond near-term hardware capabilities. The paper asks: can we dramatically reduce this qubit count to a scale achievable with current and near-future neutral-atom technology?
💡 解决
的亮点		 • **高码率量子纠错码**：采用高码率QLDPC码替代传统表面码，大幅降低每个逻辑量子比特所需的物理量子比特数量。 • **高效逻辑指令集**：设计了适配中性原子平台的逻辑门指令集，充分利用原子阵列可重构特性实现高度并行化。 • **电路级联合优化**：将Shor算法的量子电路与底层硬件架构深度协同设计，将所需物理量子比特压缩至~10,000个，较此前最优估计降低约100倍。 • **High-rate quantum error-correcting codes**: Uses high-rate QLDPC codes instead of conventional surface codes, dramatically reducing the physical-to-logical qubit overhead ratio. • **Efficient logical instruction set**: Designs a hardware-native logical gate set tailored to reconfigurable neutral-atom arrays, maximizing parallelism. • **Co-designed circuit & architecture**: Deep co-optimization of Shor's algorithm circuits with the neutral-atom hardware architecture slashes qubit requirements by ~100× compared to prior best estimates.
🧪 实验
内容		 本文主要是理论分析与资源估算研究，而非传统意义上的实验论文。研究者对两个密码学标准目标——P-256椭圆曲线离散对数问题和RSA-2048整数分解——进行了详细的量子资源估算，分析了在不同物理量子比特规模（10,000至26,000+）下的运行时间与并行度之间的权衡，并与已有文献中基于表面码的最优资源估算进行了对比。 This is primarily a theoretical resource-estimation study rather than a hardware experiment. The researchers perform detailed quantum resource analyses targeting two cryptographic benchmarks: the P-256 elliptic curve discrete logarithm problem and RSA-2048 integer factorization. They analyze the tradeoff between physical qubit count and runtime across system sizes ranging from ~10,000 to 26,000+ qubits, and compare against prior state-of-the-art resource estimates based on surface codes.
📊 实验

⚙️ Method · 方法详解

研究团队从三个层面联合优化：首先，采用高码率量子低密度奇偶校验码（QLDPC）来代替传统的表面码，使每个逻辑比特只需更少的物理比特来保护；其次，针对中性原子平台的可重构阵列特性，设计了一套高效的容错逻辑指令集，支持大量逻辑门并行执行；最后，在算法电路层面对Shor算法的量子模幂运算进行深度优化，减少门数和电路深度。三者结合，将密码学相关计算所需的物理量子比特数从数百万大幅压缩至约10,000个。 The team co-optimizes three layers simultaneously. First, they replace traditional surface codes with high-rate QLDPC codes, which encode more logical qubits per physical qubit and reduce error-correction overhead. Second, they design a fault-tolerant logical instruction set exploiting the reconfigurability of neutral-atom arrays to enable massive parallelism. Third, they deeply optimize Shor's algorithm circuits — particularly modular exponentiation — to reduce gate counts and circuit depth. Together, these techniques compress the physical qubit requirement from millions down to roughly 10,000.
