Mamba-3: Improved Sequence Modeling using State Space Principles Mamba-3：基于状态空间原理的改进序列建模

🧑‍🔬 Aakash Lahoti, Kevin Y. Li, Berlin Chen, Caitlin Wang et al.

图 Figure · Page 4 of paper

🎯 Mamba-3用三项SSM改进突破线性模型质量瓶颈
Mamba-3 advances linear sequence models with three SSM-inspired improvements, beating prior models in quality and efficiency.

❓ Problem · 解决什么问题

大型语言模型（LLM）主要依赖Transformer架构，但其推理计算复杂度是序列长度的平方，内存占用也随之线性增长，推理成本极高。虽然已有研究提出了"亚二次"线性模型来降低计算量，但这些模型往往在模型质量上做出妥协，尤其在状态追踪等任务上表现欠佳，且理论上的线性推理在实际硬件上也并不高效。 Transformer-based LLMs suffer from quadratic compute and linear memory costs at inference time, making them expensive to deploy at scale. Existing sub-quadratic linear models reduce these costs but sacrifice model quality — particularly on tasks like state tracking — and their theoretical efficiency often doesn't translate to real hardware gains.

⚙️ Method · 方法

Mamba-3从"以推理为核心"的视角出发，引入了三项关键改进：第一，基于SSM离散化推导出表达能力更强的循环结构；第二，采用复数值状态更新规则，使模型能够追踪更丰富的状态信息；第三，引入多输入多输出（MIMO）公式，在不增加解码延迟的前提下提升模型性能。三项改进结合架构层面的优化，共同构成了Mamba-3模型。 Mamba-3 introduces three core methodological improvements grounded in the state space model (SSM) perspective. First, it derives a more expressive recurrence from SSM discretization principles. Second, it adopts a complex-valued state update rule to enable richer state tracking capabilities. Third, it introduces a multi-input, multi-output (MIMO) formulation that boosts model performance without adding decode latency. These are combined with broader architectural refinements to form the full Mamba-3 system.

📐 Key Formulas · 关键公式

\[h_t = \bar{A} h_{t-1} + \bar{B} x_t, \quad y_t = C h_t\]
💡 SSM的核心离散化循环公式：隐藏状态 $h_t$ 由上一时刻状态和当前输入共同更新，输出 $y_t$ 由当前状态线性映射得到。Mamba-3从此出发推导了更强的循环表达式。 The core discretized SSM recurrence: the hidden state $h_t$ is updated from the previous state and current input, and the output $y_t$ is a linear readout. Mamba-3 derives a more expressive recurrence from this foundation.
\[h_t \in \mathbb{C}^N\]
💡 Mamba-3采用复数值隐藏状态（而非实数值），使模型能够表达更丰富的周期性和旋转动态，从而大幅提升状态追踪能力。 Mamba-3 uses complex-valued hidden states (rather than real-valued), enabling the model to represent richer periodic and rotational dynamics, significantly improving state tracking performance.

📊 Results · 结果

在1.5B参数规模下，Mamba-3相比次优模型（Gated DeltaNet）平均下游任务准确率提升了0.6个百分点；其MIMO变体进一步提升1.2个百分点，累计总提升达1.8个百分点。在状态大小实验中，Mamba-3仅使用Mamba-2一半的状态维度，却能达到与Mamba-2相当的困惑度（perplexity），在检索、状态追踪和语言建模等任务上均有显著提升，有效推进了性能与效率的帕累托前沿。 At the 1.5B parameter scale, Mamba-3 improves average downstream accuracy by 0.6 percentage points over the next best model (Gated DeltaNet), with its MIMO variant adding a further 1.2 points for a total gain of 1.8 percentage points. Notably, Mamba-3 achieves comparable perplexity to Mamba-2 while using only half the state size, demonstrating improved parameter efficiency. Gains are observed across retrieval, state-tracking, and general language modeling benchmarks, advancing the performance-efficiency Pareto frontier.

🌟 Why It Matters · 为什么重要

随着"推理时扩展计算"成为提升LLM性能的重要方向，推理效率与模型质量的平衡变得至关重要。Mamba-3证明了线性模型不必以牺牲质量为代价换取效率，为构建既高效又强大的下一代语言模型提供了新路径。 As inference-time compute scaling becomes a key lever for LLM performance, building models that are both high-quality and hardware-efficient is critical. Mamba-3 demonstrates that linear models can close the quality gap with Transformers without sacrificing efficiency, offering a viable path toward next-generation language models that excel on both fronts.

Mixture-of-Depths Attention 混合深度注意力机制（MoDA）

🧑‍🔬 Lianghui Zhu, Yuxin Fang, Bencheng Liao, Shijie Wang et al.

图 Figure · Page 4 of paper

🎯 让注意力头同时关注当前层与历史深层特征，缓解深层LLM的信号衰减问题
MoDA lets attention heads access both current and past-layer KV pairs to combat signal degradation in deep LLMs.

❓ Problem · 解决什么问题

大型语言模型（LLM）越来越深，但深度增加并不总是带来更好的效果。浅层网络形成的有用信息特征，在经过一层层残差更新后会被逐渐"稀释"，到深层时已经很难恢复，导致模型性能打折扣。这一问题被称为"信号衰减"，是深度扩展的核心障碍。 As LLMs are scaled deeper, they suffer from signal degradation: useful features learned in shallow layers get progressively diluted through repeated residual updates and become difficult to recover in deeper layers. This undermines the expected gains from increasing model depth, limiting the effectiveness of depth scaling as a strategy for improving LLM performance.

⚙️ Method · 方法

MoDA为每个注意力头增加了一种新能力：除了像普通注意力一样关注当前层的序列KV对，还可以直接跨层关注来自前面各层的"深度KV对"，从而让深层模块直接复用浅层的信息特征，避免信号被反复稀释。为了解决跨层访问带来的非连续内存访问问题（这会降低GPU效率），论文设计了一套硬件高效算法，使其达到FlashAttention-2在序列长度64K时97.3%的效率。此外，研究还发现将MoDA与Post-Norm（后归一化）结合使用效果优于Pre-Norm（前归一化）。 MoDA introduces a new type of attention where each head can attend not only to the current layer's sequence KV pairs but also to "depth KV pairs" from preceding layers, directly bridging shallow and deep representations. This cross-layer attention lets deep layers recover and reuse informative features without relying solely on the residual stream. To handle the non-contiguous memory access patterns this creates on GPUs, the authors design a hardware-efficient algorithm that reorganizes memory access, achieving near FlashAttention-2 efficiency. They also find that pairing MoDA with post-norm normalization yields better results than pre-norm.

📐 Key Formulas · 关键公式

\[\text{MoDA}(Q, K^{\text{seq}}, V^{\text{seq}}, K^{\text{depth}}, V^{\text{depth}}) = \text{Attention}\left(Q,\ [K^{\text{seq}}; K^{\text{depth}}],\ [V^{\text{seq}}; V^{\text{depth}}]\right)\]
💡 MoDA的核心计算：每个注意力头的查询（Q）同时与当前层的序列KV对和来自前面各层的深度KV对做注意力计算，两部分KV在维度上拼接后统一处理。 The core of MoDA: each attention head's query (Q) attends jointly to the current layer's sequence KV pairs and depth KV pairs from preceding layers, which are concatenated together before computing standard attention.
\[\text{Overhead} = \frac{\Delta \text{FLOPs}}{\text{FLOPs}_{\text{baseline}}} \approx 3.7\%\]
💡 与基线模型相比，MoDA引入的额外浮点运算量（FLOPs）仅约为3.7%，表明其计算开销极小。 The additional floating-point operations introduced by MoDA relative to the baseline model is only approximately 3.7%, demonstrating that the cross-layer attention mechanism is computationally very lightweight.

📊 Results · 结果

在1.5B参数规模的模型上，MoDA在10个验证基准上的平均困惑度（Perplexity）降低了0.2，在10个下游任务上的平均性能提升了2.11%，而额外引入的计算开销仅为3.7% FLOPs。在硬件效率方面，其专用算法在序列长度64K时达到了FlashAttention-2效率的97.3%，几乎无性能损失。 On 1.5B-parameter models, MoDA improves average perplexity by 0.2 across 10 validation benchmarks and boosts average performance by 2.11% across 10 downstream tasks, with only a 3.7% FLOPs computational overhead. The hardware-efficient algorithm achieves 97.3% of FlashAttention-2's throughput at a sequence length of 64K, making the cross-layer attention practically deployable at scale.

🌟 Why It Matters · 为什么重要

深度扩展是提升LLM能力的核心路径之一，但信号衰减问题长期制约其上限。MoDA以极低的额外计算代价（仅3.7% FLOPs）有效缓解了这一瓶颈，为构建更深、更强的语言模型提供了新的基础组件。其硬件友好的实现方式也使其具备实际部署价值，而非仅停留在理论层面。 Depth scaling is one of the primary levers for improving LLMs, but signal degradation has been an underappreciated bottleneck limiting its effectiveness. MoDA offers a practical, hardware-efficient solution that consistently improves model quality at negligible cost, making it a valuable building block for future deep LLM architectures.

Code-A1: Adversarial Evolving of Code LLM and Test LLM via Reinforcement Learning Code-A1：通过强化学习对代码大模型与测试大模型进行对抗性协同进化

🧑‍🔬 Aozhe Wang, Yuchen Yan, Nan Zhou, Zhengxi Lu et al.

图 Figure · Page 2 of paper

🎯 双模型对抗框架让代码生成与测试生成相互博弈、共同进化
Adversarial two-model RL framework co-evolves a Code LLM and Test LLM with opposing objectives.

❓ Problem · 解决什么问题

用强化学习训练代码生成模型需要高质量的单元测试作为奖励信号，但高质量测试集非常稀缺，现有数据集覆盖有限，且静态奖励无法随模型能力提升而自适应调整。现有"自博弈"方法将代码生成与测试生成合并在同一个模型中，导致模型"自我勾结"——要么生成过于简单的测试以轻松获得奖励，要么因隔离访问而只能生成通用测试，无法发现特定实现中的隐藏bug。 Training code generation models with reinforcement learning requires high-quality unit tests as reward signals, but such test suites are scarce and existing datasets have limited coverage. Prior self-play approaches that use a single model for both code and test generation face a fundamental dilemma: white-box access causes self-collusion (trivial tests for easy rewards), while black-box restriction produces generic tests that miss implementation-specific bugs.

⚙️ Method · 方法

Code-A1 将代码生成与测试生成分离为两个独立的模型，赋予它们相互对立的目标：Code LLM 努力通过更多测试，Test LLM 努力暴露更多代码缺陷。由于两个模型相互独立，Test LLM 可以安全地以"白盒"方式检查候选代码，针对性地设计对抗性测试用例，同时完全避免自我勾结问题。此外，框架引入了"错误本"（Mistake Book）机制用于经验回放，记录历史失败案例以指导后续训练；并设计了复合奖励函数，在测试有效性与对抗难度之间取得平衡，整体采用强化学习端到端优化。 Code-A1 separates code generation and test generation into two distinct LLMs with opposing RL objectives: the Code LLM is rewarded for passing more tests, while the Test LLM is rewarded for exposing more defects. Because the models are architecturally independent, the Test LLM can safely inspect candidate code in a white-box manner to craft targeted adversarial tests without any risk of self-collusion. A "Mistake Book" mechanism records historical failure cases for experience replay, and a composite reward function balances test validity against adversarial difficulty, with both models trained end-to-end via reinforcement learning.

📐 Key Formulas · 关键公式

\[R_{\text{Code}} = \frac{1}{|T|} \sum_{t \in T} \mathbf{1}[\text{code passes } t]\]
💡 代码模型的奖励：通过的测试用例数量占总测试数量的比例，鼓励代码尽可能多地通过测试。 The Code LLM's reward is the fraction of test cases it passes, incentivizing it to write code that survives as many adversarial tests as possible.
\[R_{\text{Test}} = \alpha \cdot V(t) + \beta \cdot D(t, \text{code})\]
💡 测试模型的复合奖励：结合测试有效性 V(t)（测试本身是否正确）和对抗难度 D(t, code)（测试是否能暴露代码缺陷），通过权重 α 和 β 平衡两者。 The Test LLM's composite reward balances test validity V(t) (is the test itself correct?) and adversarial difficulty D(t, code) (does it expose bugs in the candidate code?), weighted by α and β.

📊 Results · 结果

在 Qwen2.5-Coder 系列模型上的实验表明，Code-A1 训练的代码生成模型性能与使用人工标注测试集训练的模型相当甚至更优，同时显著提升了测试生成能力。具体而言，Code-A1 在代码生成基准上匹配或超越了依赖人工标注测试的基线模型，而无需任何人工标注的测试套件，验证了对抗协同进化框架在消除对高质量静态测试集依赖方面的有效性。 Experiments on Qwen2.5-Coder models show that Code-A1 achieves code generation performance matching or exceeding models trained on human-annotated test suites, while simultaneously delivering significant improvements in test generation capability. Notably, these results are achieved without relying on any manually curated test annotations, demonstrating that the adversarial co-evolution framework can self-sufficiently replace static, human-labeled reward signals.

🌟 Why It Matters · 为什么重要

这项工作从根本上解决了代码强化学习中奖励信号质量与自适应性的核心瓶颈，为在无需人工标注测试集的情况下持续提升代码大模型提供了新范式。对抗协同进化的思路也可推广至其他需要"生成者—验证者"协同的领域，具有广泛的方法论意义。 This work directly addresses the core bottleneck of reward signal quality and adaptivity in code-focused reinforcement learning, offering a new paradigm for continuously improving code LLMs without expensive human-annotated test suites. The adversarial co-evolution principle is broadly applicable to other generator-verifier settings, making it methodologically significant beyond code generation.
