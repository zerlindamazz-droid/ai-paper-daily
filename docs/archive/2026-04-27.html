StructMem: Structured Memory for Long-Horizon Behavior in LLMs StructMem：面向大语言模型长时行为的结构化记忆

🧑‍🔬 Buqiang Xu, Yijun Chen, Jizhan Fang, Ruobin Zhong et al.

图 Figure · Page 3 of paper

🎯 结构化层级记忆框架，提升LLM长程对话推理能力
A hierarchical structured memory framework enabling LLMs to reason across long conversations efficiently.
维度 中文 English
❓ 待解决
的问题		 长期对话智能体需要记住对话历史中事件之间的关系，而不仅仅是孤立的事实，才能回答涉及多跳推理和时序关系的复杂问题。现有记忆方案面临两难困境：扁平记忆效率高但无法建模关系结构；基于图的记忆支持结构化推理，但构建代价昂贵且脆弱。如何在保留结构性推理能力的同时降低系统开销，是核心挑战。 Long-term conversational agents must capture relationships between events—not just isolated facts—to handle temporal reasoning and multi-hop questions over extended dialogues. Existing memory systems face a fundamental trade-off: flat memory is lightweight but lacks relational structure, while graph-based memory supports structured reasoning but is expensive and brittle to build. The challenge is achieving structured reasoning capability without the prohibitive computational cost.
💡 解决
的亮点		 • **双视角时序锚定**：从主客观双视角对事件进行时间戳绑定，保留事件级上下文绑定，避免信息丢失。 • **周期性语义整合**：定期对记忆进行语义层面的聚合与压缩，在保留跨事件连接的同时大幅减少token消耗。 • **层级结构设计**：在扁平记忆与图记忆之间找到平衡点，无需显式构建知识图谱即可支持多跳推理。 • **Dual-perspective temporal anchoring**: Events are grounded with timestamps from two complementary viewpoints, preserving event-level context bindings and preventing information loss. • **Periodic semantic consolidation**: Memory is periodically compressed and consolidated at the semantic level, maintaining cross-event connections while drastically cutting token usage. • **Hierarchical structure without explicit graphs**: Achieves structured relational reasoning without the fragile and expensive step of constructing explicit knowledge graphs.
🧪 实验
内容		 实验在长程对话基准数据集LoCoMo上进行，该数据集专门用于测试长期记忆和多跳推理能力。与多个代表性记忆系统基线进行对比，包括扁平记忆、基于图的记忆等方案。评估指标覆盖问答准确率（时序推理类和多跳类子任务），以及系统效率指标如token用量、API调用次数和运行时间。 Experiments were conducted on the LoCoMo benchmark, a dataset specifically designed to evaluate long-term conversational memory, temporal reasoning, and multi-hop question answering. StructMem was compared against multiple baseline memory systems including flat memory and graph-based memory approaches. Evaluation covered both task performance (accuracy on temporal reasoning and multi-hop QA subtasks) and system efficiency metrics including token usage, API call counts, and wall-clock runtime.
📊 实验
结果		 StructMem在LoCoMo基准上的时序推理和多跳问答子任务上均优于现有记忆系统基线。在效率方面，与图结构记忆方案相比，StructMem大幅降低了token消耗、API调用次数和运行时间（具体数字见原文及GitHub仓库），在保持甚至提升推理精度的同时实现了显著的计算节省。 StructMem outperforms prior memory system baselines on both temporal reasoning and multi-hop QA subtasks of the LoCoMo benchmark. On the efficiency front, compared to graph-based memory systems, StructMem substantially reduces token usage, API call counts, and runtime (specific figures available in the paper and GitHub repository), achieving notable computational savings while maintaining or improving reasoning accuracy.
🏁 结论 StructMem证明了通过层级化结构与双视角时序锚定，可以在无需显式知识图谱的前提下，同时提升长程对话智能体的推理能力与系统效率。该工作为设计可扩展、低成本的LLM长期记忆系统提供了新的范式。 StructMem demonstrates that hierarchical structure with dual-perspective temporal anchoring can simultaneously improve reasoning performance and reduce computational overhead in long-term conversational agents, without relying on explicit knowledge graphs. This work establishes a new paradigm for scalable, cost-efficient long-term memory in LLMs.

⚙️ Method · 方法详解

StructMem构建了一个层级化的结构增强记忆框架。它首先从双视角（如主观感受与客观事实）对每个对话事件进行时间锚定，形成带有时序绑定的事件级记忆单元。随后，系统周期性地对累积记忆进行语义整合，将相关事件归纳为更高层次的摘要节点，从而在不显式建图的情况下隐式编码跨事件关系。检索时，系统可以沿层级结构导航，定向支持时序推理和多跳问答，同时避免了图数据库的高昂构建与维护成本。 StructMem builds a hierarchy of structured memory units over long conversations. Each dialogue event is temporally anchored from dual perspectives (e.g., subjective and objective views), creating event-level bindings that preserve relational context. Periodically, the system performs semantic consolidation—merging related event memories into higher-level summary nodes—implicitly encoding cross-event connections without an explicit knowledge graph. During retrieval, the hierarchical structure allows targeted navigation to support temporal and multi-hop reasoning, while the consolidation step keeps token usage and API calls low.

🌟 Why It Matters · 为什么重要

随着LLM被部署为长期个人助手和对话智能体，如何让模型高效记忆并推理跨越数周乃至数月的对话历史，是走向实用化的关键瓶颈。StructMem提供了一种兼顾推理能力与计算效率的结构化记忆方案，直接推动了长程对话AI系统的落地可行性。 As LLMs are increasingly deployed as long-term personal assistants and dialogue agents, efficiently reasoning over months of conversation history is a critical bottleneck for real-world adoption. StructMem offers a practical structured memory solution that balances reasoning capability with computational efficiency, directly advancing the feasibility of deployed long-horizon conversational AI systems.

❓ 待解决
的问题		 大语言模型存在文化偏见已是共识，但以往研究主要关注"西方中心主义"偏向。实际上，当被问及开放性文化问题时，LLM究竟偏爱哪些国家或文化？这些偏见又是在训练的哪个阶段形成的？这些问题尚未得到系统研究。 LLMs are known to exhibit cultural biases, but prior work mostly focused on Western/Anglocentric skews. It was unclear which specific countries or cultures LLMs actually favor when answering open-ended cultural questions, and at what stage of training these biases emerge.
💡 解决
的亮点		 • 提出CROQ新基准数据集，构建了系统分类体系，能够量化衡量LLM的文化偏好。 • 发现反直觉现象：LLM的文化偏向并非简单的"西方中心主义"，而是显著偏爱日本，颠覆了以往研究的主流假设。 • 明确指出文化偏见首次出现在监督微调（SFT）阶段而非预训练阶段，为偏见消除提供了可操作的靶向建议。 • Introduces CROQ, a new benchmark of Culture-Related Open Questions with a comprehensive taxonomy, enabling systematic measurement of LLM cultural preferences. • Reveals a surprising counter-narrative: LLMs lean toward Japan rather than simply amplifying Western/US-centric views, challenging the dominant assumption in prior bias research. • Identifies supervised fine-tuning (SFT) — not pre-training — as the critical stage where cultural bias first clearly emerges, providing actionable insight for bias mitigation.
🧪 实验
内容		 研究使用自建的CROQ数据集，涵盖多类文化话题。实验用英语及多种高资源语言向多个主流LLM提问，并统计各国被提及的频率与多样性。此外，研究还对同一模型的不同训练阶段（预训练、SFT、RLHF）进行对比实验，以定位文化偏见的产生节点。 The study uses the newly created CROQ dataset, which spans a broad taxonomy of cultural topics. Multiple state-of-the-art LLMs are evaluated by prompting them in English and several other high-resource languages. Experiments also include probing different training-stage checkpoints of the same model (pre-training vs. SFT vs. RLHF) to isolate when cultural bias arises. The evaluation metric centers on country-mention frequency and diversity in model outputs.
📊 实验
结果		 在多个LLM的文化类问题回答中，日本被提及的频率最高，超过美国及其他西方国家，与以往"英语中心"偏见的结论相反。使用英语或其他高资源语言提问时，模型输出的地理多样性更高，且不会过度强调该语言对应的官方国家。文化偏见在SFT阶段后首次显著出现，基础预训练模型的偏斜程度明显更低。论文中报告了各模型、各语言下国家提及频率的具体数字分布。 Japan consistently emerges as the most frequently referenced country across multiple LLMs in cultural questions, surpassing both the US and other Western nations — a result that contradicts prior Anglocentric bias findings. When prompted in English or other high-resource languages, models produce more geographically diverse outputs and are less likely to over-represent the country whose language is used. Cultural bias first becomes statistically prominent after SFT, with base pre-trained models showing significantly less skew. Specific numeric breakdowns of country mention frequencies are reported across models and languages in the paper's experiments.
🏁 结论 本文证明LLM的文化偏见远比"西方中心主义"更复杂——模型系统性地偏爱日本文化，且这一偏见主要在监督微调阶段引入，而非预训练阶段。CROQ基准和上述发现为社区提供了新的评估工具，也指明了偏见干预的关键节点。 This paper proves that LLM cultural bias is more nuanced than a simple Western skew — models systematically favor Japanese culture, and this bias is largely introduced during supervised fine-tuning rather than pre-training. The CROQ benchmark and these findings provide the community with both a new evaluation tool and a targeted stage for bias mitigation.

Thinking Without Words: Efficient Latent Reasoning with Abstract Chain-of-Thought 无词思考：基于抽象思维链的高效潜在推理

🧑‍🔬 Keshav Ramji, Tahira Naseem, Ramón Fernandez Astudillo

图 Figure · Page 2 of paper

🎯 用抽象离散符号替代自然语言推理链，推理效率提升11.6倍
Replace verbose natural language reasoning chains with short abstract token sequences for 11.6× faster inference.
维度 中文 English
❓ 待解决
的问题		 大型语言模型在做复杂推理时，通常需要生成很长的"思维链"（CoT）文字过程，推理成本极高。现有的非语言推理方法虽然生成更短，但准确率明显落后于语言推理。如何在不牺牲性能的前提下大幅缩短推理长度，是亟待解决的关键问题。 Large language models performing complex reasoning must generate lengthy chain-of-thought (CoT) text, making inference expensive. Non-verbal reasoning methods generate shorter sequences but fall short in accuracy compared to verbalized CoT. The core challenge is drastically reducing reasoning length without sacrificing performance.
💡 解决
的亮点		 • **抽象思维链（Abstract-CoT）**：引入一段保留词表中的离散抽象token序列，作为自然语言CoT的替代，在生成答案前充当"内部思考"。 • **两阶段预热训练**：先通过遮蔽语言CoT进行监督微调（瓶颈压缩），再利用受限解码进行自蒸馏，使模型从提示直接生成抽象token。 • **强化学习优化**：预热完成后，使用暖启动RL在受限解码下进一步优化抽象序列生成，使性能进一步提升。 • **涌现幂律分布**：抽象词表上自发涌现出类似自然语言的幂律分布，揭示了学习到的"抽象推理语言"内在规律。 • **Abstract Chain-of-Thought**: A short sequence of discrete tokens from a reserved vocabulary replaces natural language CoT as a compact "inner monologue" before answering. • **Policy iteration warm-up**: Alternates between (1) bottleneck SFT via CoT masking and (2) self-distillation with constrained decoding, bootstrapping the abstract vocabulary from scratch. • **Warm-started RL**: After warm-up, reinforcement learning under constrained decoding further refines abstract token generation for task performance. • **Emergent power-law**: The abstract vocabulary spontaneously develops a Zipf-like power-law distribution across training phases, mirroring natural language statistics.
🧪 实验
内容		 实验覆盖数学推理（GSM8K、MATH）、指令遵循（IFEval）和多跳推理（MuSiQue）等多个基准。基线包括标准CoT、无CoT直接生成、以及COCONUT等连续潜在推理方法。实验在多个模型家族（包括不同规模的LLM）上进行，重点评估推理token数量与任务准确率之间的权衡。 Experiments span mathematical reasoning (GSM8K, MATH), instruction-following (IFEval), and multi-hop reasoning (MuSiQue) benchmarks. Baselines include standard verbalized CoT, no-CoT direct generation, and continuous latent reasoning methods such as COCONUT. Evaluations are run across multiple LM families to test generalization, measuring the trade-off between reasoning token count and task accuracy.
📊 实验
结果		 Abstract-CoT实现了最高 **11.6倍** 的推理token压缩，同时在数学推理、指令遵循和多跳推理任务上与完整语言CoT保持相当的性能。相比COCONUT等连续潜在推理基线，Abstract-CoT在准确率上取得明显优势，跨多个模型家族均一致有效。 Abstract-CoT achieves up to **11.6× fewer reasoning tokens** while maintaining comparable performance to full verbalized CoT across mathematical reasoning, instruction-following, and multi-hop reasoning benchmarks. It consistently outperforms continuous latent reasoning baselines such as COCONUT in accuracy, and the gains generalize across multiple LM families.
🏁 结论 本文证明，语言模型可以通过学习一套离散的抽象推理符号，在大幅减少推理开销的同时保持与语言思维链相当的性能，且这种"抽象推理语言"会自发涌现出类似自然语言的统计规律，为高效推理后训练提供了新范式。 This paper demonstrates that LLMs can learn a compact discrete abstract reasoning language that matches verbalized CoT performance at up to 11.6× lower token cost, with an emergent power-law structure in the learned vocabulary, establishing a new post-training paradigm for efficient latent reasoning.

⚙️ Method · 方法详解

Abstract-CoT将模型词表中的一部分token"保留"为抽象符号，模型在回答问题前先生成这些符号作为隐式推理过程。训练分两步：首先进行预热循环——交替做"遮蔽CoT监督微调"（把原有自然语言推理链遮蔽，迫使模型用抽象token承载信息）和"受限解码自蒸馏"（让模型仅从输入提示生成抽象token）；预热完成后，再用强化学习（RL）以任务奖励信号优化抽象序列的生成策略。整个流程为后训练机制，可跨模型家族通用。 Abstract-CoT reserves a subset of vocabulary tokens as "abstract" symbols; the model emits a short sequence of these before generating its final answer, acting as a compact latent reasoning trace. Training proceeds in two stages: a warm-up loop that alternates between masked CoT supervised fine-tuning (forcing information into abstract tokens) and self-distillation via constrained decoding (generating abstract tokens from the prompt alone); followed by warm-started reinforcement learning that optimizes abstract sequence generation using task-level reward signals. The entire pipeline is a post-training mechanism applicable across different LM families.

The emergent Zipf-like distribution over the abstract vocabulary also opens new avenues for understanding how models internally structure reasoning beyond natural language.

❓ 待解决
的问题		 标准全注意力Transformer在处理超长文本时，计算量和显存消耗急剧增长，推理效率极低。现有高效替代方案往往以牺牲模型性能为代价，或需要大量重训练。如何设计一个既能保持性能、又能高效处理长上下文、还能跨GPU和神经形态芯片部署的基础模型，是亟待解决的难题。 Full-attention Transformers become prohibitively slow and memory-hungry as context length grows, making long-sequence inference impractical. Existing efficient alternatives often sacrifice significant model performance or require massive retraining. There is a pressing need for foundation models that are efficient at long contexts, lightweight for edge deployment, and compatible across GPU and neuromorphic hardware platforms.
💡 解决

⚙️ Method · 方法详解

SpikingBrain2.0用双空间稀疏注意力（DSSA）替代全注意力，在不同层间交替部署MoBA（选取最相关的top-k块进行稀疏softmax注意力）和SSE（基于脉冲激活的稀疏线性注意力），只在最关键的层保留二次复杂度注意力，其余层采用线性复杂度，从而大幅降低长序列的显存和算力需求。模型通过优化的T2H流水线，从预训练Transformer（Qwen3-4B）出发，利用精选开源数据同步转换语言模型和多模态模型，无需大规模重训练。推理时提供两条路径：FP8路径用于高吞吐GPU部署，INT8-脉冲路径利用二进制脉冲激活在神经形态芯片上实现硬件级稀疏性和节能效果。 SpikingBrain2.0 replaces the standard full-attention mechanism with DSSA, which interleaves MoBA (a sparse softmax attention that selects top-k relevant blocks) and SSE (a sparse linear attention with spiking-inspired gating) across layers. This hybrid design keeps quadratic attention only where it matters most while using linear-complexity attention elsewhere, dramatically reducing memory and compute at long contexts. The model is initialized from a pretrained Transformer (Qwen3-4B) via an optimized T2H pipeline that jointly converts both LLM and VLM variants using curated open-source datasets, minimizing costly continued pretraining. At inference time, the model offers two execution paths: FP8 for high-throughput GPU deployment and INT8-Spiking for neuromorphic chips, leveraging binary spike activations to achieve hardware-level sparsity and power savings.

📐 Key Formulas · 关键公式

\[\text{DSSA}(x) = \text{MoBA}(x) \oplus \text{SSE}(x)\]
💡 双空间稀疏注意力（DSSA）在不同层间交替部署稀疏Softmax注意力（MoBA）和稀疏线性注意力（SSE），⊕表示层间交替组合，而非逐元素运算。 Dual-Space Sparse Attention alternates between sparse softmax attention (MoBA, for top-k block selection) and sparse linear attention (SSE, spiking-gated) across layers — ⊕ denotes inter-layer alternation, not element-wise addition.
\[\text{Sparsity} = 1 - \frac{\|s\|_0}{N}, \quad s = \text{Spike}(x) \in \{0, 1\}^N\]
💡 脉冲稀疏率定义为激活中零值脉冲占总神经元数的比例，SpB2.0实现了64.31%的激活稀疏率，直接映射为神经形态芯片上的节能效益。 Activation sparsity is the fraction of zero-valued spikes among all N neurons; SpB2.0 achieves 64.31% sparsity, which directly translates to energy savings on neuromorphic hardware.

🌟 Why It Matters · 为什么重要

随着LLM被推向百万级甚至更长的上下文处理，全注意力的显存与算力瓶颈日益突出，本文提供了一套有原则、有实际收益的混合解决方案。将GPU加速（FP8）与神经形态硬件兼容性（脉冲稀疏）统一于单一模型，为可持续的边缘AI部署开辟了超越纯软件优化的全新路径。 As LLMs are pushed to handle increasingly long contexts (e.g., 1M+ tokens), the memory and compute wall of full attention becomes a critical bottleneck — this work offers a principled hybrid solution with dramatic real-world efficiency gains. By unifying GPU acceleration (FP8) and neuromorphic hardware compatibility (spiking sparsity) in a single model, SpikingBrain2.0 opens a credible path toward sustainable, edge-deployable AI that goes beyond software optimization alone.
