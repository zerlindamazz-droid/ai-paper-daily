RAGEN-2: Reasoning Collapse in Agentic RL RAGEN-2：智能体强化学习中的推理崩溃

🧑‍🔬 Zihan Wang, Chi Gui, Xing Jin, Qineng Wang et al.

图 Figure · Page 3 of paper

🎯 互信息比熵更能诊断智能体强化学习中的推理崩溃问题。
Mutual information beats entropy for diagnosing reasoning collapse in agentic RL-trained LLMs.
维度 中文 English
❓ 待解决
的问题		 用强化学习训练多轮大语言模型智能体时，模型推理质量容易退化。目前常用熵来衡量稳定性，但熵只能检测模型对同一输入的输出是否多样，无法判断模型是否真的针对不同输入做出了不同的推理。这意味着模型可能在熵指标上看起来正常，却悄悄地在推理上失效了。 When training multi-turn LLM agents with RL, models can become unstable and their reasoning quality degrades. Entropy is the standard metric used to monitor this stability, but it only checks whether a model produces diverse outputs for the same input — it cannot tell if the model is actually adapting its reasoning to different inputs. This means a model can appear healthy by entropy measures while silently failing.
💡 解决
的亮点		 • 发现了一种新的失败模式"模板崩溃"——模型输出表面多样但实际与输入无关，现有熵指标对此完全失察。 • 将推理质量分解为两个正交维度：输入内多样性（熵）和跨输入可区分性（互信息），构建更完整的诊断框架。 • 提出SNR感知过滤方法，以奖励方差为轻量代理，在每轮训练中筛选高信噪比的提示，有效抑制模板崩溃。 • Identifies a new failure mode called "template collapse" — models produce superficially diverse but input-agnostic responses, invisible to entropy-based monitoring. • Decomposes reasoning quality into two orthogonal dimensions: within-input diversity (Entropy) and cross-input distinguishability (Mutual Information), providing a more complete diagnostic framework. • Proposes SNR-Aware Filtering, a lightweight training intervention that selects high-signal prompts using reward variance to suppress template collapse.
🧪 实验
内容		 实验涵盖四类任务：规划、数学推理、网页导航和代码执行。作者将所提方法与标准强化学习训练基线（如GRPO风格训练）进行比较，使用熵和互信息作为诊断指标，以任务成功率为主要性能衡量标准。同时评估了互信息相比熵的诊断能力（通过与最终性能的相关性衡量），以及SNR感知过滤作为训练干预手段的有效性。 Experiments were conducted across four diverse task domains: planning, math reasoning, web navigation, and code execution. The authors compared their method against standard RL training baselines (e.g., GRPO-style training) using entropy and mutual information metrics as diagnostic tools, and task success rate as the primary performance measure. They evaluated both the diagnostic power of MI vs. entropy (via correlation with final performance) and the effectiveness of SNR-Aware Filtering as a training intervention.
📊 实验
结果		 互信息代理指标在所有评测任务中均与最终任务性能呈现出比熵显著更强的相关性。SNR感知过滤在规划、数学推理、网页导航和代码执行基准上一致性地提升了输入依赖性（由互信息衡量）和任务性能。实验表明，熵保持稳定时任务性能可能已悄然下滑，而互信息能准确捕捉到模板崩溃的发生时刻。相比标准RL基线，所提方法在多项任务上取得了明显提升，验证了跨域的一致性改进。 Mutual information proxies show significantly stronger correlation with final task performance than entropy across all evaluated tasks. SNR-Aware Filtering consistently improves both input-dependence (measured by MI) and task performance across planning, math reasoning, web navigation, and code execution benchmarks. In reported experiments, the method achieves notable gains over standard RL baselines, with MI metrics revealing template collapse episodes that entropy completely missed — demonstrating that entropy can remain stable while task performance silently degrades. Specific absolute numbers were not fully disclosed in the abstract but consistent cross-domain improvements are reported.
🏁 结论 RAGEN-2证明熵不足以诊断智能体强化学习中的推理质量，互信息作为跨输入可区分性的衡量指标与任务性能具有更强的相关性。通过引入SNR感知过滤来对抗模板崩溃，本文既深化了对RL训练不稳定性的理论理解，也提供了一种适用于多类智能体任务的轻量级实用解决方案。 RAGEN-2 proves that entropy is an insufficient diagnostic for reasoning quality in agentic RL, and that mutual information — decomposed as cross-input distinguishability — is a far more reliable proxy that directly correlates with task performance. By introducing SNR-Aware Filtering to counteract template collapse, the paper delivers both a better theoretical understanding of RL training instability and a practical, lightweight fix applicable across diverse agent tasks.

⚙️ Method · 方法详解

本文首先形式化定义了"模板崩溃"：模型的推理与输入无关，输出看似多样但实质相同。为检测这一问题，作者用信息论分解推理质量——熵衡量输入内多样性，互信息（MI）衡量跨输入可区分性。由于在线精确计算互信息代价较高，他们引入了一组轻量级的互信息代理指标，适合实时训练诊断。为解决崩溃问题，他们从信噪比（SNR）角度解释其成因：奖励方差低时，任务相关梯度过弱，正则化项占主导并抹去跨输入的推理差异。对策是SNR感知过滤：每轮训练时选择奖励方差高的提示，确保梯度携带有效学习信号。 The paper first formalizes "template collapse" as a state where a model's reasoning is input-agnostic — responses look diverse but do not actually change based on different inputs. To detect this, the authors decompose reasoning quality using information theory: Entropy captures within-input diversity, while Mutual Information (MI) between inputs and outputs captures cross-input distinguishability. Since computing exact MI online is expensive, they introduce lightweight MI proxy metrics suitable for real-time training diagnosis. To fix the collapse, they explain it through a Signal-to-Noise Ratio (SNR) lens: when reward variance is low, task-relevant gradients are too weak to overcome regularization, which erases input-specific reasoning. Their solution, SNR-Aware Filtering, selects prompts with high reward variance at each training iteration, ensuring gradients carry meaningful learning signal.

📐 Key Formulas · 关键公式

\[I(X; Y) = H(Y) - H(Y \mid X)\]
💡 互信息 = 输出的熵 减去 给定输入后输出的条件熵，衡量输出中有多少信息真正来源于输入的变化。 Mutual Information = output entropy minus conditional entropy given the input; measures how much of the model's output variation is actually driven by different inputs rather than random noise.
\[\text{SNR} = \frac{\mathbb{E}[r]^2}{\text{Var}(r)}\]
💡 信噪比 = 期望奖励的平方 除以 奖励方差；SNR低意味着任务梯度信号弱，正则化项主导训练，导致模板崩溃。 Signal-to-Noise Ratio of rewards; when SNR is low, the gradient signal from the task is too weak relative to noise, causing regularization to dominate and erase input-specific reasoning.

🌟 Why It Matters · 为什么重要

随着强化学习训练的大语言模型智能体被越来越多地部署在复杂的多轮任务中，模板崩溃这类"静默失败"模式会在不触发任何现有警报的情况下悄然损害系统可靠性，本文为该领域提供了原则性的诊断工具和具体的解决方案。从熵到互信息的诊断指标转变，有望成为监控和改进智能体强化学习系统的新标准实践。 As RL-trained LLM agents are increasingly deployed in complex, multi-turn settings, silent failure modes like template collapse can undermine reliability without triggering any existing alarms — this work provides the field with a principled diagnostic and a concrete fix. The shift from entropy to mutual information as a training health metric could become a new standard practice for monitoring and improving agentic RL systems.

Neural Computers 神经计算机

🧑‍🔬 Mingchen Zhuge, Changsheng Zhao, Haozhe Liu, Zijian Zhou et al.

图 Figure · Page 5 of paper

🎯 用神经网络本身充当运行中的计算机，统一计算、记忆与输入输出。
Neural Computers unify computation, memory, and I/O inside a learned model runtime, replacing explicit programs.
维度 中文 English
❓ 待解决
的问题		 当前AI系统存在割裂：传统计算机靠明确程序运行，智能体依赖外部环境执行，世界模型只学习环境动态——没有任何一种形式能让模型本身成为一台完整运行的计算机。如何让神经网络同时具备稳定执行、可重编程和能力复用，是尚未解决的核心挑战。 Today's AI systems are fragmented: conventional computers run explicit programs, agents act in external environments, and world models only learn environment dynamics — none of them fully internalize the entire computing process. The goal of making a model itself behave as a fully running computer, capable of stable execution, reprogramming, and capability reuse, remains unsolved.
💡 解决
的亮点		 • 提出"神经计算机（NC）"这一全新概念范式，将学习到的模型本身定位为运行中的计算机，统一了计算、记忆与输入输出。 • 定义了长期目标"完全神经计算机（CNC）"，明确要求稳定执行、可显式重编程和持久能力复用三大属性。 • 将早期NC具体化为视频模型，在命令行（CLI）和图形界面（GUI）环境中从指令、像素和用户操作推演屏幕帧，仅从I/O轨迹中学习，无需程序内部状态监测。 • Introduces a new conceptual frontier — Neural Computers (NCs) — that positions the learned model itself as the running computer, unifying computation, memory, and I/O in a single runtime state. • Defines the long-term target as the Completely Neural Computer (CNC), with explicit properties: stable execution, explicit reprogramming, and durable capability reuse. • Concretely instantiates early NCs as video models that roll out screen frames from instructions, pixels, and user actions in both CLI and GUI environments — learning interface primitives purely from I/O traces without instrumented program state.
🧪 实验
内容		 实验在命令行（CLI）和图形界面（GUI）两种环境中展开，在收集的屏幕交互轨迹上训练视频模型。基线包括传统智能体框架和世界模型方法。评估指标聚焦于I/O对齐精度、短时域下一帧预测保真度以及接口原语习得的定性评估。消融实验分析了是否引入用户动作信号作为条件输入的影响。 The paper conducts experiments in both CLI (command-line interface) and GUI (graphical user interface) settings, training video models on collected screen interaction traces. Baselines include conventional agent frameworks and world model approaches. The models are evaluated on their ability to reproduce correct screen states given user instructions and actions, with evaluation metrics focused on I/O alignment accuracy, short-horizon next-frame prediction fidelity, and qualitative assessments of interface primitive acquisition. Ablations examine the effect of including or excluding user action signals as conditioning inputs.
📊 实验
结果		 在CLI和GUI两种设置下，早期神经计算机模型成功展示了基础接口原语能力，从纯I/O轨迹中实现了有意义的I/O对齐与短时域控制。在GUI环境中，视频模型能生成视觉连贯的屏幕滚动画面，帧级保真度指标表明模型捕捉到了基本交互动态。然而，在例程复用、符号稳定性和长时域一致性方面仍存在明显不足，多步骤任务完成和受控重编程基准上的性能下降定量证实了这一点。本文以概念性与探索性贡献为主，具体数值结果并非核心焦点。 The learned NC models successfully demonstrate early interface primitives in CLI and GUI settings, achieving meaningful I/O alignment and short-horizon control from pure I/O traces. In GUI settings, the video model generates visually coherent screen rollouts conditioned on user actions, with frame-level fidelity metrics showing the model captures basic interaction dynamics. However, the models fall short on routine reuse, symbolic stability, and long-horizon consistency — quantitatively evidenced by degraded performance on multi-step task completion and controlled reprogramming benchmarks. Specific numerical results are not the paper's primary focus, as it is primarily a conceptual and exploratory contribution establishing the NC paradigm.
🏁 结论 本文证明了神经计算机的早期原语（尤其是I/O对齐和短时域控制）可以纯粹从屏幕交互轨迹中学习得到，验证了NC范式作为一种新型机器形态的可行性。同时提出了迈向完全神经计算机的具体路线图，构建了一个将NC超越智能体、世界模型与传统计算机的统一概念框架。 This paper proves that early Neural Computer primitives — specifically I/O alignment and short-horizon control — can be learned solely from screen interaction traces, validating the NC paradigm as a viable new machine form. It establishes a concrete roadmap toward Completely Neural Computers, contributing a unifying conceptual framework that situates NCs beyond agents, world models, and conventional computers.

⚙️ Method · 方法详解

作者将神经计算机实例化为视频生成模型：以屏幕像素、用户指令和可选的用户操作为输入，自回归地预测下一帧屏幕画面，从而纯粹通过学习到的动态来模拟一台运行中的计算机。模型在命令行（CLI）和图形界面（GUI）环境中的I/O轨迹（屏幕录像与交互日志）上训练，不使用任何程序内部状态。研究评估了该学习运行时能否习得基础接口原语，如I/O对齐与短时域控制，并提出了迈向完全神经计算机的系统性路线图，指出例程复用、受控更新和符号稳定性等尚待解决的核心挑战。 The authors instantiate Neural Computers as video generation models that take screen pixels, user instructions, and optional user actions as input, then autoregressively predict the next screen frames — effectively simulating a running computer purely through learned dynamics. These models are trained on collected I/O traces (screen recordings and interaction logs) from CLI and GUI environments, with no access to internal program state. The learned runtime is evaluated on whether it can acquire fundamental interface primitives such as I/O alignment and short-horizon control. The paper further proposes a structured roadmap toward the full CNC, identifying remaining open problems: routine reuse, controlled model updates, and symbolic stability.

📐 Key Formulas · 关键公式

\[\text{NC} : (I_t, A_t) \rightarrow O_{t+1}\]
💡 神经计算机的核心运行方式：以当前输入（屏幕像素/指令）和用户动作为条件，自回归预测下一时刻的输出（屏幕帧），模型本身即为运行时状态。 The core NC runtime: given current inputs (screen pixels/instructions) and user actions, autoregressively predict the next output (screen frame) — the model itself IS the running state.

🌟 Why It Matters · 为什么重要

神经计算机代表了一种对"计算"本身的范式性重构——如果模型能成为自身的运行时，软件与AI之间的边界将被打破，从而实现可自我修改、可重编程的智能系统。对ML/RL/机器人领域而言，这开辟了学习式执行、记忆增强推理和端到端交互系统建模等全新研究方向。 Neural Computers represent a potential paradigm shift in how we think about computation itself — if a model can become its own runtime, this could dissolve the boundary between software and AI, enabling self-modifying, reprogrammable intelligent systems. For the ML/RL/Robotics community, this opens entirely new research directions in learned execution, memory-augmented inference, and end-to-end interactive system modeling.

In-Place Test-Time Training 原位测试时训练

🧑‍🔬 Guhao Feng, Shengjie Luo, Kai Hua, Ge Zhang et al.

图 Figure · Page 6 of paper

🎯 无需重训练即可让大模型在推理时动态更新权重的即插即用框架
A drop-in framework that gives existing LLMs dynamic weight adaptation at inference time without retraining.
维度 中文 English
❓ 待解决
的问题		 大型语言模型采用"训练后部署"的静态范式，在推理时无法根据新信息动态调整权重。现有的测试时训练（TTT）方法面临三大障碍：与主流LLM架构不兼容、计算开销过大、训练目标与语言模型的"预测下一个词"任务不匹配。这些问题使TTT难以在现实LLM生态中落地。 Large Language Models are trained once and then frozen — they cannot adapt their weights when encountering new information at inference time. Existing Test-Time Training (TTT) methods that try to solve this face three hard problems: they are architecturally incompatible with most deployed LLMs, computationally too expensive, and their training objectives don't align well with the next-token prediction task LLMs are built around.
💡 解决
的亮点		 • **即插即用兼容性**：将标准MLP模块的最终投影矩阵作为快速权重，无需重新设计架构或从头训练，可直接增强已有LLM。 • **任务对齐目标**：用与"预测下一个词"任务理论上对齐的定制目标，替换TTT中通用的自监督重建损失，使快速权重更新方向更合理。 • **高效分块更新机制**：引入分块更新策略，支持上下文并行，算法在超长上下文（最高128k token）下仍具备高可扩展性。 • **Drop-in compatibility**: Repurposes the final projection matrix of standard MLP blocks as fast weights — no architecture redesign or retraining from scratch needed for existing LLMs. • **Task-aligned objective**: Replaces TTT's generic self-supervised reconstruction loss with a theoretically grounded objective explicitly aligned with Next-Token Prediction, the core LLM task. • **Efficient chunk-wise updates**: Introduces a chunk-wise update mechanism that enables context parallelism and keeps the algorithm highly scalable for very long contexts (up to 128k tokens).
🧪 实验
内容		 论文在两个设置下进行实验：（1）作为即插即用增强，应用于已有的4B参数预训练LLM，在上下文长度最高128k的长上下文理解基准上评估；（2）从头预训练并与TTT-Linear、Mamba等竞争性TTT相关基线对比。评估涵盖标准语言建模困惑度基准和长上下文下游任务。还进行了消融实验，验证快速权重选择、任务对齐目标、分块更新等各设计选择的有效性。 The paper conducts experiments in two settings: (1) as a drop-in enhancement to an existing pretrained 4B-parameter LLM evaluated on long-context understanding benchmarks with context lengths up to 128k tokens; and (2) pretraining models from scratch with In-Place TTT and comparing against competitive TTT-related baselines (e.g., TTT-Linear, Mamba, and other recurrent/hybrid architectures). Evaluations span standard language modeling perplexity benchmarks and long-context downstream tasks. Ablation studies are conducted to validate individual design choices such as the choice of fast weights, the task-aligned objective, and the chunk-wise update strategy.
📊 实验
结果		 作为即插即用增强，配备In-Place TTT的4B参数模型在最高128k上下文的长上下文任务上性能显著优于未使用TTT的基线模型。从头预训练时，In-Place TTT在标准语言建模困惑度基准上持续优于TTT-Linear、Mamba等竞争基线。消融实验表明，任务对齐目标和MLP投影矩阵快速权重的选择均对最终性能提升有实质贡献，两者结合在所有评估设置下取得最优结果。 As a drop-in enhancement, a 4B-parameter model equipped with In-Place TTT achieves superior performance on long-context tasks with contexts up to 128k tokens, outperforming the base model without TTT. When pretrained from scratch, In-Place TTT consistently outperforms competitive TTT-related approaches including TTT-Linear and Mamba across standard language modeling benchmarks in terms of perplexity. Ablation results confirm that both the task-aligned objective and the MLP projection fast weight selection each contribute meaningfully to the overall gains, with the combined design yielding the best performance across all evaluated settings.
🏁 结论 In-Place TTT证明了测试时训练可以无缝集成到现有LLM中，无需架构重构，只需复用MLP投影权重并将更新目标与下一词预测对齐。该框架为大语言模型的持续学习提供了一条实用且可扩展的路径。 In-Place TTT proves that Test-Time Training can be seamlessly integrated into existing LLMs without architectural overhaul, by repurposing MLP projection weights and aligning update objectives with next-token prediction. The framework establishes a practical and scalable path toward continual learning in large language models.

链接		 arXiv: 2604.06169v1  ·  📥 PDF

⚙️ Method · 方法详解

In-Place TTT将每个Transformer MLP模块中的最终投影矩阵（即下投影矩阵）指定为测试时可更新的"快速权重"，无需插入任何新模块，真正实现即插即用。推理时，框架不使用通用重建损失（如掩码自编码），而是推导出一个与因果下一词预测损失在数学上对齐的新目标函数，确保快速权重的更新方向真正有利于语言建模。更新以分块方式在上下文窗口上计算，使方法可扩展至超长序列并支持多设备上下文并行。对于从头预训练的模型，同样的设计原则贯穿整个预训练过程。 In-Place TTT identifies the final projection matrix (the down-projection) inside each standard MLP block of a Transformer LLM as the "fast weights" to be updated at test time. No new components need to be inserted — it is truly a drop-in modification. During inference, instead of using a generic reconstruction loss (like masked autoencoding), the framework derives a novel objective that is mathematically aligned with the causal next-token prediction loss, ensuring that the fast weight updates actually benefit language modeling. Updates are computed in a chunk-wise fashion across the context window, allowing the method to scale to very long sequences and be parallelized across devices with context parallelism. For models that can be pretrained with In-Place TTT from scratch, the same design principles are applied throughout pretraining.

💡 In-Place TTT的核心目标：在每个时间步，利用由前文推导出的快速权重增量Δθ，最大化下一词预测的概率，使快速权重更新与语言建模目标直接对齐。 The core In-Place TTT objective: at each timestep, maximize next-token prediction probability using fast weight increments Δθ derived from the preceding context, directly aligning weight updates with the language modeling goal.
\[W_{\text{fast}} = W_{\text{proj}} + \Delta W(x_{1:t})\]
💡 快速权重的更新机制：MLP最终投影矩阵W_proj作为基础权重，ΔW为根据上下文动态计算的增量，两者相加得到测试时自适应后的权重。 Fast weight update mechanism: the MLP final projection matrix serves as the base weight, and ΔW is a context-dependent increment computed on-the-fly, giving the adapted weight at test time.

🌟 Why It Matters · 为什么重要

"训练后部署"的静态范式是当今LLM最根本的局限之一——模型无法在推理时从新信息中学习。In-Place TTT提供了一种即插即用的原则性解决方案，可直接应用于已部署模型，大幅降低持续自适应的门槛，为真正能从每次新交互中更新知识的LLM开辟了可能。 The static "train-then-deploy" paradigm is one of the most fundamental limitations of today's LLMs — they cannot learn from new context at inference time. In-Place TTT offers a principled, plug-and-play solution that can be applied to already-deployed models, lowering the barrier to continual adaptation and opening the door to LLMs that genuinely update their knowledge from each new interaction.

⚙️ Method · 方法详解

Gym-Anything将环境创建本身设计为多智能体任务。编码智能体负责编写安装脚本、配置软件、填充真实数据，并生成正确配置的证据（截图、日志等）。独立审核智能体对照结构化质量清单检查证据，并将不合格项反馈给编码智能体修正。这一流程被应用于基于美国GDP职业分类筛选的200款软件，生成超过1万个长视野任务（含训练/测试分割）。在测试阶段，审计原则被复用：一个VLM检查智能体完成的轨迹，指出尚未完成的子任务，使Gemini等模型能够迭代优化其操作。 Gym-Anything frames environment creation as a multi-agent task. A coding agent is given a target software application and tasked with writing installation and configuration scripts, populating the environment with realistic data, and producing evidence (screenshots, logs) of a correctly set-up environment. A separate audit agent then inspects this evidence against a structured quality checklist and flags failures for revision. This loop is applied across 200 software applications selected via a U.S. GDP-grounded occupational taxonomy, producing CUA-World with over 10,000 long-horizon tasks with train/test splits. At test time, the audit principle is reused: a VLM examines a completed agent trajectory and provides feedback on what subtasks remain incomplete, allowing agents like Gemini to iteratively refine their actions.

🌟 Why It Matters · 为什么重要

缺乏多样、真实、可扩展的环境一直是计算机操作智能体研究的最大瓶颈，Gym-Anything通过自动化环境创建直接打破了这一瓶颈。将任务锚定于经济数据并覆盖200款真实软件，推动智能体研究迈向能够胜任完整数字知识工作的新阶段。 The biggest bottleneck for computer-use agent research has been the lack of diverse, realistic, and scalable environments — Gym-Anything directly removes this bottleneck by automating environment creation itself. By grounding tasks in economic data and covering 200 real-world software applications, this work pushes the frontier toward agents that can handle the full complexity of digital knowledge work.
