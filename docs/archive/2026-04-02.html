S0 Tuning: Zero-Overhead Adaptation of Hybrid Recurrent-Attention Models S0调优：混合递归-注意力模型的零推理开销自适应方法

🧑‍🔬 Jack Young

图 Figure · Page 7 of paper

🎯 仅调优初始递归状态矩阵，实现零推理开销微调，超越LoRA
Tuning only the initial recurrent state matrix achieves zero-overhead PEFT, beating LoRA on code generation.
维度 中文 English
❓ 待解决
的问题		 高效微调大语言模型非常重要，尤其是在标注数据稀缺时。现有方法如LoRA会增加推理开销或在切换任务时需要合并权重。而混合递归-注意力模型拥有独特的"递归状态"结构，这一特性尚未被用于模型微调。 Fine-tuning large language models efficiently is critical, especially when labeled data is scarce. Existing methods like LoRA add trainable weight adapters that increase inference cost or require weight merging when switching tasks. Hybrid models combining recurrent layers and attention have a unique architectural feature — the recurrent state — that has not been explored as a tuning surface.
💡 解决
的亮点		 - **全新调优入口**：只优化每个递归层的初始隐藏状态矩阵（S0），所有模型权重完全冻结，是混合模型中全新的参数高效微调范式。\n- **零推理开销**：与LoRA不同，推理时无需额外计算；切换任务只需加载约48MB的状态文件，无需合并权重或重载模型。\n- **轨迹导向机制**：该方法通过在第一个token处引导递归轨迹发挥作用，解释了其在相关领域（代码→数学）的迁移能力，以及在结构差异大的任务（文本转SQL）上无法迁移的原因。 - **Novel tuning surface**: Optimizes only the initial hidden state matrix (S0) of each recurrent layer, leaving all model weights frozen — a completely new PEFT paradigm for hybrid models.\n- **Zero inference overhead**: Unlike LoRA, no extra computation is added at inference time; task switching requires only loading a ~48 MB state file with no weight merging or model reload.\n- **Trajectory-steering mechanism**: The method works by steering the recurrent trajectory from the very first token, explaining both its effectiveness on related domains (code→math) and its failure to transfer to structurally dissimilar tasks (text-to-SQL).
🧪 实验
内容		 实验在HumanEval（代码生成）、MATH-500（数学推理）、GSM8K（算术推理）和Spider（文本转SQL）四个基准上进行。主要基线为LoRA和前缀调优（Prefix Tuning）。测试模型包括Qwen3.5-4B（GatedDeltaNet混合架构）、FalconH1-7B（Mamba-2混合架构）和Qwen2.5-3B（纯Transformer，作为控制组）。训练数据约为48条执行验证的HumanEval解答，实验通过多随机种子（3-10个）验证统计显著性。 Experiments were run on four benchmarks: HumanEval (code generation), MATH-500 (math reasoning), GSM8K (arithmetic), and Spider (text-to-SQL). The main baselines are LoRA and prefix tuning. Models tested include Qwen3.5-4B (GatedDeltaNet hybrid), FalconH1-7B (Mamba-2 hybrid), and Qwen2.5-3B (pure Transformer as a control). Training uses only ~48 execution-verified HumanEval solutions, and statistical significance is verified across multiple random seeds (3–10 seeds per experiment).
📊 实验
结果		 在Qwen3.5-4B上，S0调优在HumanEval的贪心pass@1指标上提升+23.6±1.7个百分点（10个种子），超越LoRA达+10.8 pp（p<0.001）。在FalconH1-7B上，S0达到71.8%±1.3，与LoRA的71.4%±2.4统计上无显著差异（3个种子），但无推理开销。跨域迁移至MATH-500提升+4.8 pp（p=0.00002，8个种子），GSM8K提升+2.8 pp（p=0.0003，10个种子）。Spider（文本转SQL）无迁移效果。逐步状态偏移变体在Qwen3.5上达到+27.1 pp，但有逐步推理开销。纯Transformer Qwen2.5-3B上的前缀调优在所有9种配置下均下降-13.9 pp。 On Qwen3.5-4B, S0 tuning improves greedy pass@1 by +23.6 ± 1.7 pp on HumanEval (10 seeds), outperforming LoRA by +10.8 pp (p < 0.001). On FalconH1-7B, S0 reaches 71.8% ± 1.3 vs. LoRA's 71.4% ± 2.4 (statistically indistinguishable, 3 seeds) but with zero inference overhead. Cross-domain transfer to MATH-500 yields +4.8 pp (p = 0.00002, 8 seeds) and GSM8K +2.8 pp (p = 0.0003, 10 seeds). Spider (text-to-SQL) shows no transfer. A per-step state-offset variant achieves +27.1 pp on Qwen3.5 but adds per-step inference cost. Prefix tuning on the pure Transformer Qwen2.5-3B degrades by -13.9 pp across all 9 configurations tested.
🏁 结论 S0调优证明了混合语言模型中初始递归状态矩阵是一个强大且实用的参数高效微调入口，仅需约48条训练样本即可以零推理开销匹配甚至超越LoRA。这确立了递归状态初始化作为一种独特且高效的适应范式，专为混合递归-注意力架构量身定制。 S0 tuning proves that the initial recurrent state matrix in hybrid language models is a powerful and practical PEFT surface that can match or surpass LoRA with zero inference overhead, using only ~48 training examples. This establishes recurrent state initialization as a distinct and highly efficient adaptation paradigm uniquely suited to hybrid recurrent-attention architectures.

⚙️ Method · 方法详解

S0调优冻结所有Transformer和递归权重矩阵，仅为混合模型中每个递归层学习一个初始状态矩阵。该初始状态在模型处理任何输入token之前就已介入，将模型的递归动态"预热"并引导至任务相关的行为轨迹。训练过程中，仅更新这些小型状态矩阵，使用执行验证的代码解答作为监督信号。推理时，只需在前向传播前注入调优后的初始状态，相较于基础模型不增加任何计算开销。 S0 tuning freezes all transformer and recurrent weight matrices and instead learns a single initial state matrix for each recurrent layer in a hybrid model. This initial state acts as a "warm start" that biases the recurrent dynamics before any input token is processed, steering the model's internal trajectory toward task-relevant behavior. During training, only these small state matrices are updated using a verified supervision signal (execution-verified code solutions). At inference time, the tuned initial states are simply injected before the forward pass, adding zero computational overhead compared to the base model.

📐 Key Formulas · 关键公式

\[\hat{s}_0^{(l)} = s_0^{(l)} + \Delta s_0^{(l)}\]
💡 S0调优的核心：每个递归层l的初始状态由基础状态加上可学习的偏移量构成，只有偏移量Δs₀被训练，基础模型权重全部冻结。 The core of S0 tuning: the initial state of each recurrent layer l is the base state plus a learnable offset. Only the offset Δs₀ is trained; all base model weights remain frozen.
\[\text{pass@1} = \mathbb{E}_{\text{problem}}[\mathbf{1}[\text{greedy solution passes tests}]]\]
💡 HumanEval评估指标：对每道编程题，用贪心解码生成一个解答，若通过所有单元测试则计为1，取所有题目的平均值。 The HumanEval evaluation metric: for each problem, one solution is generated greedily; it scores 1 if it passes all unit tests, and the score is averaged across all problems.

🌟 Why It Matters · 为什么重要

随着混合递归-注意力模型（如Mamba和GatedDeltaNet混合架构）与纯Transformer的竞争力日益增强，本研究开辟了一种架构原生的调优维度，既数据高效又在推理时完全无开销。对于需要快速、低成本任务适配的从业者——尤其是在代码、数学或推理领域——S0调优提供了一种极具吸引力的LoRA替代方案，仅需一个小文件即可实现任务切换。 As hybrid recurrent-attention models (like Mamba and GatedDeltaNet hybrids) become increasingly competitive with pure Transformers, this work opens a new, architecturally-native tuning dimension that is both data-efficient and computationally free at inference. For practitioners needing rapid, low-cost task adaptation — especially in code, math, or reasoning domains — S0 tuning offers a compelling alternative to LoRA with the practical bonus of trivial task switching via a single small file.

Online Reasoning Calibration: Test-Time Training Enables Generalizable Conformal LLM Reasoning 在线推理校准：测试时训练实现可泛化的共形LLM推理

🧑‍🔬 Cai Zhou, Zekai Wang, Menghua Wu, Qianyu Julie Zhu et al.

图 Figure · Page 4 of paper

🎯 ORCA通过测试时校准将LLM推理计算成本降低高达67%
ORCA calibrates LLM sampling at test time to cut reasoning compute costs by up to 67% with formal guarantees.
维度 中文 English
❓ 待解决
的问题		 当前顶尖的大语言模型推理（如生成大量思维链）计算成本极高。根本原因在于模型本身"校准不足"——不知道什么时候该停止、需要多少样本。主流采样策略（如多数投票）也缺乏科学依据，导致在简单问题上浪费大量算力。 State-of-the-art LLM reasoning via test-time scaling (e.g., generating many long chains-of-thought) is extremely expensive. The root cause is that post-trained models are miscalibrated — they don't know when to stop or how many samples are actually needed. Popular sampling strategies like majority voting also lack principled calibration, wasting compute on problems the model can already solve easily.
💡 解决
的亮点		 • **共形预测 + 测试时训练的创新融合**：首次将共形预测（提供理论风险保证）与测试时训练（逐样本自适应）结合，用于LLM采样校准。 • **元学习校准模块**：通过轻量级元学习，对每个新输入动态更新校准头，实现对推理模式和提示分布漂移的实时适应。 • **无需标注标签**：ORCA可使用自一致性标签（无需人工标注），在实际部署中无需真实答案即可运行，极具实用价值。 • **Conformal + Test-Time Training fusion**: ORCA uniquely combines conformal prediction (for theoretical risk guarantees) with test-time training (for per-input adaptation), a novel pairing not previously explored for LLM sampling calibration. • **Meta-learning calibration module**: A lightweight meta-learning procedure updates the calibration head for each new input, enabling the model to adapt to distributional shifts in reasoning patterns and prompt distributions on the fly. • **Label-free operation**: ORCA works with self-consistency labels (no human supervision required), making it practical for real-world deployment without ground-truth answers.
🧪 实验
内容		 实验使用Qwen2.5-32B及其他模型家族，在MATH-500等数学和推理基准上测试。基线包括静态共形校准和标准多数投票采样。评估覆盖分布内和零样本跨域两种设置。在固定风险水平δ=0.1下，同时度量计算节省率（token/样本减少量）和实际误差率。分别评估了有监督标签和自一致性标签两个版本。 Experiments used Qwen2.5-32B and models from other families tested on math and reasoning benchmarks including MATH-500 and additional downstream tasks. Baselines included static conformal calibration and standard majority-voting sampling. Evaluation covered both in-distribution and zero-shot out-of-domain settings. Metrics tracked compute savings (token/sample reduction) and empirical error rate at a fixed risk level δ=0.1. Both supervised-label and self-consistency-label variants of ORCA were evaluated.
📊 实验
结果		 在风险水平δ=0.1下，ORCA在分布内任务上最高节省**47.5%**算力（有监督标签），使用自一致性标签时节省**40.7%**（Qwen2.5-32B）。在MATH-500零样本跨域设置下，ORCA将静态校准基线的节省率从**24.8%大幅提升至67.0%**，同时保持低实际误差率。这些增益在多个模型家族和下游基准上均一致成立。 At risk level δ=0.1, ORCA achieves up to **47.5% compute savings** on in-distribution tasks with supervised labels and **40.7%** with self-consistency (label-free) labels for Qwen2.5-32B. In zero-shot out-of-domain settings on MATH-500, ORCA improves savings from the static calibration baseline's **24.8% to 67.0%** while maintaining a low empirical error rate. These gains hold consistently across multiple model families and downstream benchmarks.
🏁 结论 ORCA证明了将共形预测与测试时元学习相结合，可以构建一个有原则、自适应的采样框架，在保持形式化精度保证的同时大幅降低LLM推理计算成本，即使在分布漂移下也成立。这是测试时扩展方法走向实用高效部署的重要一步。 ORCA proves that combining conformal prediction with test-time meta-learning yields a principled, adaptive sampling framework that dramatically reduces LLM reasoning compute costs while maintaining formal accuracy guarantees, even under distribution shift. It represents a significant step toward practical, efficient deployment of test-time scaling methods.

⚙️ Method · 方法详解

ORCA将采样过程建模为一个有校准的预测问题。对于每个新输入，通过元学习用少量在线样本（有监督标签或自一致性伪标签）快速更新轻量级校准模块。该模块输出置信度分数，决定达到目标精度所需的推理样本数量（如思维链条数）。共形预测理论为误差率提供形式化上界，即使在分布漂移下也能保证统计有效性。最终实现一个自适应采样器：模型有把握时提前停止，真正不确定时才投入更多算力。 ORCA frames the sampling process as a calibrated prediction problem. For each input, a small calibration module is updated via meta-learning using a handful of online samples (either supervised labels or self-consistency pseudo-labels). This module produces confidence scores that determine how many reasoning samples (e.g., chain-of-thought rollouts) are actually needed to achieve a target accuracy level. Conformal prediction theory is used to provide a formal bound on the error rate (risk), ensuring that the calibration is statistically valid even under distribution shift. The result is an adaptive sampler that stops early when the model is confident and invests more compute only when genuinely uncertain.

📐 Key Formulas · 关键公式

\[\mathbb{E}[L(\hat{Y}, Y)] \leq \delta\]
💡 共形风险保证：ORCA确保预测集合上的期望损失（错误率）被控制在用户设定的风险水平δ以内，提供统计有效性保证。 Conformal risk guarantee: ORCA ensures the expected loss (error rate) over the prediction set is bounded by the user-specified risk level δ, providing a formal statistical validity guarantee.
\[\hat{\theta}_x = \arg\min_{\theta} \mathcal{L}_{\text{meta}}(\theta; \{z_i\}_{i=1}^{n})\]
💡 测试时元学习目标：对每个输入x，通过优化元学习损失，利用少量在线样本{z_i}快速更新校准参数θ，实现逐样本自适应。 Test-time meta-learning objective: for each input x, the calibration parameters θ are updated by minimizing a meta-learning loss over a small set of online samples {z_i}, enabling per-input adaptation.

🌟 Why It Matters · 为什么重要

随着LLM越来越依赖昂贵的测试时算力（长思维链、大量采样）处理复杂推理任务，校准成为高效部署的关键瓶颈。ORCA提供了一个有理论支撑、可实际部署的解决方案，可在不牺牲可靠性的前提下将计算成本削减一半以上，直接影响前沿AI系统的运营经济性。 As LLMs increasingly rely on expensive test-time compute (long chains-of-thought, many samples) for hard reasoning tasks, calibration becomes a critical bottleneck for cost-effective deployment. ORCA offers a theoretically grounded and practically deployable solution that can slash compute by over half without sacrificing reliability, directly impacting the economics of running frontier AI systems.

Screening Is Enough 筛选即足够

🧑‍🔬 Ken M. Nakanishi

图 Figure · Page 4 of paper

🎯 用阈值筛选替代Softmax注意力，参数减少40%性能不降
Multiscreen replaces softmax attention with threshold-based screening, achieving 40% fewer parameters with comparable performance.
维度 中文 English
❓ 待解决
的问题		 标准Softmax注意力机制的本质是把固定的"总权重"分配给所有键（Key），因此相关性只能是相对的，模型无法直接"忽略"某个不相关的键。这种全局竞争机制导致计算冗余，在超长上下文场景中尤为明显，大多数键其实毫不相关却仍被迫参与计算。 Standard softmax attention forces the model to assign weights to ALL keys simultaneously, meaning relevance is always relative — there's no way to simply say "this key is irrelevant, ignore it." This global competition among keys causes inefficiency and makes it hard for the model to truly reject noise. The problem compounds in long-context settings where most keys are irrelevant.
💡 解决
的亮点		 • **绝对相关性筛选**：每个键独立与显式阈值比较，不相关的键被直接丢弃，而非仅被降权，从根本上改变了注意力的计算逻辑。 • **消除全局键竞争**：键之间不再相互竞争注意力权重，各自独立判断是否被聚合，结构更简洁。 • **参数效率与推理提速**：在相近验证损失下参数量减少约40%，100K上下文长度下推理延迟降低最高3.2倍。 • **更好的长度外推**：在训练上下文长度之外的检索任务中几乎无性能退化，解决了Transformer的已知痛点。 • **Absolute relevance via screening**: Instead of softmax redistribution, each key is independently evaluated against an explicit threshold — irrelevant keys are hard-discarded, not just down-weighted. • **No global key competition**: Keys are evaluated independently, removing the architectural coupling that forces all keys to "compete" for attention mass. • **Parameter efficiency + speed**: Achieves comparable language modeling quality with ~40% fewer parameters and up to 3.2× faster inference at 100K context length. • **Better length generalization**: Retrieval performance degrades little to none beyond the training context length, a known weakness of standard Transformers.
🧪 实验
内容		 作者将Multiscreen作为语言模型与标准Transformer基线在验证困惑度（语言建模损失）上进行对比，涵盖多个模型规模。长上下文评估测试超出训练长度的序列困惑度。检索任务（类"大海捞针"形式）测试在远超训练上下文长度时定位关键信息的能力。推理延迟在100K上下文长度下实测。同时测试更大学习率下的优化稳定性。 The authors train and evaluate Multiscreen as a language model, comparing it against a standard Transformer baseline on validation perplexity (language modeling loss). They test across different model sizes to examine parameter efficiency. Long-context evaluations measure perplexity on sequences longer than the training length. Retrieval benchmarks (e.g., needle-in-a-haystack style tasks) probe the model's ability to locate relevant information far beyond the training context. Inference latency is measured at 100K context length to quantify wall-clock speedup. Learning rate stability is also tested to show optimization advantages.
📊 实验
结果		 • 达到相近验证损失时，参数量**减少约40%**。 • 在100K上下文长度下推理延迟**降低最高3.2倍**。 • 支持**显著更大的学习率**且训练稳定，优化鲁棒性提升。 • 在远超训练上下文长度的检索任务中**几乎无性能退化**，长度外推能力强。 • 长上下文困惑度保持稳定，无标准Transformer常见的退化现象。 • **~40% fewer parameters** to reach comparable validation loss vs. Transformer baseline. • **Up to 3.2× reduction in inference latency** at 100K context length. • Supports **substantially larger learning rates** with stable training (specific LR multiples not stated in abstract but described as significant). • **Little to no degradation** in retrieval performance even far beyond the training context window — strong length generalization. • **Strong long-context perplexity** maintained beyond training length, contrasting with typical Transformer degradation.
🏁 结论 本文证明，用基于阈值的绝对筛选机制取代Softmax的相对权重分配，不仅足够支撑语言建模，而且在参数效率、推理速度和长度外推上全面超越标准Transformer。Multiscreen为语言模型提供了一个极具说服力的Softmax注意力替代方案。 This paper proves that replacing softmax's relative redistribution with a threshold-based absolute screening mechanism is sufficient — and in fact superior — for language modeling, delivering dramatic gains in parameter efficiency, inference speed, and length generalization. Multiscreen establishes screening as a viable and compelling alternative to softmax attention as a foundational building block for language models.

⚙️ Method · 方法详解

Multiscreen的核心操作"筛选（Screening）"：对每个查询（Query），将每个键（Key）的得分与一个阈值进行独立比较，低于阈值的键直接被丢弃，只对剩余相关键做加权求和聚合。这彻底替代了Softmax归一化——后者必须看所有键才能算分母。由于键之间不再需要全局通信，长上下文下计算量大幅降低。整个Multiscreen语言模型以此筛选注意力为核心模块构建。 Multiscreen introduces a "screening" operation: for each query, each key is independently scored and compared against a learned or dynamic threshold. Keys that fall below the threshold are discarded entirely before aggregation — the model computes a weighted sum only over the surviving, relevant keys. This replaces the softmax normalization step, which inherently requires all keys to be present for the denominator computation. Because keys are evaluated independently, the architecture naturally avoids global communication across the full context, enabling both parameter savings and faster long-context inference. The overall language model architecture (Multiscreen) is built around this screening attention as its core building block.

📐 Key Formulas · 关键公式

\[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]
💡 标准Softmax注意力：将所有键的得分归一化为概率分布后加权求和，所有键全局竞争权重。 Standard softmax attention: scores across all keys are normalized into a probability distribution and used to weight values — every key competes globally.
\[\text{Screening}(q, K, V) = \frac{\sum_{i: s(q, k_i) \geq \tau} s(q, k_i) \cdot v_i}{\sum_{i: s(q, k_i) \geq \tau} s(q, k_i)}\]
💡 筛选注意力（概念公式）：只对得分超过阈值τ的键进行聚合，低于阈值的键被直接丢弃，实现绝对相关性判断。 Screening attention (conceptual): only keys whose score exceeds threshold τ are aggregated; irrelevant keys are hard-discarded, enabling absolute rather than relative relevance.

🌟 Why It Matters · 为什么重要
